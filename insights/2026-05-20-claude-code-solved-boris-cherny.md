## 픽업
- Claude Code 는 의도적으로 PMF 6개월 전부터 "다음 모델" 을 겨냥해 만들었다. 처음 6개월은 거의 안 쓸 정도로 안 좋았고 초기 출시도 히트가 아니었다. 지수 성장은 5월 Opus 4 에서 시작돼 4.5·4.6·4.7 매 릴리스마다 꺾여 올라갔다.
- "나에겐 코딩은 100% 해결됐다." 작년 10~11월부터 모델이 자기 코드를 100% 작성한다. 하루 수십 PR, 지난주엔 한계를 보려고 하루 150 PR 까지 찍었다. Claude Code 코드베이스는 TypeScript+React 로 단순한데, 시작 당시 모델이 덜 똑똑해서 "분포에 잘 맞는" 언어/프레임워크가 중요했기 때문이다.
- 이제 작업 대부분을 폰에서 한다. Claude 앱 코드 탭에 세션 5~10개, 그 안에 수백 개 에이전트, 매일 밤 수천 개가 더 깊은 작업을 돌린다.
- 가장 즐겨 쓰는 건 /loop — Claude 가 cron 으로 반복 작업을 스케줄하는 "되는 것 중 가장 단순한 것". PR 돌보기(CI 고치고 자동 리베이스), CI 헬스 유지(플레이키 테스트 수정), 30분마다 트위터 피드백 클러스터링 등 수십 개 루프를 상시 돌린다. "루프가 미래다." 노트북을 닫아도 서버에서 도는 routines 도 같은 개념.
- AI 가 비즈니스 "파워" 중 일부를 약화시킨다 — 전환비용(모델로 갈아타기 쉬움), 프로세스 파워(4.7 은 목표만 주면 끝날 때까지 알아서 hill-climb). 반면 네트워크 효과·규모의 경제·독점 자원은 그대로다. 작게 시작해 AI 네이티브로 만든 스타트업이 대기업과 정면 경쟁 가능해, 향후 10년 파괴적 스타트업 수가 10배 늘 것이다.
- 1400년대 인쇄술이 유럽 문해율을 10%→70%로 끌어올렸듯, 소프트웨어도 누구나 하는 기술로 완전 민주화될 것이며 그 속도는 50년보다 훨씬 빠를 것이다. 회계 소프트웨어를 가장 잘 만들 사람은 엔지니어가 아니라 뛰어난 회계사다 — 코딩은 쉬운 부분이고 도메인 지식이 어려운 부분이기 때문이다.
- Anthropic 의 진짜 우위는 기술이 아니라 조직/프로세스다(같은 모델은 모두에게 열려 있음). 사내에서 Claude 를 모든 일에 쓰며, 코딩 중인 Claude 들이 Slack 으로 서로 대화해 미지의 것을 풀어낸다. 수작업 코드는 회사 어디에도 없고 모든 SQL 도 모델이 짠다. 모델이 좋아질수록 하니스 비중은 줄고, 권한 모드·휴먼인더루프 같은 안전장치도 덜 중요해질 것이다.

# AAB 40MB 파일을 AppleScript 로 직접 못 넣어서 막힘

- **발생 일자:** 2026-04-15
- **해결 일자:** 2026-04-15
- **심각도:** medium (업로드 자동화 블로킹)
- **재발 가능성:** medium (다른 대용량 업로드 경로 재현 가능)
- **영향 범위:** 메모요 AAB 업로드 자동화 (Play Console)

## 증상
Play Console 에 AAB 를 업로드하려고 AppleScript 파일 입력 다이얼로그로 넣으려 했으나, 40MB 이상 되는 파일을 단일 호출로 전달할 수 없어 자동화가 중간에 끊김. 수동으로 파일 선택창까지 진입은 되지만 파일 경로 전달 단계에서 AppleScript 가 응답 없음 상태로 빠짐.

## 원인
AppleScript-osa 브리지가 단일 호출에서 다룰 수 있는 **페이로드 크기 상한** 에 걸림. 텍스트 전달처럼 다뤘더니 실제로는 파일 내용을 메모리에 통째로 올리려다가 실패. 바이너리 대용량을 AppleScript 한 줄로 주입하는 경로 자체가 부적합.

## 조치
- 파이썬으로 AAB 파일을 `base64` 로 인코딩
- 2MB 씩 28개 청크로 분할해서 AppleScript 로 순차 전달
- AppleScript 쪽에서는 청크를 Blob 조립 방식으로 이어 붙여 임시 파일로 재구성
- 재구성된 파일 경로를 Play Console 업로드 다이얼로그에 전달
- 업로드 성공 확인

## 예방 (Forcing function 우선)
- 앞으로 파일 업로드 자동화는 AppleScript 대신 **파일을 먼저 디스크에 두고 경로만 전달** 하는 패턴을 기본값으로. AppleScript 브리지는 "텍스트" 전달에만 쓰고 바이너리는 쓰지 않는다.
- 이 규칙을 메모요 업로드 스크립트 상단 주석에 박아둠.
- 향후 새 자동화 작성 시 1MB 이상 페이로드가 예상되면 base64 청크 방식 대신 `osascript` 에 파일 경로만 넘기는 방식을 우선 검토.

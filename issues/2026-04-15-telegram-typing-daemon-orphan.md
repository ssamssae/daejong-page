# 텔레그램 typing-start 데몬이 세션 종료 후에도 살아남아 누적

- **발생 일자:** 2026-04-15
- **해결 일자:** 2026-04-15
- **심각도:** medium (리소스 누수, 다음 세션 혼선)
- **재발 가능성:** medium (다른 장시간 데몬에 같은 패턴 재현 가능)
- **영향 범위:** Claude Code 세션 수명주기에 딸린 백그라운드 데몬 전반

## 증상
세션 종료 후에도 `typing-start` 데몬 프로세스가 살아 있어서 다음 세션이 시작될 때 또 붙음. 반복 세션을 열 때마다 같은 이름의 데몬이 누적돼 프로세스 리스트에 여러 개 보이고, 텔레그램 쪽에서도 "입력 중" 표시가 엉뚱한 타이밍에 뜨는 증상이 따라옴.

## 원인
데몬 종료 조건이 "같은 PID 의 부모 세션이 살아있는지" 기반이 아니라 **idempotency 가 약한 heuristic** 에 의존하고 있었음. 세션이 정상 exit 하지 않고 강제 종료되면 종료 신호가 데몬까지 전달되지 않아 오펀 프로세스로 남음.

## 조치
- 종료 스크립트를 `pkill` 패턴 매칭 기반으로 재작성
- 매 세션 시작 전 동일 패턴의 기존 프로세스를 전부 죽이고 다시 올리도록 변경
- 종료 시에도 같은 pkill 을 호출해 확실한 cleanup
- 기존 누적된 오펀 프로세스도 한 번 청소

## 예방 (Forcing function 우선)
- 모든 장시간 백그라운드 데몬은 **`pkill -f <고유패턴>` 로 전/후 청소하는 것을 기본 패턴** 으로. 데몬 기동 스크립트 상단에 이 규칙 주석으로 박아둠.
- 세션 시작 훅(`SessionStart`) 에서도 관련 데몬이 1개 이하인지 확인하고, 1개 초과면 알림 + 자동 pkill. (아직 설치 안 됨 — 재발 관찰 후 추가 검토)

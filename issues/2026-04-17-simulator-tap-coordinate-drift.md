# iOS 시뮬레이터에서 메모 탭 좌표가 한 칸씩 어긋남

- **발생 일자:** 2026-04-17
- **해결 일자:** 2026-04-17
- **심각도:** low (스크린샷 자동화 흐름 지연)
- **재발 가능성:** medium (시뮬레이터 해상도/스케일 변경 시 재현 가능)
- **영향 범위:** 메모요 스크린샷·테스트 자동화 (iOS Simulator)

## 증상
AppleScript 로 특정 메모 항목을 탭 하려고 좌표를 계산해서 전달했는데, 목표 항목이 아닌 **바로 위 혹은 아래 한 행**이 열림. 매번 일정한 방향으로 어긋나는 것도 아니고 리스트 위치에 따라 ±1 행씩 밀림.

## 원인
시뮬레이터 창의 **논리 좌표와 실제 디스플레이 좌표 스케일이 어긋남**. macOS Retina 스케일링(2x)과 시뮬레이터 자체의 하드웨어 스케일이 겹치면서 AppleScript 가 전달하는 좌표가 앱이 받는 좌표와 한 행 단위로 밀림. 특히 리스트 아이템 높이가 작을수록 오차 누적이 행 하나 수준에 도달.

## 조치
- AppleScript 좌표 계산에 **스케일 보정 상수** 추가
- 계산된 좌표로 탭 → 예상 항목이 열렸는지 검증 → 실패 시 ±1 행 보정 후 재시도하는 **재시도 로직** 삽입
- 여러 번 테스트 후 리스트 전체 범위에서 정확한 항목이 열리는 상태로 수렴

## 예방 (Forcing function 우선)
- 시뮬레이터 좌표 기반 자동화는 **단일 탭으로 끝내지 말고 탭 후 결과 검증 → 필요시 재시도** 패턴을 기본으로. "한 번에 맞으면 좋고 아니면 재시도" 가 디폴트.
- 가능하면 좌표 대신 **accessibility identifier** 기반 조작을 먼저 검토(AppleScript 로 한계가 있으면 XCUITest 계열로 넘어감). 좌표 탭은 최후 수단.
- 메모요 스크린샷 스크립트에 스케일 상수와 검증-재시도 패턴 주석으로 박아둠.

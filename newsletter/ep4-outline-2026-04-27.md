### 2막 — 표준 길은 모두 막혀있다

탐색한 옵션 4개:
1. **Substack import** — RSS feed 만들어서 import → RSS 가 다시 마크다운 → 같은 문제
2. **WordPress migration** — Substack 에 워드프레스 글로 import → 무료 호스팅 워드프레스 셋업 비용 +1
3. **Markdown-to-Rich JavaScript snippet** — 브라우저 개발자 도구로 변환 → 매번 손이 감, 자동화 아님
4. **Substack API** — public API 없음 (Substack 의 정책)

표준 길 다 막혀있다는 결론. **비표준 길이 필요했다.**

### 3막 — NSPasteboard 클립보드 핵

발견 (어디서 본 누구의 트릭이었는지 회상):
- macOS NSPasteboard 는 같은 클립보드에 여러 representation 을 동시에 담는다
- HTML 을 `public.html` representation 으로, RTF 를 `public.rtf` representation 으로 같이 넣으면
- 받는 앱이 자기가 선호하는 포맷으로 꺼낸다
- **Substack 에디터는 RTF 를 우선한다** (이 발견이 핵심)

```bash
# 마크다운 → HTML (pandoc)
pandoc -f markdown -t html "$md_file" -o /tmp/post.html

# HTML → NSPasteboard 의 RTF representation 으로 강제 주입
osascript -e "set the clipboard to (read POSIX file \"/tmp/post.html\" as «class RTF »)"
```

실행하면: Substack 에디터에 cmd+V → 표·굵은글씨·헤더 다 살아난 채로 붙여진다.

표 처리 함정:
- HTML 표는 RTF 로 변환되면 깨진다 (Substack RTF 파서가 table tag 못 읽음)
- 우회: 표를 마크다운 단계에서 리스트(`- 컬럼: 값`) 로 미리 변환
- 작은 비용 이 큰 마찰을 푼다 — Ep.3 의 "빼기" 와 같은 결.

### 4막 — Playwright MCP 로 로그인·붙여넣기·발행 자동화

- Playwright MCP 가 macOS Claude Code 안에서 브라우저 직접 컨트롤
- Substack 로그인은 magic link → Gmail 에서 토큰 받아서 redirect URL 까지 추출 → 1회 인증 후 60일 세션 쿠키 보존
- 새 글 만들기 → 제목 입력 → 본문 영역 클릭 → cmd+V (위 NSPasteboard 클립보드에서 자동 붙여넣기) → 발행 버튼

전체 파이프라인:
```
markdown 작성
  ↓
pandoc 변환 (md → html → RTF clipboard)
  ↓
Playwright MCP (login → new post → cmd+V → publish)
  ↓
Substack 발행 완료
```

소요: 약 90초 (대부분 Playwright 의 페이지 로딩 시간).

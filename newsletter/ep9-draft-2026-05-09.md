# 3기기가 대화하게 만들기 — 에이전트 메시 구축기

_바이브코딩 뉴스레터 Ep.9 초안_

_2026-05-09_

---

> "맥북 Claude 한테 말했는데, WSL Claude 도 알아야 하지 않나?"
>
> 그 한 마디가 이 삽질의 시작이었다.

나는 현재 세 기기에서 AI를 쓴다. 맥북(Claude), WSL(Claude), 맥미니(Codex). 각자 역할이 다르다. 맥북은 주 대화 창구, WSL은 코드 작업 실행기, 맥미니는 빌드와 외부 작업 엔진. 그런데 이 세 명이 서로 말을 못 했다. 내가 중간에서 복붙을 해야 했다.

그걸 자동화하기로 했다.

---

## 아이디어는 간단했다

"맥북에서 작업 지시를 내리면 WSL이 실행하고, 결과를 다시 맥북에 보고한다."

단방향이면 `wsl-directive.sh`, 반대 방향이면 `mac-report.sh`. 이 두 스크립트만 있으면 됐다.

wsl-directive.sh 는 맥북에서 SSH 로 WSL 에 접속해 tmux 세션에 명령을 붙여넣는다. mac-report.sh 는 WSL 이 작업을 끝낸 뒤 맥북 tmux 'claude' 세션에 보고서를 붙여넣어 Claude 를 깨운다.

여기에 맥미니까지 붙이면서 `agent-msg-notify.sh` 가 추가됐다. 텔레그램 봇 3개가 각자 다른 채팅방에 연결돼 있고, 스크립트가 "누구에서 누구로 어떤 내용" 형식의 메시지를 알맞은 채팅방에 쏜다.

---

## 첫 번째 삽질: 봇은 봇 메시지를 못 받는다

텔레그램 봇을 중간 라우터로 쓰면 편할 것 같았다. 맥북 봇이 WSL 봇에게 메시지를 보내면 WSL Claude 가 읽고 실행하는 구조.

완전히 틀렸다. 텔레그램은 봇이 다른 봇의 메시지를 수신할 수 없다. privacy 모드를 꺼도 마찬가지다. 봇↔봇 채널은 플랫폼 설계상 막혀 있다.

그래서 실제 채널은 텔레그램이 아니라 **SSH + tmux** 가 됐다. 텔레그램은 사람이 보는 알림 채널로만 쓰고, 에이전트 간 실제 명령 전달은 SSH paste 방식을 쓴다.

---

## 두 번째 삽질: relay 재발 사고

초기에 `codex-session-relay` 라는 데몬을 만들었다. 맥미니 Codex 채팅방에 오는 메시지를 맥북 채팅창으로 미러링하는 용도였다.

의도는 좋았는데 부작용이 심했다. Codex 에게 온 메시지가 맥북 채팅에 그대로 들어오니까, Claude 가 그걸 자신에게 온 메시지로 읽고 반응하기 시작했다. 맥미니 작업 내역이 맥북 세션에 섞여들어서 컨텍스트가 엉켰다.

이후 relay 데몬은 의도적으로 비활성화했다. 재기동 금지가 지금도 규칙으로 남아있다. 미러링은 편리해 보이지만 에이전트가 두 개면 각자의 컨텍스트는 분리돼야 한다.

---

## 세 번째 삽질: inject vs. notify

맥미니 Codex 에 명령을 보내는 방법을 두고 한참 헤맸다.

처음엔 `openclaw system event --text "..."` 명령어를 썼다. 그런데 이게 시스템 컨텍스트 주입 방식이라 Codex 채팅창에는 아무것도 안 보였다. Codex 는 내가 보낸 명령을 받은 적도 없는데 나는 보냈다고 착각한 것.

실제로 Codex 가 보려면 **채팅방에 메시지가 와야 한다.** `agent-msg-notify.sh` 에 올바른 채팅방 ID 를 연결해서 거기로 텔레그램 메시지를 보내는 방식이 정답이었다. 단 이 경우 결과 알림용이지, 명령 주입용은 아니다. 명령 주입은 별도 `codex-directive.sh` 경로를 써야 한다.

---

## 지금 구조

```
맥북 Claude
  ↓ wsl-directive.sh (SSH + tmux paste)
WSL Claude
  ↓ mac-report.sh (SSH + tmux paste)
맥북 Claude

맥북 Claude
  ↓ agent-msg-notify.sh → 텔레그램 @ssamssae_claw_bot
맥미니 Codex
```

완전 자율은 아니다. 여전히 명령 시작은 내가 맥북에서 한다. 하지만 "지시 → 실행 → 보고" 루프는 내 손을 안 거친다. WSL 이 4시간짜리 작업을 하는 동안 나는 맥북에서 다른 일을 한다. 끝나면 맥북 tmux 에 보고서가 paste 되고 Claude 가 검토한다.

3기기가 완전히 대화하지는 않는다. 하지만 내가 중간에서 복붙하는 건 없어졌다.

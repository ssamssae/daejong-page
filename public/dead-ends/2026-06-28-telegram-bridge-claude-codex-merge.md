# 텔레그램 브릿지 claude·codex 겸용 통합 — 분리로 선회

- **드롭 시점:** 2026-06-28
- **분류:** 판단으로 접음
- **목표:** claude·codex 양쪽을 미러링하는 텔레그램 브릿지를 단일 겸용 코드베이스로 통합

## 뭘 하려 했나

- 두 브릿지(claude-telegram-bridge / codex-telegram-bridge)를 하나로 합쳐 공통 레이어로 운용 시도
- 겸용화하면 토큰/설정/미러 로직을 한 곳에서 관리 가능하다는 기대

## 왜 드롭했나

- 겸용 코드베이스는 한쪽 기능을 고칠 때 다른 쪽까지 영향이 번질 위험이 큼
- 1인 유지보수 환경에서 "잘못 건드리면 둘 다 꼬인다"는 회귀 리스크가 통합 이득보다 크다고 판단

## 결론

겸용 통합은 막다른 길. 코덱스 전용 / 클로드 전용 2개 프로그램으로 분리 개발 — 각자 단순하게 유지.

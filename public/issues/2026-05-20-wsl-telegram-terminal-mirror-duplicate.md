# WSL 텔레그램 답변 중복 송신 (터미널 미러 + reply)

- 날짜: 2026-05-20
- 노드: 🪟 WSL (<desktop-host>, @Myclaude2_ssamssae_bot)
- 증상: 같은 답변 문장이 강대종 텔레그램에 2번 떠서 "왜 두번말하냐" 지적받음.

## 무슨 일이 있었나

텔레그램 인바운드("ㅎㅇ")에 답하면서:
1. 답변 본문("네 형님, 대기 중입니다. 부르실 일 있으세요?")을 **터미널 텍스트로 출력**.
2. reply 툴 호출은 안 함 → Stop hook `telegram-reply-check.sh` 가 block.
3. 훅 지시대로 **같은 문장을 reply 툴로 재전송**.

강대종 텔레그램 화면에는 결과적으로:
- `🪟 네 형님...` (1번째)
- `💬 터미널 응답` (터미널 출력 미러)
- `🪟 네 형님...` (2번째, reply)

즉 3덩어리가 떠서 같은 답이 두 번 보임.

## 근본 원인

이 WSL 노드에는 **터미널(tmux) 출력을 텔레그램으로 미러링하는 forwarder** 가 붙어 있다 (`💬 터미널 응답` prefix). 따라서 이 노드에서는 "터미널 출력은 사용자에게 안 보인다"는 일반 가정이 깨진다 — 터미널에 쓴 본문이 그대로 사용자에게 보인다. 본문을 터미널과 reply 양쪽에 쓰면 **반드시 중복**.

기존 메모리 `feedback_telegram_reply_tool_mandatory` 는 "reply 를 빼먹는" 누락 패턴만 다뤘는데, 이번은 그 반대 부작용(중복)이다. 둘 다 원인은 같다: **본문을 터미널에 쓰는 습관.**

## 재발 방지

- 텔레그램 인바운드에 답할 때 **본문은 reply 툴에만** 담는다.
- 터미널 출력은 비우거나, reply 와 **다른** 짧은 상태 한 줄(예: "(reply 전송 완료)")만.
- 같은 본문을 터미널에 적었다가 reply 로 또 보내는 동작 금지.
- 메모리 `feedback_telegram_reply_tool_mandatory` 에 2026-05-20 anti-pattern 추가 완료.

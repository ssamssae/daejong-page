# 텔레그램 복붙 메시지는 순수 블록만 — 라벨·안내 텍스트 혼입 금지

- **첫 발견:** 2026-04-26 (PowerShell 명령 라벨 혼입 → syntax error)
- **재사용 영역:** 텔레그램 reply 워크플로우 전체 — shell 명령, config snippet, flutter/adb/git 명령 등 복붙 전용 블록 전부.

## 한 줄 요약

폰에서 텔레그램 메시지를 long-press copy 하면 **메시지 전체가 클립보드에 들어간다.** 명령어 블록에 `[관리자 PowerShell 에 그대로 복붙]` 같은 안내 텍스트를 같이 보내면 그 텍스트도 실행줄에 붙어 syntax error 가 난다. **복붙 전용 메시지에는 실행 가능한 코드 라인만** 들어가야 한다.

## 원칙

1. **0줄 컨텍스트** — 복붙 블록 메시지에는 설명·라벨·괄호 주석 0줄. "실행하세요" 한 마디도 없이 코드만.
2. **메시지 분리** — 설명이 필요하면 **앞 메시지**(설명) + **뒤 메시지**(순수 블록) 로 2통 분리 전송.
3. **fenced block 도 동일 기준** — ` ```bash ... ``` ` 안에도 주석(#) 라인은 최소화. 실행 흐름에 없는 줄은 넣지 않는다.

## 패턴 (올바른 분리 예)

**메시지 1 — 설명**
```
관리자 PowerShell 창에 붙여넣으세요.
```

**메시지 2 — 순수 블록**
```
Set-ExecutionPolicy Bypass -Scope Process; iex (iwr ...).Content
```

메시지 1은 읽고 버려도 되고, 메시지 2는 long-press copy → 붙여넣기 → 엔터 3단계로 끝난다.

## 식별 방법

보내기 직전 자가 체크:

- 이 메시지를 long-press copy 해서 터미널에 그대로 붙여넣으면 에러 없이 실행되는가?
- 한글·영문 안내 텍스트가 한 줄이라도 있는가? → 있으면 별도 메시지로 분리

## 함정

- `# 주석`도 shell 에서는 주석으로 처리되어 실행엔 문제 없으나, PowerShell 에서는 단어 순서에 따라 다를 수 있음 → 가능하면 주석도 제거.
- 짧은 한 줄 안내 ("아래 코드 실행 후 재시작") 는 얼핏 OK 처럼 보이지만 긴 명령 블록 사이에 끼면 오류 유발 → 예외 없이 분리.

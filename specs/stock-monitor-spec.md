# 주식 모니터링 앱 기획서

> 작성: 2026-05-03 (WSL, parking-lot directive)
> 상태: 초안 (코드 작성 전 기획 단계)

---

## 1. 앱 개요

**앱명 (가칭):** 주식알람 / StockPing

**한 줄 설명:** 국내 주식 관심종목의 가격 변동을 실시간으로 알려주는 심플 모니터링 앱.

**핵심 가치:** 증권사 앱은 무겁고 알림이 과하다. 이 앱은 "내가 설정한 종목이 목표가에 도달했을 때만" 알려준다. 그 이상 아무것도 안 한다.

---

## 2. 타깃 사용자

- 국내 주식(코스피/코스닥)에 소액 투자하는 개인 투자자
- 증권사 HTS/MTS 알림이 너무 많아서 피로감을 느끼는 사람
- 가격 알람 하나만 필요하고, 차트 분석이나 자동매매는 원하지 않는 사람
- 1인 개발자 본인 (dogfood — 실제로 로또는 아니고 ETF 소액 보유)

**타깃 제외:**
- 전문 투자자 (Bloomberg, 키움 HTS 수준의 데이터가 필요한 사람)
- 자동매매/알고리즘 트레이딩 사용자
- 해외주식 전용 사용자 (MVP 범위 밖)

## 3. 기술 스택 후보

### 프론트엔드
- **Flutter** — 기존 앱들과 통일, iOS/Android 동시 출시, 코드 재사용
- 상태관리: Riverpod (기존 앱들과 동일)
- 로컬 알림: `flutter_local_notifications`
- 백그라운드 fetch: `workmanager` (Android) / BGAppRefreshTask (iOS)

### 데이터 소스
- **KRX 공공데이터 포털 (data.krx.co.kr)**
  - 종목별 시세 API: 코스피/코스닥 전 종목 현재가, 전일대비, 등락률
  - 인증: API 키 발급 (무료, 회원가입 필요)
  - 제한: 실시간 아님 (15~20분 지연), 장중만 업데이트
  - 장점: 공식 데이터, ToS 명확, 스토어 리젝 위험 없음

- **한국투자증권 Open API** (대안)
  - 실시간 시세 가능
  - 계좌 연동 필수 → MVP 진입 장벽 높음 → 2차 고려

### 백엔드 (최소화)
- MVP: 백엔드 없음. 앱이 직접 KRX API 호출 + 로컬 알림
- 2단계: 서버리스 함수(Supabase Edge Function 또는 Cloudflare Workers)로 가격 폴링 → 푸시 알림 (앱이 꺼져도 알림 가능)

## 4. 주요 기능

### MVP (1.0)

| 기능 | 설명 |
|---|---|
| 관심종목 추가 | 종목명/코드 검색 → 위시리스트 추가 (최대 10종목) |
| 현재가 조회 | KRX API 폴링으로 현재가 / 전일대비 / 등락률 표시 |
| 가격 알람 설정 | 종목별 목표가 설정 → 도달 시 로컬 푸시 알림 |
| 일봉 차트 (간단) | 최근 20거래일 종가 라인 차트 (KRX 일별시세 API 사용) |
| 장 시간 표시 | 장중(09:00~15:30) / 장외 상태 표시, 장외엔 "지연 데이터" 표시 |

### 2단계 고려 (출시 후)

- 서버 푸시 (앱 종료 상태에서도 알림)
- 52주 고/저가 알람
- 여러 알람 조건 AND/OR
- 간단 메모 기능 (내가 왜 이 종목을 담았나)

## 5. 피해야 할 것

### 5-1. 스크래핑 (ToS 위반 + 스토어 리젝 위험)

아래 소스는 **절대 사용하지 않는다:**

| 소스 | 이유 |
|---|---|
| 네이버 금융 (`finance.naver.com`) | 공개 API 없음, HTML 스크래핑은 ToS 위반. Apple/Google 모두 스크래핑 앱 리젝 사례 있음 |
| 다음 금융 | 동일 이유 |
| Yahoo Finance | `query1.finance.yahoo.com` 비공식 API — 한국 주식 데이터 불안정, 서비스 종료 이력 있음 |
| 증권사 앱 내부 API | 인증 리버스엔지니어링 — ToS 위반, 앱 리젝, 법적 위험 |

**원칙:** 공식 API 키가 있고 ToS 에 "앱 사용 허가" 가 명시된 소스만 사용.

### 5-2. 자동매매 / 투자 추천 (법적 위험)

- **자동매매**: 주문 API 연동 시 투자일임업 영역 → 금융위원회 등록 필요
- **투자 추천/분석**: "이 종목을 사야 한다" 형태의 기능 → 투자자문업 영역 → 등록 필요

MVP 는 "가격이 얼마다" 만 알려준다. "사야 한다/팔아야 한다" 는 하지 않는다.

### 5-3. 광고 기반 모델 피하기

- 소규모 금융 앱에 광고 SDK 붙이면 심사 거절 사례 많음 (Google AdMob + 금융 앱 조합)
- 수익 모델은 유료 앱 또는 구독 (나중에)

## 6. 추천/자동매매 제외 이유 (상세)

### 법적 근거

- **투자자문업**: 금융투자업규정상, 이익을 얻을 목적으로 불특정 다수에게 투자 판단(매수/매도/보유)을 제공하면 투자자문업에 해당. 금융위원회 인가 필요.
- **투자일임업**: 고객 자산을 자동으로 운용(매수/매도 실행)하면 투자일임업. 마찬가지로 인가 필요.

1인 인디 앱이 이 인가를 받는 건 현실적으로 불가능하다. 따라서 앱은 **정보 제공에만 국한**.

### 실용적 이유

- 자동매매가 잘못 동작할 경우 사용자 피해 → 책임 문제
- 증권사 Open API 는 계좌 연동을 요구하고, 실제 주문 전송은 보안 인증 과정이 복잡
- MVP 에 필요하지 않음 — "알람" 만으로 충분히 유용한 앱 가능

## 7. MVP 범위 정의

### 포함

- 종목 검색 (KRX 공공데이터)
- 관심종목 10개 로컬 저장
- 현재가 / 전일대비 / 등락률 조회 (폴링 간격: 장중 5분)
- 목표가 알람 1종목 1알람 (로컬 푸시)
- 일봉 차트 20일 (라인 차트)
- 장중/장외 상태 표시

### 제외 (MVP 이후)

- 서버 푸시 (앱 꺼진 상태 알람)
- 해외 주식
- 종목 비교
- 포트폴리오 관리 (손익 계산)
- 증권사 계좌 연동
- 자동매매 / 투자 추천

| 리스크 | 대응 |
|---|---|

| KRX API 구조 변경 | 공식 채널 모니터링, 앱 버전 업데이트 대응 |

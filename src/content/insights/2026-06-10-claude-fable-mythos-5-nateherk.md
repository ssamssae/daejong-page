## 픽업
- Fable 5 와 Mythos 5 는 후드 아래 동일 모델이고, 차이는 Mythos 5 만 사이버 세이프가드를 해제한 버전이라는 점. Fable 5 는 오늘 전체 공개, Mythos 5 는 Glasswing 파트너(미 정부 협업)에만 제한 배포된다고 발화.
- 가격은 둘 다 입력 100만 토큰당 $10 / 출력 $50 로 Opus 의 2배. 발화자는 "기대만큼 유능하다면 큰 그림에서 그렇게 비싼 건 아니다"라고 평가.
- Fable 5 는 6월 22일까지 Pro Max·Team·Seat 기반 Enterprise 플랜에 추가비용 없이 포함되지만, 6월 23일부터 구독에서 빠지고 토큰 기반 usage credit 으로 전환. Anthropic 은 용량이 확보되면 다시 구독에 복원할 계획이라 밝혔다고 전달.
- Mythos class 는 Opus 바로 위 capability 티어. 4월 Project Glasswing 으로 나온 첫 Mythos preview 는 Opus 의 5배 비용이었는데, 이번 Fable/Mythos 5 는 2배 비용이라 "풀 모델 capability 를 다 주는 건 아니다"라고 해석.
- 거의 모든 벤치마크에서 SOTA — 소프트웨어 엔지니어링·knowledge work·비전·과학 연구. Opus 4.8, GPT 5.5 대비 knowledge work·agentic coding·사이버 보안에서 큰 점프가 보인다고 언급(단 벤치는 늘 걸러 보라는 단서).
- 에이전트 루프 유행에 경계: "에이전트를 프롬프트하는 루프를 돌려라"는 모델 제공사 권유는 결국 토큰을 더 쓰게 하려는 동기가 있고, 일반 knowledge work 대부분은 상시 도는 루프가 필요 없으며 열린 프롬프트를 쏴두면 세션 한도만 소진한다고 지적.
- effort 레벨 감각 — 지금 Opus 4.8 의 X-high 가 대략 Fable 5 의 low 수준이고 Fable low 가 더 싸다. max effort 는 X-high 위로 한 단계 올렸지만, Opus 4.8 에서는 X-high 와 max 차이를 체감 못 했다고 발화. 비전 개선은 verification loop 에 특히 중요하다고 강조.

## 용어
- **Fable 5** [모델 · 구독]: 이 영상에서 발표된, Opus 위 "Mythos class" 티어의 일반 공개 모델. 복잡한 장기 실행 작업용으로 포지셔닝.
- **Mythos 5 / Mythos class** [모델 · 구독]: Fable 5 와 동일 모델이되 사이버 세이프가드를 해제한 제한 배포 버전, 그리고 Opus 위에 위치한 모델 티어 명칭.
- **Project Glasswing** [기타]: Mythos preview·Mythos 5 를 미 정부·사이버 방어 그룹 등 신뢰 그룹에만 제한 배포하는 프로그램.
- **usage credits** [빌링 · 운영]: 구독 포함이 끝난 뒤 모델 사용분을 토큰 기반으로 별도 과금하는 크레딧 방식.
- **effort levels (low / X-high / max)** [하니스 · 패턴]: 같은 모델에서 추론 깊이·비용을 조절하는 등급. 등급 간 체감 차이와 비용이 비선형이라는 점이 논의됨.
- **agentic loops** [하니스 · 패턴]: 에이전트가 다른 에이전트를 반복 프롬프트하며 장시간 자율 동작하는 패턴. 토큰 과소비·세션 한도 소진 리스크가 지적됨.
- **verification loops** [하니스 · 패턴]: 모델이 비전 등으로 산출물을 스스로 검증·반복 점검하는 루프. 비전 성능 향상이 여기에 직접 기여.

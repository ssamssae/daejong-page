## 픽업
- Fable 5 는 Mythos 클래스(Opus 위 티어)가 일반 유료 플랜에 풀린 첫 모델 — 이전까지 이 티어는 검증된 보안 파트너 전용이었고, 테스트한 거의 모든 벤치마크에서 SOTA.
- 코딩·비전 수치: Stripe 는 "수개월치 엔지니어링을 며칠로 압축"했고, 5천만 줄 Ruby 코드베이스 풀 마이그레이션을 하루 만에(사람 팀 기준 2개월+). 스크린샷만으로 웹앱 소스 재구성이 가능하고, 포켓몬 FireRed 를 raw 스크린샷만으로 처음부터 끝까지 클리어(구 모델은 헬퍼 하니스 필요).
- 가장 중요한 능력은 long-horizon 집중 — 수백만 토큰에 걸쳐 락인 유지. 파일 기반 메모리(노트 쓸 파일만 제공)로 Slay the Spire 를 돌리면 최종 막 도달률이 Opus 4.8 의 3배.
- 영상 제작 파이프라인 전부 자율: 발표문 리서치+팩트체크 → 본인 transcript 로 만든 voice playbook 으로 스크립트 작성 → ElevenLabs 보이스클론은 1분 미만 청크로 쪼개 생성(길게 뽑으면 보이스 드리프트) → HeyGen Avatar 5 렌더 → ffmpeg 스티칭 + 모션그래픽을 HTML+GSAP 코드로 작성 → 장면별 프레임을 렌더해 시각적으로 자가 리뷰하고 어긋난 건 고쳐 재렌더.
- 비용 실측: /goal 한 방, 1시간, 약 38~40만 토큰 — $200 Max 플랜의 약 40% 를 1시간에 소진. "진짜 조심해야 한다"고 직접 경고. 단 워크플로우의 서브에이전트들은 Fable 이 아니었음.
- 프롬프트 팁: 목표 끝에 "이건 내 유튜브 채널에 나간다, 평판 손상 리스크가 있다, 100% 확신할 때만 멈춰라"처럼 *왜 하는지* 컨텍스트를 주면 이해도가 올라간다. 빌드 후엔 dynamic workflow 로 모션그래픽 타이밍·화면 이탈·심미성까지 시각 검증하라고 명시.
- 같은 프롬프트를 복붙해도 같은 결과는 안 나온다(본인의 Hyperframes 스킬들이 깔려 있어서) — 그리고 한 번 구축한 지금은 스킬로 만들면 Sonnet 으로도 이 스타일을 재현할 수 있을 것 같다고.

## 용어
- **Mythos 클래스** [모델 · 구독]: Opus 위에 새로 생긴 Anthropic 최상위 모델 티어. Fable 5 가 일반 유료 플랜에 공개된 첫 Mythos급.
- **long-horizon (장기 호라이즌)** [모델 · 구독]: 수백만 토큰·수 시간짜리 작업에서 목표를 잃지 않고 집중을 유지하는 능력 축.
- **파일 기반 메모리** [컨텍스트 · 캐시]: 모델에게 노트를 적을 파일만 쥐여 주고 장기 작업 상태를 스스로 기록·회수하게 하는 패턴.
- **/goal** [하니스 · 패턴]: Claude Code 에서 목표 하나를 주면 완료 판정까지 자율로 굴러가는 커맨드. 객관적으로 판정 가능한 목표에 적합.
- **dynamic workflow** [하니스 · 패턴]: 서브에이전트들을 fan-out 해 스크린샷 캡처·검증 같은 단계를 오케스트레이션하는 실행 방식.
- **Hyperframes** [하니스 · 패턴]: 모션그래픽을 HTML 코드로 만들어 렌더하는 발화자의 자체 스킬 세트 — 결과 재현의 숨은 전제.
- **voice playbook** [지식 · 컨텍스트 자산]: 본인 영상 transcript 에서 추출한 말투·표현 가이드. AI 가 "내 목소리로" 글을 쓰게 하는 자산.
- **보이스 드리프트** [기타]: TTS 생성이 길어질수록 클론 목소리가 원본에서 점점 벗어나는 현상. 1분 미만 청크 분할이 우회책.
- **HeyGen Avatar 5** [도구 통신 (MCP · CLI · API)]: HeyGen 의 최신 아바타 모션 엔진. 한동안 API 미노출이라 Playwright 로 브라우저를 직접 몰았으나 신 API 로 직통.
- **GSAP** [기타]: 웹 애니메이션 JS 라이브러리. 모션그래픽을 영상 클립이 아닌 코드로 만들어 단어 타이밍에 맞춰 재생.

# I Made Opus 4.8 and Fable 5 Build the Same App (RAW RESULTS) — Pat Simmons

## 한 줄 요약
동일한 프롬프트로 이커머스 스토어, 3D 아트 뮤지엄, 에이지 오브 엠파이어 클론 세 앱을 Opus 4.8과 Fable 5에게 원샷으로 빌드시킨 결과, 속도·품질·토큰 효율 모든 면에서 Fable 5가 압도적으로 앞섰다. Fable 5는 병렬 에이전트 자체 조율, 실시간 QA, 이미지 프롬프팅 최적화까지 수행했고 Opus 4.8은 3D 게임 빌드에서 결과물 자체가 동작 불가 수준이었다.

## 픽업
- Fable 5는 세 빌드 모두에서 Opus 4.8보다 약 15~20분 빠르게 완료했다. 이커머스 빌드 기준 Fable 5 약 35분, Opus 4.8 약 50분.
- 비용 측면에서 Fable 5는 Opus 4.8보다 약 37% 비싸지만 출력 토큰 수가 훨씬 적어 실효 비용은 낮다. 이커머스 빌드 기준 Opus 4.8 약 $21, Fable 5 약 $37이었으나 3D 뮤지엄 빌드에선 Opus $46, Fable 5 $64 추정.
- Fable 5는 이미지 생성 모델(GPT Image 2)에 넘길 프롬프트를 스스로 최적화했다. 사용자가 이미지 프롬프팅 방법을 별도로 지시하지 않았는데도 캔들 분위기에 맞는 조명·질감·그림자를 살린 이미지를 생성했다.
- Fable 5는 3D 뮤지엄 빌드 중 Wikipedia API에서 `Francis Bacon (artist)` 처럼 괄호 표기가 붙은 카테고리명이 파싱에서 실패하는 엣지 케이스를 스스로 발견하고 수정했다. 수백 명의 아티스트를 순회하면서 발생한 엣지케이스를 능동 탐지한 것.
- Fable 5는 에이전트를 자체 스폰·조율하며 "이 에이전트가 끝나면 재호출하겠다"고 명시했다. 병렬 에이전트 오케스트레이션을 외부 하니스 없이 모델 단독으로 수행한 점을 스피커가 주목했다.
- Max 20× 구독 플랜에서 3D 뮤지엄 두 모델을 30분간 돌렸을 때 전체 usage의 40%가 소진됐다고 밝혔다. Fable 5의 usage-based pricing은 영상 기준 2주 후 전환 예정이라고 언급했다.
- Opus 4.8은 에이지 오브 엠파이어 빌드에서 캐릭터가 blob 수준이었고 맵 이동 자체가 불가능한 상태로 완전히 동작하지 않았다. 반면 Fable 5는 실제 게임에 근접한 3D 그래픽과 유닛 AI, 지도 탐색이 모두 작동하는 결과물을 냈다.

## 용어
- **원샷 빌드 (one-shot build)** [하니스 · 패턴]: 수정 없이 프롬프트 한 번으로 완성 결과물을 뽑는 평가 방식. 모델 품질 비교 시 기준선으로 사용됨.
- **Dynamic Workflows** [하니스 · 패턴]: Anthropic이 제공하는 기능으로 하나의 큰 작업을 Claude가 30개 이상의 병렬 에이전트 스웜으로 자동 분할해 처리하는 방식. 스피커는 이전 영상에서 결과에 실망했다고 밝힘.
- **병렬 에이전트 자체 스폰 (self-spawning parallel agents)** [하니스 · 패턴]: 모델이 외부 오케스트레이터 없이 스스로 서브 에이전트를 생성하고 완료 시 재호출을 예약하는 동작. Fable 5가 3D 뮤지엄 빌드에서 보여준 행동.
- **usage-based pricing** [빌링 · 운영]: 정액 구독이 아닌 실제 토큰 소비량 기준으로 과금하는 방식. Anthropic이 Max 구독 플랜에 도입 예정이라고 스피커가 언급.
- **Max 20× 구독 플랜** [빌링 · 운영]: Anthropic의 고용량 구독 티어. 스피커 기준 30분 양 모델 대형 빌드에서 usage 40%가 소진될 만큼 토큰 소비가 큰 작업에 적합.
- **GPT Image 2** [도구 통신 (MCP · CLI · API)]: OpenAI의 이미지 생성 모델. Fable 5가 이커머스 빌드에서 캔들 이미지 30장을 생성할 때 내부적으로 사용했으며, 스피커는 결과 품질에 별도로 감탄함.
- **Neon DB** [하니스 · 패턴]: 브라우저 기반 서버리스 Postgres. 3D 뮤지엄 빌드에서 Wikipedia API에서 추출한 수백 명 아티스트·767점 그림 데이터를 저장하는 백엔드로 사용됨.
- **GSAP** [기타]: GreenSock Animation Platform. 웹 애니메이션 라이브러리. Fable 5가 아티스트 클릭 시 애니메이션 전환에 사용했으며 스피커가 프롬프트에 요청했다고 언급.
- **token efficiency (토큰 효율)** [빌링 · 운영]: 동일 작업 대비 소비 출력 토큰 수. 스피커는 Fable 5가 Opus 4.8보다 출력 토큰이 적으면서도 품질이 높아 실효 비용이 낮다고 분석함.

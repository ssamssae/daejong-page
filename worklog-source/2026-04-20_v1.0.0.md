## 기억이 챗봇 — 텔레그램 BadRequest 크래시 + 답변 품질 개선

### 크래시 원인 규명

- 증상: "답변이 안 온다" 텔레그램 메시지로 디버깅 시작
- WSL systemd 로그: `gieogi-bot.service` 는 살아있지만 텔레그램 BadRequest 로 반복 크래시
- 근본 원인: `src/기억이/src/bot.py:102` 의 `parse_mode=Markdown` 설정. LLM 답변에 짝 안 맞는 언더바·별표가 섞여 텔레그램이 거부

### 조치

- Markdown 전송 실패 시 **plain text 로 재시도하는 fallback 분기** 추가
- systemd 재시작 요청 → 사용자에게 "재시작 = 컴퓨터 재부팅 아님, 봇 프로세스만 다시 올리는 것" 안내

- 설치 모델: `qwen2.5` + `nomic-embed` 둘뿐 → 한국어 강화는 프롬프트 레벨이 최선

- 파라미터 튜닝:
  - `num_predict`: 1500 (답변 길이 확보)
  - `num_ctx`: 8192 (컨텍스트 확대)
  - `TOP_K`: 12 (retrieval 편향 완화)
- Footer 출력에서 worklog 파일명 `.md` 제거 — 텔레그램이 `.md` 를 몰도바 국가 도메인으로 인식해 자동 링크 걸어버리는 충돌 회피

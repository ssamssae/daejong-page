## 멀티기기 Q&A (대화 요약)

오늘 세션에서 대종님이 던진 질문 4개와 답:

1. **"오전 6:45 데일리싱크 깃 풀 이해했는데 푸시는 각 기기에서 따로 하잖아. 꼬이지 않을까?"**
   - 답: 맥·WSL 이 각자 다른 파일(또는 같은 파일 다른 섹션) 을 건드릴 경우 rebase 성공. 동일 라인 수정 시만 conflict. todos.md 같은 단일 라인 경합 파일은 이미 "두 기기 동시 편집 금지" 가 규칙 (WSL 은 텔레그램 트리거로 맥 위임).
2. **"스킬 파일이 맥/윈 양쪽에 동일한데 읽어들일 때 두 배 로드 아닌가?"**
   - 답: 각 기기의 Claude Code 프로세스는 **자기 호스트의 `~/.claude/skills/` 만** 읽음. 상대 기기 파일은 안 읽힘. 동일 내용이어도 토큰 이중 카운트 없음 — 로컬 파일 1세트 × 1 Claude 세션 = 1회 로드.
3. **"토큰 절약하게 된 계기는 내가 요금제 결제한 지 며칠 됐지?"**
   - 답: 청구주기/사용량은 ccusage 로만 측정 가능. `/usage` 스킬 호출하면 주간 사용량 % 를 ASCII 그래프로 보여줌. 지금은 구체 수치 확인 안 함.
4. **"더이상 최적화할거 없다이거지?"**
   - 답: 오늘 건드린 것 외에 눈에 띄는 과투입 후보 없음. MEMORY.md / launchd / skill description 은 이미 감사 완료. 다음 최적화 여지는 훅·프롬프트 엔지니어링 수준이라 굵직한 절감 효과 기대 어려움.

# 2026.04.11 작업일지 v1.1.0 — 하루 안에 메모 앱 하나가 서다

> 아침, 빈 Flutter 프로젝트 `simple_memo_app` 을 팠다.
> 오후, 스와이프 제스처 방향을 아이폰 메모앱 반대로 깔았다가 금방 뒤집었다.
> 밤, 처음 실행은 되는데 **두 번째 실행에서 무조건 크래시** 라는 제보가 들어왔다.

메모요의 첫 날이다. 아침부터 코드 한 줄·APK 빌드·아이콘 재작업·iOS 재실행 크래시 디버깅까지 **프로젝트 라이프사이클의 주요 단계가 하루 안에 압축**됐다. 중간에 개인 홈페이지 daejong-page 초판까지 올렸다. 이 버전은 그 하루를 다시 추적한 소급판이다.

## 1. 한 줄만 받는 메모장

Flutter 로 뼈대를 잡을 때 결정한 건 단 하나였다. **제목 없이 본문만.** 입력란은 하나, 목록에는 본문 첫 줄만 노출. 로컬 저장은 `SharedPreferences`. UI 는 처음부터 한글로 썼고 다크 테마를 기본값으로 잡았다. `+` 버튼과 폰트 색상 톤을 다듬는 데는 오후 전에 끝냈다.

> **배운 것** — "제목 + 본문" 같은 관습적 필드는 앱의 본질을 흐린다. 메모는 적는 게 목적이지 분류하는 게 목적이 아니다. 필드 하나를 과감히 지우면 UI 전체 복잡도가 같이 내려간다.

## 2. 스와이프 방향을 뒤집은 이유

첫 구현은 **오른쪽 스와이프 = 삭제(빨강), 왼쪽 스와이프 = 즐겨찾기(노랑 별)**. 만들고 나서 아이폰 기본 메모 앱을 열어 봤다. **정반대**였다.

즉시 뒤집었다. 왼쪽 = 삭제, 오른쪽 = 즐겨찾기. 삭제 버튼 색도 빨강 대신 더 톤 맞는 것으로 교체. 이어지는 동작 규칙은 "스와이프하면 버튼이 그대로 노출된 채 고정. 버튼을 직접 누르거나 바깥을 완전히 탭하면 닫힘". 여기서 버그 세 개가 연달아 나왔다.

- 살짝만 손가락이 움직여도 버튼이 닫히는 버그
- 다른 곳을 탭해도 버튼이 안 닫히는 버그
- 버튼이 열려 있는데 그 위의 클릭이 안 먹는 버그

세 개를 연달아 잡고 나서야 제스처 레이어가 손에 익은 방식으로 동작했다.

## 3. ReorderableListView 를 두 개로 찢다

드래그로 메모 순서를 바꾸는 기능이 오래 붙잡혔다. 즐겨찾기와 일반 메모를 **한 화면에 같이** 보여주는 건 쉬웠다. 문제는 드래그 도중 **한 그룹의 메모가 다른 그룹 영역으로 넘나들며 섞이는 현상**. 같은 그룹 안에서만 인덱스를 계산하도록 여러 번 고쳐 봤지만 드래그 프리뷰에서 경계가 무너졌다.

결국 **기존 구현을 버리고** `ReorderableListView` 를 두 개로 분리했다. 즐겨찾기용 하나, 일반용 하나. 각자가 독립된 리스트가 되자 드래그 중 서로 영향을 주지 않고 각 리스트가 자기 안에서만 움직였다. 그룹 사이 구분선도 지웠다.

> **배운 것** — "하나의 컴포넌트로 둘의 행동을 조율하는 것" 보다 **"둘로 분리해서 각자가 자기 역할만 하도록 두는 것"** 이 경계 문제를 제거한다. 같은 패턴이 나중에 worklog/done/todos 세 종류의 페이지를 분리할 때도 반복된다.

## 4. 아이콘이 너무 작게 보였다

다운로드 폴더의 메모요 SVG 아이콘을 기반으로 APK 앱 아이콘을 만들었다. 처음 빌드하고 홈화면에 깔아 보니 **너무 작게 보였다**. 여러 번 다시 뽑는 과정이 반복됐다.

- 검은 배경을 없앤다
- 고양이와 노트만 더 크게
- 노트 줄은 검게 더 굵게
- 배경은 투명

동시에 `android-commandlinetools` 를 설치해 툴체인을 정비하고, 앱 이름은 영어 기본값에서 한글 **메모요** 로 바꿨다. 빌드 마지막에는 용량 최적화도 한 번 돌려 결과물을 가볍게 만들었다.

## 5. 응원하기 버튼, 그리고 홈페이지 초판

메모요 우측 상단에 **카카오페이 응원 버튼**. 탭하면 `qr.kakaopay.com` 의 후원 페이지가 외부 브라우저로 열린다. 다크 테마와 어울리도록 작고 세련된 톤으로 배치. 이름은 처음 "후원하기" 였다가 오후 늦게 "응원하기" 로 교체. 하트 아이콘 위치도 8 픽셀 왼쪽으로 미세 조정.

응원 버튼이 가리키는 **랜딩 페이지가 필요했다**. 그래서 저녁에 `daejong-page/index.html` 을 신규로 만들고 다크 모드 개발자 소개 페이지 한 장을 깔았다. 이름 강대종, 소개 문구, 카카오페이 응원하기 버튼. 이걸 GitHub Pages 로 띄워 앱 → 홈페이지 링크를 연결했다.

**이날 두 저장소의 첫 커밋이 나왔다.** `simple_memo_app` 의 첫 커밋과 `daejong-page` 의 첫 커밋.

## 6. 두 번째 실행에서 터진다

밤 늦게 들어온 제보는 짧았다. *첫 실행은 정상. 종료하고 다시 열면 바로 크래시.* iOS 에서 재실행 시 **무조건** 터지는 패턴. lib 전반에서 원인 후보를 전수조사하기 시작했다.

- `main.dart` 부터 진입 흐름
- `MemoStorage` 의 `SharedPreferences` 로드
- `jsonDecode` 의 예외 경로
- `late` 변수와 `!` 연산자가 null 을 밟는 지점
- `initState` 안의 비동기 처리와 null 가드

후보를 좁혀 가며 `try-catch` 와 null 가드를 보강했지만 **완전한 해결은 다음 날로 넘어갔다**. 이 크래시는 04-12 의 첫 과제가 된다.

# 2026.04.13 작업일지 v1.1.0 — 약먹자, 원격 저장소의 공식 첫 커밋

> 새벽, 복용량 입력 UI 를 복용 시간과 같은 모양으로 다시 맞췄다.
> 오후, 빌드 clean + release run 한 줄 커맨드를 CLAUDE.md 에 적었다.
> 밤, **`4981029 Initial commit`** 이 `yakmukja` 저장소의 공식 시작점이 됐다.

코드 작업량은 앞 이틀보다 적었다. 그 대신 **약먹자를 깃허브에 처음 올리는 작업** 에 중심이 쏠렸고, 새벽 내내 UI 결을 맞추다 릴리즈 빌드까지 뽑은 뒤 초기 커밋으로 밀어냈다. 저녁부터는 하루의 성격이 리서치로 넘어가 텔레그램·Claude Code Channels 의 연동 가능성에 긴 대화가 이어졌다. 이 버전은 그 조용했던 하루를 다시 정리한 소급판이다.

## 1. 복용량이 복용 시간과 같은 모양이 되기까지

새벽 약먹자 작업은 **UI 일관성** 에서 출발했다. 복용량 입력은 별도 레이아웃을 쓰고 있었고 복용 시간은 다른 형태였다. 이걸 한 눈에 같은 종류로 읽히도록 맞추는 작업.

- 복용량 네모 카드 안에 라벨 + 추가 버튼을 복용 시간과 동일한 배치로
- 추가 버튼 탭 → 팝업으로 입력 창
- 입력 후 가운데 정렬로 찍히던 값 → **왼쪽 정렬** (복용 시간 쪽과 동일하게)
- 복용 시간 칸 아래에 **다음 복용까지 남은 시간** 실시간 카운트다운 추가
- 복용 체크 후 카운트다운이 사라지면서 레이아웃이 튀던 문제 → 고정 높이 고정
- 같은 약·같은 시간 중복 등록 방지 로직 추가
- 등록된 약이 하나도 없을 때 화면이 스크롤로 흔들리던 문제 → 내용이 있을 때만 스크롤되는 반응형으로 교체

## 2. 서브에이전트에게 표기를 물었다

시간 선택 휠의 **오전 0시 / 오후 12시 표기** 가 직관적이지 않다는 지적이 들어왔다. 두 표기는 실제로 같은 자정을 가리킬 수 있어 혼란이 생긴다. 어느 방향이 더 명확한지 서브에이전트 여러 명과 토론하고 결과를 합의했다.

결론: **중복 없는 24시간 관점 표기**. 오전/오후 체계를 버리는 대신 혼동의 여지를 원천 제거.

> **배운 것** — UI 용어가 "관습상 그렇게 쓴다" 는 이유만으로 그대로 유지되면 중의성이 따라붙는다. 도메인(약 복용)이 **중복 등록을 원천 차단해야 하는 곳** 이면 시간 표기도 거기 맞춰야 한다.

## 3. clean + release run 한 줄을 공용 규칙에 적다

UI 결이 어느 정도 자리 잡은 뒤 릴리즈 빌드 단계. 매번 `flutter clean` → `flutter run --release` 를 따로 치다 보니 **한 줄 커맨드로 묶어서 CLAUDE.md 에 적자** 는 이야기가 나왔다. 이 규칙이 약먹자 한 프로젝트에만 묶이면 안 된다는 판단도 같이 왔다.

결정: **공용 설정** 쪽 CLAUDE.md 에 넣는다. 그 결과 지금의 `/irun` 스킬(아이폰 clean + release 재빌드·실행)과 `/arun` 스킬(안드로이드 버전) 의 원형이 되는 규칙이 이 날 자리 잡았다.

추가로 같이 돌린 것:
- APK 최적화 한 바퀴 (용량 감소)
- 아이폰 빌드 커맨드도 동일 방식으로 한 줄화해 기록

## 4. 약먹자, 공식 시작점

준비가 되자 기존 시도 흔적을 걷어 낸 깨끗한 상태로 **초기 커밋** 을 만들었다.

```
4981029 Initial commit 약먹자 약 복용 알림 Flutter 앱
```

이 커밋이 `yakmukja` 저장소의 공식 시작점이다. 원격 반영 끝. 새벽 한 시 무렵이었다.

## 5. 메모요는 보정만, 본 브랜치 push 없음

이날 메모요는 워크트리 안에서만 이어졌다.

- 아이폰 빌드 커맨드를 clean + release run 한 줄 형태로 정리
- 메모 선택 시 상하 여백이 아래로 쏠려 가운데 정렬이 무너지던 것 수정
- 메모 사이 **노란색 구분선** 이 좌우 여백 때문에 화면 끝까지 안 채워지던 문제 → 패딩을 걷어 전체 너비로

본 브랜치 push 없음. 다음 버전 배포 빌드에 같이 포함하려고 일부러 보관.

## 6. Channels 를 리서치한 저녁

저녁부터 하루의 성격이 리서치로 바뀌었다. 질문은 *"텔레그램이나 디스코드로 페이크 채팅 같은 UI 를 연동할 수 있을까?"* 에서 출발해 **Claude Code Channels** 로 넘어갔다.

정리한 것:

| 항목 | Claude Code Channels | Claude 데스크탑 디스패치 |
|---|---|---|
| 단계 | 리서치 프리뷰 | 정식 기능 |
| 실행 위치 | 로컬 맥 | 로컬 맥 |
| 사용량 풀 | Claude Max 요금제와 공유 | 동일 |
| 인터페이스 | 슬랙·텔레그램 같은 **채팅 계층을 가운데** | 폰 앱이 맥에 직접 명령 |
| 성격 | 비동기 알림 + 대화 | 원격 실행 트리거 |

결론: **텔레그램을 채널로 붙여 사이드 프로젝트 브리핑 흐름을 만들고 싶다** 는 방향. 실제 Channels 설정은 Claude Code 세션이 아니라 별도 채팅 세션에서 진행하기로 미룸. (이 결정이 나중에 `@MyClaude` / `@Myclaude2` 봇 + `morning-briefing` / `todo-reminder` 파이프라인의 씨앗이 된다.)

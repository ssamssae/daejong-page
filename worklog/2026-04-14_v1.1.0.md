# 2026.04.14 작업일지 v1.1.0 — 텔레그램 봇이 드디어 페어링됐다

> 새벽, Channels 기능이 내 계정에 아직 열려 있지 않아 응답이 돌아오지 않았다.
> 아침, 새로 만든 **`@MyClaude`** 봇으로 페어링 코드까지 받아냈다.
> 밤, 알림이 도착하지 않는 문제를 여러 번 재현해 Stop 훅 필터 방향을 굳혔다.

이 날의 중심은 **텔레그램 봇을 Claude Code 에 물리는 것**. 동시에 메모요 UI 를 애플 다크 투톤으로 전면 리디자인하고 TestFlight 에 새 빌드를 밀어냈다. 이 버전은 그 두 갈래 흐름을 다시 짚은 소급판이자, **v1.0.0 시점부터 "그날 모든 세션을 한 문서로 합치는 통합 일지 포맷"** 이 정착된 첫 날의 기록이다.

## 1. Claude Max 계정에 플래그가 아직 안 열려 있었다

새벽 첫 시도: Claude Code Channels 로 텔레그램 봇을 붙이려 했는데 **응답이 돌아오지 않았다**. 구성은 맞게 됐는데 reply 가 오지 않았다. 원인 후보를 좁혀 보니 **계정 단위 서버 플래그** 가 아직 열려 있지 않을 가능성이 짙었다. 재시도를 돌리면서 플래그 활성화를 기다렸다.

> **배운 것** — 리서치 프리뷰 기능은 코드가 완벽해도 **서버측 플래그** 가 내 계정에 없으면 작동하지 않는다. 디버깅 전에 "이 기능이 내 계정에서 켜져 있는가?" 를 먼저 확인해야 토큰 낭비가 없다.

## 2. 페어링이 성공한 순간

아침, 새로 만든 **`@MyClaude`** 봇 기준으로 텔레그램 구성 스킬을 돌리고 토큰을 넣어 **페어링 코드** 까지 받아냈다. 실행 방식도 같이 굳혔다.

```bash
cc -c --channels plugin:telegram@claude-plugins-official
```

(이 실행 커맨드는 나중에 `~/bin/cc` 래퍼로 들어간다. 플래그가 빠지면 인바운드 먹통이라는 건 그 이후의 경험이 추가해 준 규칙이다.)

낮에는 친구 초대 가능 여부와 보안 설정(`/telegram:access` 의 원형) 을 정리. 저녁·밤에는 알림이 도착하지 않는 증상을 여러 번 재현하며 Stop 훅 설정과 필터 방향을 손봤다.

## 3. 텔레그램 세션일 때 Stop 알림은 skip

반복 재현 중 방향을 잡았다. **텔레그램 세션일 때는 Stop 훅 알림을 skip** 하는 게 자연스럽다. 텔레그램에서 출발한 발화에 대한 응답은 텔레그램 reply 로 이미 돌아가고 있으니 같은 내용을 Stop 훅이 별도로 또 보내면 중복 알림이 된다.

이 방향이 옳다는 확신이 서자 **메모리에 박아 두었다**. 앞으로 같은 고민을 반복하지 않도록. (이 메모리 한 줄이 지금까지 수차례 "알림 왜 안 오지?" 질문이 들어올 때마다 방향을 뒤집지 않게 막아 주는 guard 다.)

## 4. 권한과 자동 진행 정책을 같이 손봤다

텔레그램 경로가 열리면서 자연스럽게 나온 질문: **원격에서 시킨 명령을 얼마나 자동으로 진행해도 되는가?** 결론을 이렇게 정리해 메모리에 기록했다.

- **가역적 작업은 바로 진행** (에디터 수정·파일 생성·테스트 실행)
- **비가역/외부영향 작업만 사전 확인** (`rm -rf`, force push, 크레덴셜 등)
- 추천 자동 적용 모드: 가역 선택지는 "추천대로 진행합니다" 고지 후 즉시 실행

(이 규칙이 현재 `.claude/CLAUDE.md` 의 "빠른 원칙" 섹션의 원형이다.)

## 5. 메모요를 애플 다크 투톤으로 전면 리디자인

봇 작업과 병렬로 메모요 UI 를 뒤집었다.

- **배경**: 어두운 두 가지 톤으로 정리한 투톤 (`#000` / `#1C1C1E` 근사)
- **포인트 컬러**: amber 단일색으로 텍스트·아이콘 통일
- **버튼**: 편집·응원·저장·뒤로가기 전부 **하트 배지 느낌** 으로 맞추고 좌우 균형 다듬기
- **메모 목록**: 둥근 카드 형태로 전환
- **스플래시 + LaunchScreen**: 검은색 배경으로 통일

## 6. 복수 선택 삭제

기능 하나 추가. 좌상단 **편집 버튼** 으로 선택 모드 진입 → 메모 여러 개 체크 → 한 번에 지우기. 실수 방지용 **확인 다이얼로그** 도 같이 넣었다.

## 7. TestFlight 에 새 빌드

웹 빌드를 올리고, 버전을 **`1.0.0+5`** 로 범프해 TestFlight 에 새 빌드 push. 사용되지 않던 **쿠퍼티노 아이콘 의존성** 은 같이 정리.

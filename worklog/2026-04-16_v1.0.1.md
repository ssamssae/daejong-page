2026.04.16 작업일지 v1.0.1

오전과 저녁에 걸쳐 v1.0.0 을 쓴 이후 자정을 넘기며 계속 작업했다. 초점은 세 갈래로 나뉜다. 첫째는 개인 홈페이지 daejong-page 의 대대적 개편, 둘째는 메모요 베타테스터 사전예약 폼과 Google Apps Script 연동, 셋째는 자정 이후 새벽까지 이어진 메모요 베타테스터 Google Groups 자동 등록 인프라 구축이다. 한 번에 세 영역이 움직인 셈이지만 모두 한 가지 목표로 수렴한다. 메모요 Play Store 출시 준비와 사전예약 모객, 그리고 20명의 베타테스터 확보 자동화다.

홈페이지 쪽에서는 v1.3.0 으로 올리며 카카오톡 오픈채팅 버튼과 할일 목록 페이지를 새로 붙였고, 더치페이 웹 계산기와 개발 타임라인 페이지도 별도로 만들었다. 이후 타임라인은 포트폴리오 상세 영역에 인라인으로 통합했고, 포트폴리오 레이아웃 자체를 왼쪽 목록과 오른쪽 상세의 2컬럼 구조로 뜯어고쳤다. 작업일지도 같은 방식으로 좌우 배치로 바꿨다. 이어서 아이콘들을 실제 앱 아이콘 이미지로 교체하고, 순서를 타임라인, 약먹자, 더치페이, 메모요로 다시 정렬했다. 포트폴리오에 홈페이지 제작 항목을 새로 넣고 기본 선택을 메모요로 변경하는 것으로 오후 작업을 일단락했다.

저녁부터는 메모요 사전예약 수집 파이프라인을 만들기 시작했다. 포트폴리오 상세 페이지 안에 사전예약 이메일 입력 폼을 붙이고, 제출하면 Google Apps Script 의 doPost 로 연결되도록 연동했다. Apps Script 쪽은 여러 차례 방향이 바뀌었다. 처음에는 Play Console REST API 로 개별 이메일을 테스터로 직접 등록하려 했지만, edits.testers 엔드포인트는 googleGroups 배열만 받을 뿐 개별 이메일을 지원하지 않았다. Cloud Identity API 쪽도 개인 Gmail 계정에서는 사용할 수 없음을 확인하고 포기했다. 결국 memoyo-beta-testers@googlegroups.com 라는 Google Group 을 새로 만들어 Play Console 비공개 테스트에 그룹으로 연결했고, Apps Script 는 사전예약 이메일을 Google Sheets 에 기록하고 5 명씩 모일 때마다 관리자에게 리마인더를 보내는 가벼운 수집기로 용도를 바꿨다. 폼 쪽에서도 실시간 등록 수 표시, Apps Script 리다이렉트와 CORS 대응, 이메일 Enter 제출 같은 잔잔한 개선을 함께 반영했다.

자정이 지난 4월 17일 새벽에는 이전 세션의 컨텍스트를 이어받아 베타테스터 Google Groups 등록을 완전 자동화하는 쪽으로 방향을 잡았다. 처음 떠올린 아이디어는 Apps Script 가 사전예약 알림을 텔레그램 봇으로 보내면 같은 봇에 연결된 Claude Code 세션이 그걸 받아 Playwright 로 Google Groups 에 멤버를 추가하는 구조였는데, Telegram Bot API 특성상 봇이 스스로 보낸 메시지를 같은 봇으로 수신할 수 없다는 제약에 막혔다. 실제로 휴대폰에는 알림이 정상 도착하지만 Claude 세션에는 이벤트가 오지 않았다. 그래서 아키텍처 자체를 바꿨다. Telegram 을 트리거로 쓰는 것을 버리고 Google Sheets 를 단일 진실원으로 두었으며, 로컬 macOS 에서 launchd 가 5 분 간격으로 Node.js 워커 하나를 돌리는 폴링 방식으로 다시 설계했다. 이 경로에는 Claude 나 ChatGPT 같은 모델이 끼지 않아 토큰 소모가 사실상 제로다.

구현은 beta-signup/worker 하위에 worker.js 와 login.js 를 두는 구조로 잡았다. 워커는 Apps Script 에 pendingGroupAdd 엔드포인트를 호출해 status 가 registered 인 이메일 목록을 받아오고, 각 이메일에 대해 Playwright 로 groups.google.com 의 해당 그룹 회원 페이지로 이동해 회원 추가 다이얼로그를 연다. 그룹 멤버 입력란에 이메일을 넣고 초대 메시지를 채운 뒤 회원 직접 추가 체크박스를 누르고 회원 추가 버튼을 클릭한다. 경우에 따라 reCAPTCHA 체크박스가 뜨는데 로봇이 아닙니다 체크 한 번으로 대부분 통과된다. 이후 "회원이 업데이트되었습니다" 헤딩이 실제로 나타났는지 검증하고, 확인 버튼까지 눌러 다이얼로그를 닫는다. 성공하면 Apps Script 의 markGroupAdded 엔드포인트로 시트 상태를 group_added 로 바꾸고, 마지막에 한 번만 텔레그램으로 결과를 보낸다. 처리할 이메일이 없을 때는 아무 알림도 보내지 않아 매 5 분마다 노이즈가 쌓이지 않는다.

Playwright 가 Chromium for Testing 으로 Google 에 로그인하려 하면 "로그인할 수 없음 브라우저 또는 앱이 안전하지 않을 수 있습니다" 경고로 차단되는 문제가 있어 실제 Chrome 채널로 전환하고, disable-blink-features 와 enable-automation 제거, navigator.webdriver 오버라이드 같은 은폐 플래그를 더했다. 그러고도 처음에는 셀렉터가 visible 을 만족하지 못해 실패했는데, ARIA role 기반 locator 와 visible 대기로 바꾸니 안정적으로 버튼을 잡았다. 헤드리스 모드에서 버튼 텍스트를 찾지 못한 초기 실패와, 가짜 테스트 이메일을 대상으로 거짓 성공이 마크된 문제도 모두 고쳤다. 성공 판정은 반드시 "회원이 업데이트되었습니다" 헤딩이 떠야만 내도록 강제했다. launchd 는 com.memoyo.beta-worker.plist 로 등록해 5 분 간격으로 돌아가게 했고, 세션 파일은 beta-signup/worker/.chromium-profile 에 유지한다. 세션이 만료되면 login.js 를 headful 로 한 번 실행해 재로그인하면 된다. 실제 테스트에서도 사전예약 폼에 입력한 ssamssae@naver.com 이 5 분 안에 Google Group 에 자동 등록되는 것까지 확인했다.

그 밖에 개인 규칙과 스킬도 다듬었다. /todo 스킬에 todos.md 변경 시 daejong-page 의 todos 폴더에도 새 버전 스냅샷을 만들고 index.json 을 갱신해 push 하도록 홈페이지 자동 동기화 규칙을 추가했다. todos.html 도 worklog 와 동일한 좌우 레이아웃으로 다시 썼다. 메모리에는 메모요 사전예약에서 Google Groups 자동 등록까지의 전체 아키텍처와 Playwright 마지막 세 단계 체크포인트를 project 메모로 저장해, 새 세션에서도 이 규칙이 바로 적용되도록 해뒀다. 사이드 주제로 Telegram Bot API 의 outbound 메시지 비수신 이슈, Google 의 자동화 감지 패턴, launchd 와 CronCreate 의 토큰 소모 차이 같은 것들도 한 차례씩 정리했다.

남은 작업은 베타테스터 실제 모객이다. Google Play 비공개 테스트는 검토 대기 중이며, 승인이 나면 Reddit 과 클리앙 같은 커뮤니티에 준비해둔 모집글을 올려 20 명을 확보할 계획이다. Apple App Store 심사와 iOS UIPasteControl 이식은 할일 목록에 그대로 남아 있다.

# 2026.04.16 작업일지 v1.1.0 — 봇이 자기 자신을 수신 못 하는 날이었다

> 오후, 개인 홈페이지를 v1.3.0 으로 올리며 포트폴리오 2컬럼으로 뜯어고쳤다.
> 저녁, 메모요 사전예약 폼을 Google Apps Script 에 붙여 이메일을 수집하기 시작했다.
> 자정 너머, **Telegram 을 트리거로 쓰려던 아키텍처가 "봇이 자기 메시지를 수신 못 한다"** 는 API 제약에 부딪혔다.

이 날의 구조는 세 갈래다. 홈페이지 개편 + 사전예약 수집 파이프라인 + 베타테스터 Google Groups 자동 등록. 셋은 독립적으로 보이지만 모두 **메모요 Play Store 클로즈드 테스트 20명 확보** 라는 한 점으로 수렴한다. 중간에 아키텍처가 한 번 뒤집혔고, 그 결과 **모델(Claude·ChatGPT) 이 끼지 않아 토큰 소모가 사실상 0** 인 launchd 폴링 구조가 자리 잡았다. 이 버전은 그 세 갈래를 다시 짚는 소급판이다.

## 1. 홈페이지 2컬럼으로 다시 짜다

`daejong-page v1.3.0`. 변경이 누적된 하루였다.

- 카카오톡 오픈채팅 버튼 + 할일 목록 페이지 신설
- 더치페이 웹 계산기 + 개발 타임라인 페이지 별도 생성 → 타임라인은 **포트폴리오 상세에 인라인 통합**
- 포트폴리오 레이아웃을 **왼쪽 목록 + 오른쪽 상세의 2컬럼** 으로 재설계
- 작업일지도 같은 2컬럼 패턴으로 교체
- 프로젝트 아이콘을 실제 앱 아이콘 이미지로 교체
- 정렬 순서: 타임라인 → 약먹자 → 더치페이 → 메모요
- 포트폴리오에 "홈페이지 제작" 항목 추가, 기본 선택 메모요로 변경

오후 작업은 이쯤에서 일단락.

## 2. 사전예약 폼, Apps Script 에 바로 붙이기

저녁부터 메모요 사전예약 수집 파이프라인. 포트폴리오 상세 페이지 안에 이메일 입력 폼을 붙이고, 제출하면 Google Apps Script 의 `doPost` 로 연결.

처음 설계는 단순했다. *폼 입력 → Apps Script → Play Console 비공개 테스터 자동 등록*. 그런데 Apps Script 쪽에서 **두 번 막혔다**.

### 2.1 `edits.testers` 엔드포인트는 googleGroups 배열만 받는다

Play Console REST API 의 `edits.testers` 는 **개별 이메일을 받지 않는다**. `googleGroups` 배열만 지원한다. 개별 초대 자동화는 이 API 로 불가.

### 2.2 Cloud Identity API 는 개인 Gmail 에서 불가

그럼 Cloud Identity 쪽은? **개인 Gmail 계정에서는 사용할 수 없음** 을 확인. 포기.

### 2.3 방향 전환 — 그룹 하나를 만들고 거기에 붙이자

결국 `memoyo-beta-testers@googlegroups.com` 이라는 **Google Group 을 새로 만들어** Play Console 비공개 테스트에 **그룹으로** 연결. Apps Script 는 용도 변경 — 사전예약 이메일을 Google Sheets 에 기록하고 5명씩 모일 때마다 관리자에게 리마인더만 보내는 **가벼운 수집기** 로 축소.

폼 쪽은 잔잔한 개선 몇 건:
- 실시간 등록 수 표시
- Apps Script 리다이렉트 + CORS 대응
- 이메일 Enter 제출

## 3. 봇이 자기 자신을 수신 못 한다

자정을 넘긴 04-17 새벽. 베타테스터 **Google Groups 등록 자동화** 쪽으로 손을 뻗었다. 처음 그림은 이랬다.

```
Apps Script 사전예약 알림 → Telegram 봇 → Claude Code 세션 수신 → Playwright → Google Groups 회원 추가
```

구현하고 테스트. **휴대폰에는 알림이 정상 도착**. 그런데 **Claude 세션에는 이벤트가 오지 않았다**.

원인은 Telegram Bot API 의 근본 제약이었다. **봇이 스스로 보낸 메시지를 같은 봇으로 수신할 수 없다**. 봇 sendMessage 의 echo 가 같은 봇의 getUpdates 로 돌아오지 않는다.

> **배운 것** — Telegram Bot API 로 "내 봇 → 내 봇 이벤트 트리거" 체인을 만들려고 하면 근본이 막힌다. 외부 사용자 메시지를 받는 데 설계된 API 다. 자체 트리거가 필요하면 **Telegram 을 중간 버스로 쓰지 말고 별도 trigger plane(파일·시트·DB) 을 두는 쪽이 빠르다**.

### 3.1 아키텍처 재설계 — 시트를 단일 진실원으로

Telegram 트리거 구조를 버리고 **Google Sheets 를 단일 진실원** 으로 두는 쪽으로 전환.

```
사전예약 폼 → Apps Script → Sheets (status=registered)
                                 ↓
     [로컬 macOS launchd 5분 간격]
                                 ↓
       Node.js 워커가 pendingGroupAdd 조회
                                 ↓
       Playwright 로 Google Groups 회원 추가
                                 ↓
       markGroupAdded → Sheets (status=group_added)
                                 ↓
       한 번만 Telegram 결과 알림
```

**모델(Claude·ChatGPT) 이 끼지 않아 토큰 소모가 사실상 0**. launchd 가 관리하는 고정 주기 폴러가 중심이다.

## 4. 구현 디테일 — Playwright 가 Google 로그인에 막히다

`beta-signup/worker/` 아래 `worker.js` + `login.js`.

**Playwright 가 Chromium for Testing 으로 Google 로그인하려 하면** `"로그인할 수 없음, 브라우저 또는 앱이 안전하지 않을 수 있습니다"` 경고로 차단. 조치:
- **실제 Chrome 채널** 로 전환
- `--disable-blink-features=AutomationControlled` 추가
- `enable-automation` 플래그 제거
- `navigator.webdriver` 오버라이드

셀렉터도 초반엔 `visible` 을 만족 못 해서 실패. **ARIA role 기반 locator + visible 대기** 로 바꾸니 안정화. 헤드리스 모드에서 버튼 텍스트를 못 찾던 초기 실패와, 가짜 테스트 이메일을 대상으로 **거짓 성공이 마크되던 문제** 도 같이 고쳤다.

**성공 판정** 은 반드시 `"회원이 업데이트되었습니다"` 헤딩이 떠야만 내도록 강제. 이걸 통과해야 `markGroupAdded` 로 시트 상태를 바꾸고 Telegram 에 결과 알림이 한 번 나간다. 처리할 이메일이 없으면 아무 알림도 안 보내 매 5분 노이즈가 쌓이지 않음.

launchd 는 `com.memoyo.beta-worker.plist` 로 등록, 5분 간격 실행. 세션 파일은 `beta-signup/worker/.chromium-profile` 에 유지. 만료 시 `login.js` 를 headful 로 한 번 돌려 재로그인.

실제 테스트: 사전예약 폼에 입력한 `ssamssae@naver.com` 이 **5분 안에 Google Group 에 자동 등록** 되는 것까지 확인. 파이프라인 end-to-end 동작 확인.

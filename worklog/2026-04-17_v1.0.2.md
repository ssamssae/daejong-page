그 뒤에는 스크린샷 삼 장을 직접 촬영하는 단계였다. 폰을 꺼내 수동으로 찍는 대신 iPhone 17 Pro 시뮬레이터를 부팅해서 자동화로 진행했다. FVM 스테이블의 Flutter 로 pub get 한 뒤 iOS 시뮬레이터용 디버그 빌드를 만들어 com.daejongkang.simpleMemoApp 번들을 설치했고, 첫 실행에서 열린 것은 이전에 수동 테스트한 더미 데이터 열한 건이었다. 숫자만 쓰여 있는 부자연스러운 메모들이라 홍보용으로 쓸 수 없어서 교체 작업을 시작했다. 메모요가 shared_preferences 에 memos 라는 키로 JSON 문자열을 저장하는 구조였기 때문에, 파이썬으로 즐겨찾기 네 건과 일반 여섯 건 총 열 건짜리 리얼한 한국어 메모 시드 JSON 을 만든 뒤 plistlib 로 플리스트에 단일 문자열로 삽입했다. 처음 PlistBuddy 방식은 JSON 값이 파싱된 형태로 저장되어 Flutter 디코드가 실패하는 문제가 있었고, 다음으로 plutil 로 쓴 뒤 앱을 다시 켜도 cfprefsd 캐시 때문에 옛 데이터가 그대로 떠올라 애를 먹었다. 결국 plist 를 파일로 만든 뒤 xcrun simctl spawn 으로 시뮬레이터 안에서 defaults import 를 돌리는 방식이 cfprefsd 에도 정상적으로 전달되어 깔끔하게 교체됐다.

새 데이터로 첫 Hero 샷을 찍고 나서 편집 화면 샷을 얻기 위해 cliclick 으로 시뮬레이터 창 좌표에 마우스 클릭을 보냈다. 시뮬레이터 창 위치와 크기를 AppleScript 로 읽어온 뒤 디바이스 픽셀에서 화면 픽셀로 스케일을 계산해 특정 메모를 탭하는 방식이었다. 첫 번째 탭은 원하는 행과 한 칸 어긋난 운동 루틴 편집 화면을 열었고, 두 번째 시도에서도 한 칸 위인 코드 리뷰 피드백 정리 화면이 열렸다. 세 번째에서 목표했던 이천이십육 Q2 목표 메모를 정확히 열 수 있었고 이 장면이 Action 샷으로 확정됐다. 여러 줄 구조화된 할일과 매주 월요일 진척 확인이라는 주석, 그리고 메모요 정식 출시 라는 메타 참조까지 들어가 있어 편집 화면 전달력이 좋았다.

세 번째 샷으로 원래 계획했던 스와이프 인터랙션은 중도 포기했다. 리스트 화면에서 뒤로 버튼과 + 버튼 주변의 탭이 일부 좌표에서 비가역적으로 먹통이 되는 현상이 반복돼서 스와이프 드래그 타이밍 샷을 만들 수 없었다. 대신 시뮬레이터 데이터를 완전히 날리는 방향으로 전환했다. xcrun simctl uninstall 로 앱을 지우고 다시 설치해 진짜 최초 실행 상태를 얻었더니 메모가 없습니다 새 메모를 추가해보세요 라는 중앙 안내 문구와 큰 노란색 + 버튼이 보이는 깨끗한 빈 화면이 나왔다. 역설적으로 스와이프 샷보다 이쪽이 미니멀 철학을 훨씬 강하게 증명하는 장면이어서 Onboarding 샷으로 채택했다. 결과적으로 Hero 리스트, Action 편집, Onboarding 빈 상태 세 장 구성이 확정됐고 screenshots 폴더에 memoyo-beta-01-list.png, memoyo-beta-02-edit.png, memoyo-beta-03-empty.png 로 저장해 앱 저장소 커밋 9dc172e 로 푸시했다.

## 3. 시뮬레이터 시드 데이터 교체기

폰 대신 **iPhone 17 Pro 시뮬레이터** 로 자동화. FVM 스테이블 Flutter → `pub get` → iOS 시뮬레이터용 디버그 빌드 → `com.daejongkang.simpleMemoApp` 설치.

첫 실행에서 열린 건 이전 수동 테스트 때 남은 **더미 데이터 11건**. 숫자만 적혀 있는 부자연스러운 메모라 홍보용으로 못 쓴다. 교체 시작.

메모요는 `shared_preferences` 에 **`memos`** 라는 키로 JSON 문자열을 저장. 파이썬으로 **즐겨찾기 4건 + 일반 6건 = 10건** 짜리 리얼 한국어 메모 시드 JSON 을 만들어 plist 에 삽입하는 방향.

여기서 **세 번의 시행착오** 가 있었다.

1. **`PlistBuddy`** 로 쓰기 → JSON 값이 "파싱된 형태" 로 저장돼 **Flutter decode 실패**
2. **`plutil`** 로 쓰기 → 앱을 다시 켜도 **`cfprefsd` 캐시 때문에 옛 데이터가 그대로** 올라옴
3. **`xcrun simctl spawn ... defaults import`** → 시뮬레이터 내부에서 `defaults` 로 쓰면 cfprefsd 에도 정상 전달. **깔끔히 교체됐다.**

> **배운 것** — iOS 시뮬레이터의 NSUserDefaults 를 외부에서 건드릴 때 `PlistBuddy`·`plutil` 둘 다 cfprefsd 캐시를 무시한다. **`xcrun simctl spawn <uuid> defaults import`** 로 "시뮬레이터 안쪽에서" 쓰는 게 유일한 안전 경로다.

## 4. Action 샷 좌표가 세 번 엇나갔다

Hero 샷을 찍고 다음은 편집 화면. `cliclick` 으로 시뮬레이터 창 좌표에 마우스 클릭. 창 위치·크기를 AppleScript 로 읽어와 디바이스 픽셀 → 화면 픽셀 스케일 계산 후 특정 메모를 탭하는 방식.

- **첫 탭**: 목표 한 칸 아래의 "운동 루틴" 편집 화면이 열림
- **두 번째 탭**: 한 칸 위의 "코드 리뷰 피드백 정리" 화면이 열림
- **세 번째 탭**: 드디어 목표한 **"2026 Q2 목표"** 메모 정확히 열림

이 메모는 여러 줄 구조화된 할일 + "매주 월요일 진척 확인" 주석 + "메모요 정식 출시" 메타 참조까지 들어가 있어 **편집 화면 전달력이 가장 좋았다**. 이게 Action 샷으로 확정.

## 5. 스와이프 인터랙션을 버린 순간

세 번째 장면으로 원래 계획한 스와이프 인터랙션. **리스트 화면에서 뒤로 버튼과 + 버튼 주변의 탭이 일부 좌표에서 비가역적으로 먹통**이 되는 현상이 반복돼 스와이프 드래그 타이밍 샷을 만들 수 없었다.

방향 전환. `xcrun simctl uninstall` → 재설치 → **진짜 최초 실행 상태** 확인. 화면에 뜬 건 *"메모가 없습니다. 새 메모를 추가해보세요."* 중앙 안내 + 큰 노란색 `+` 버튼. **미니멀 철학을 스와이프보다 훨씬 강하게 증명**. Onboarding 샷으로 채택.

결과 3장 구성 확정:

```
memoyo-beta-01-list.png   # Hero (채워진 리스트)
memoyo-beta-02-edit.png   # Action (2026 Q2 목표 편집)
memoyo-beta-03-empty.png  # Onboarding (빈 상태)
```

## 7. TherapyNote AI 가 반려된 이유

중간에 사이드 프로젝트 방향 조율. 오늘 자 브리핑이 추천한 **TherapyNote AI 심층 분석** 6파트를 보냈는데 피드백이 돌아왔다. *규제 러닝커브, 법무 자문 비용, 도메인 깊이 요구가 초보 바이브코더 입장에서 너무 무겁다.*

재제안: 저비용·저리스크 아이디어 5개. 채택된 건 **한줄일기 + AI 응원 앱**. 토큰비를 GPT-4o mini 기준 일 1회로 가정하면 1000명 규모에서도 **월 3천 원 수준**. API 키 보호는 **Cloudflare Workers 서버리스 프록시 50줄**.

실제 구현은 현재 메모요 출시 트랙이 우선이라 todos 의 사이드 프로젝트 아이디어 섹션에 백로그로 저장. 미리알림 Claude 목록에도 동시 동기화 후 커밋 `042337b` 로 기록. 이 상담을 반영해 **사용자 프로필 메모리** 에 "초보 비개발자 바이브코더, 저비용·저리스크 선호, 규제·에이전트·복잡 백엔드 기피" 기준 저장.

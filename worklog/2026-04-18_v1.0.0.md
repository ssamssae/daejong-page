오늘은 홈페이지 쪽 시각적·구조적 개편이 하루의 중심이었다. 전날 밤 대충 정리해둔 FAB 와 포트폴리오 히어로의 스케치를 실제 코드로 옮기는 일부터 시작했고, daejong-page 전체에 대대적 리디자인을 반영했다. 상단 FAB 는 이메일과 카카오톡을 좌측 그룹으로, 하트를 단독으로 우측에 두어 32x32 원형 스타일로 통일했고, 하단 FAB 다섯 개는 그리드·연필·체크리스트·체크·마법봉 아이콘으로 교체하면서 포트폴리오 카드를 1번 자리에 올려 시선 흐름을 만들었다. 포트폴리오 섹션 자체도 단일 리스트에서 히어로(메모요+사전예약) + 4카드 그리드 구조로 옮겨 무엇을 먼저 보여줄지 우선순위를 시각화했다. 전체 UI 대개편은 커밋 bc54e11 로 한 번에 반영됐다.

다음은 세 개의 대시보드 페이지를 연달아 개편하는 일이었다. todos.html 은 히어로 카드·필터 칩·D-day 배지·스냅샷 드로어 네 가지를 새로 얹어, 단순히 "오늘 할일 목록"이던 페이지를 "현재 상태와 이력을 동시에 보는 대시보드"로 바꿨고, 이 작업은 커밋 4618412 로 기록됐다. done.html 은 스트릭 카운터·깃허브 스타일 히트맵·프로젝트 태그 분포·카테고리별 막대 그래프 네 블록을 추가해서 "끝낸 일을 어떻게 쌓아왔는지" 한눈에 보이도록 재설계했고 커밋 9e2b463 로 반영됐다. 세 번째로는 보류·취소 페이지(cancelled.html)를 아예 신설했다. 기존에 todos.md 안 "보류/취소" 섹션으로만 남아있던 항목들이 공개 타임라인 위로 올라왔고, 초기 버전은 세 건 취소를 반영한 상태로 커밋 59b301f 에 담았다.

페이지 개편과 나란히 과거 기록 정리 트랙도 돌렸다. 4/11 부터 4/14 까지는 /todo 스킬이 도입되기 전 날짜라 홈페이지 todos 디렉토리에 스냅샷 파일 자체가 없었는데, worklog 를 역참조해 4건을 사후 재구성했고 커밋 265d61a 로 반영했다. todos 스냅샷의 파일명 규칙도 "하루 1파일" 덮어쓰기 체계로 통일하는 리팩터를 커밋 2a798c5 에 넣어 04-15 첫날 스냅샷까지 같이 복원했다. 작업일지 쪽도 기존 7일치 산문체 파일을 스탠드업+태그 포맷으로 평행하게 재작성해 커밋 bdd2ccf 에 담았는데, 기존 v1.0.X 산문체 버전 파일은 그대로 두고 통합 스탠드업 파일만 새로 생성하는 방식이라 과거 내러티브는 손상 없이 보존됐다.

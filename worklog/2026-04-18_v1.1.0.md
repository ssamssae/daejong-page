# 2026.04.18 작업일지 v1.1.0 — 홈페이지가 대시보드가 된 날

> 오전, 전날 밤에 스케치만 남은 FAB 배치를 실제 코드로 옮기기 시작했다.
> 오후, `todos.html` 과 `done.html` 이 "오늘 할일 목록" 과 "끝낸 것의 기록" 에서 각각 **대시보드** 로 바뀌었다.
> 저녁, 자기소개 문구를 바꾸고 "AI 옆에 앉은 사람" 이라는 포지셔닝을 메인 페이지에 그대로 드러냈다.

이 날의 중심은 **홈페이지 시각적·구조적 대개편**. 개별 페이지를 하나씩 뜯어고쳤고, 과거 기록을 사후 재구성하는 정리 트랙이 병행됐다. 스킬 쪽에서도 두 건이 늘었다 — `/review-status-check` 와 `/morning-briefing`. 할일 관리 계보도 많이 정리됐다. 이 버전은 그 여러 갈래를 다시 이어 본 소급판이다.

## 1. FAB 배치 + 포트폴리오 히어로

전날 밤 대충 정리해 둔 FAB 와 포트폴리오 히어로 스케치를 코드로 옮기는 일부터 시작.

- **상단 FAB**: 이메일·카카오톡을 **좌측 그룹**, 하트를 **단독 우측**. 32×32 원형 스타일로 통일
- **하단 FAB 5개**: 그리드·연필·체크리스트·체크·마법봉 아이콘으로 교체
- 포트폴리오 카드를 **1번 자리** 에 올려 시선 흐름 만들기
- 포트폴리오 섹션을 **단일 리스트 → 히어로(메모요+사전예약) + 4카드 그리드** 구조로 재편

전체 UI 대개편은 커밋 `bc54e11` 한 번에 반영.

- 히어로 카드
- 필터 칩
- D-day 배지
- **스냅샷 드로어** — 날짜별 이력 훑기

## 3. done.html 을 "어떻게 쌓아왔는지" 로 재설계

- 스트릭 카운터
- GitHub 스타일 **히트맵**
- 프로젝트 태그 분포
- 카테고리별 막대 그래프

커밋 `9e2b463`.

## 4. cancelled.html 신설 — 보류·취소를 공개 타임라인에

기존 `todos.md` 안에 "보류/취소" 섹션으로만 남아 있던 항목들을 **공개 타임라인 위로** 끌어올렸다. 초기 버전은 **3건 취소** 를 반영한 상태로 시작.

커밋 `59b301f`.

> **배운 것** — 끝내지 못한 일을 숨기지 않고 **왜 드롭됐는지** 까지 공개로 띄우면 포트폴리오의 신뢰 축이 한 단계 위로 간다. 성공만 보여주는 쪽보다 "여기까지 했고 이 이유로 멈췄다" 가 더 전달력이 있다.

2026.04.20 작업일지 v1.0.2

v1.0.1 스냅샷이 오전 8시 44분에 찍힌 뒤부터 10시 20분 대까지는 거의 전부 맥 본진에서 돌아간 스킬과 자동화 정비의 연속이었다. 앱 코드 자체의 수정은 없었고, 대신 아이폰 배포 파이프라인과 이슈 기록 체계, 그리고 매일 새벽에 돌아가는 동기화 자동화의 뼈대가 이 구간에 거의 다 올라갔다. 흐름이 하나의 주제로 이어져 있어서 이 버전은 "개발 환경 자체를 한 단계 굳힌 오전" 이라고 불러도 좋다.

먼저 9시 15분에 맥 쪽 종단 스킬인 land가 올라갔다. 윈도우 데스크탑에서 to-iphone 슬래시 커맨드로 쏜 Flutter 앱 URL을 맥이 받아서 clone이나 pull을 정리하고 iOS 플랫폼 세팅과 pub get, pod install을 거쳐 최종적으로 irun으로 아이폰에 릴리즈 설치까지 이어 주는 역할이다. 같은 시각 daejong-page의 skills 페이지에도 land 카드를 같이 추가해 홈페이지에서 파이프라인의 맥 쪽 종단이 어떤 모양인지 바로 보이도록 했다. 이어서 9시 17분에는 WSL 쪽 짝인 to-iphone 스킬이 원샷 형태로 붙었고, 10분쯤 뒤인 9시 52분에는 초기 설계에서 들어가 있던 SSH primary 경로를 걷어 내고 텔레그램 트리거를 기본 경로로 바꾸는 정리가 한 번 더 들어갔다. 결과적으로 WSL에서 to-iphone 한 줄을 치면 GitHub 푸시와 텔레그램 트리거로 맥이 깨어나 land가 돌고, 아이폰에 앱이 재설치되는 엔드투엔드 루프가 손에 잡히는 상태가 됐다.

land 스킬의 후속으로 9시 54분과 55분에 todo 저장소와 daejong-page에 각각 2건의 자동화 내장 항목이 들어갔다. signing team 처리와 Bundle ID 관리를 land 스킬 안에서 자동으로 해 주도록 다듬어야 한다는 메모가 그중 하나고, 나머지는 향후 다듬기 지점을 휘발되지 않게 문서화한 성격이다. 홈페이지 todos 탭에는 같은 내용이 즉시 반영돼 있어 이후 세션이 열리면 바로 이어서 볼 수 있다.

10시 3분에는 그동안 맥과 WSL에 흩어져 있던 launchd 플리스트와 systemd 유닛을 한 곳에 묶기 위해 claude-automations 저장소를 새로 파고 첫 입주로 context-threshold-alert을 넣었다. 이 저장소의 install.sh 하나만 돌리면 기기 종류에 맞게 자동 스케줄이 걸리는 구조를 지향하기 때문에, 앞으로 새 자동화가 생기면 skills 저장소가 아니라 이쪽으로 들어가게 된다. 직후인 10시 22분대에는 daily-sync-and-learn 스크립트가 이 저장소에 얹혔다. 이 스크립트는 매일 새벽 6시 45분에 맥과 WSL 양쪽에서 각자 독립적으로 실행되면서 claude-skills와 claude-automations를 git pull rebase로 당겨 오고, 실패하면 경고만 남기고 계속 진행한다. 더불어 skills의 issues 폴더를 스캔해 reference_issue_history 메모리를 재생성하고, 최근 30일 이슈 중 무작위 3건을 골라 "오늘의 복기" 로 텔레그램에 요약을 보낸다.

이 자동 실행 버전과 짝을 이루는 수동 스킬 sync가 같은 10시 22분에 skills 저장소에 추가됐다. 자정이나 방금 막 데스크탑에서 push 한 내용을 맥에서 바로 받고 싶을 때 sync를 호출하면 동일한 daily-sync-and-learn 스크립트가 즉시 돌기 때문에, 자동과 수동의 동작 일관성이 보장된다. 중요한 점은 이 스킬이 호출한 기기만 pull 한다는 것이다. 맥에서 sync를 돌리면 맥의 skills와 automations만 최신화되고, WSL은 여전히 자체적으로 sync나 새벽 6시 45분 자동 실행을 거쳐 GitHub 허브에서 당겨 오는 구조다.

10시 15분과 21분 사이에는 issue 스킬이 신설되고 오늘 몫 이슈 2건이 소급해서 retro 형태로 기록됐다. 맥에서든 WSL에서든 같은 포맷으로 issues 저장소의 YYYY-MM-DD-slug.md 에 쌓이도록 해서, 기기 간 이슈 히스토리가 같은 파일 구조로 공유되는 것이 포인트다. 이 위에 앞서 만든 daily-sync-and-learn이 돌기 때문에 이슈 파일이 하나 늘어날 때마다 다음 날 새벽에 메모리가 갱신되고 내 기기들이 같은 학습을 반복해서 하는 순환이 완성된다. 직후 10시 21분에는 맥에서 별도로 올라온 피드백 4건 중 1,2,3번이 코드로 반영되고 4번은 비공개 처리로 확정됐다.

마무리는 10시 23분의 gitignore 정리였다. 초기 커밋 때 실수로 올라간 pyc 캐시 파일들을 털어 내고 __pycache__ 패턴을 ignore 에 넣었다. 파이썬 스크립트를 실제로 자주 돌리는 저장소인 만큼 이 청소가 이후 git status 를 계속 깨끗하게 유지해 줄 것이다.

v1.0.1까지에서 WSL이 밤새 혼자 한컵과 mini_expense 앱, Dev Trend Curator와 flutter-factory 네 조각을 쌓아 올린 흐름을 잇는 오전이었다. 그 결과물을 맥 쪽에서 어떻게 받아 아이폰까지 보낼지, 그리고 앞으로 생길 새 이슈와 자동화가 두 기기 사이에서 어떻게 한 덩어리로 흐를지를 정하는 연결 조직이 이번 버전에서 대부분 자리를 잡은 셈이다.

남은 작업은 to-iphone에서 land로 이어지는 엔드투엔드 루프를 한컵이나 mini_expense 같은 실앱으로 한 번 돌려 실제 설치까지 확인하는 것, land 스킬의 signing team과 Bundle ID 자동화 2건을 실제 구현으로 당겨 오는 것, 그리고 내일 새벽 6시 45분에 처음으로 자동 실행될 daily-sync-and-learn의 텔레그램 요약이 예상한 모양대로 오는지 확인하는 것이다.

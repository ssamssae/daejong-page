2026.04.20 작업일지 v1.0.3

v1.0.2 가 오전 10시 23분에 마감된 뒤부터 저녁 18시대까지는 전부 맥 본진에서 이어진 흐름이었다. 오전 버전이 배포 파이프라인과 자동화의 뼈대를 세운 구간이었다면, 이번 버전은 그 위에 이슈 공개 체계와 스킬 자동화, 그리고 외부에 내보내는 첫 결과물인 뉴스레터 1호까지 얹은 오후였다.

가장 먼저 달린 건 issues 공개 체계였다. 오전에 만든 /issue 스킬이 저장소 내부에서는 이슈를 YYYY-MM-DD-slug.md 포맷으로 쌓게 돼 있었지만, 외부에서 이게 어떻게 보이는지가 비어 있었다. 그래서 daejong-page에 /issues 라는 독립 페이지를 신설했다. 홈페이지 안에 포스트모템 타임라인 전용 뷰를 넣고, 각 이슈마다 심각도 뱃지와 재발 여부 뱃지를 붙여서 한 번 쓴 이슈가 또 반복되면 시각적으로 바로 잡히도록 만들었다. 카드 구조는 증상과 원인, 조치, 재발 방지의 4섹션으로 고정해 포스트모템 포맷이 누가 열어도 같은 눈금으로 읽히게 했다.

페이지만 놓고 보니 맥락이 비어 있어서 4월 11일부터 어제까지 흩어져 있던 과거 이슈를 retro 로 9건 소급 기록했다. claude-skills 저장소 쪽에 먼저 원본을 심고, 그걸 daejong-page 공개본으로 다시 옮기는 순서를 거쳤다. 이어서 이슈가 하나도 없던 날짜에는 빈 날짜 노드를 "없음" 상태로 표시해서 타임라인이 끊겨 보이지 않게 처리했다. 이 패턴은 cancelled.html 에서 이미 한 번 검증한 방식이라 그대로 재사용했다.

같은 흐름으로 /issue 스킬 본체에도 한 단계를 더 새겨 넣었다. SKILL.md 에 14-d 단계를 추가해서, 앞으로 이슈가 새로 기록될 때마다 스킬이 스스로 daejong-page 공개본 동기화까지 자동으로 처리하도록 바꿨다. 오늘처럼 수동으로 한 번씩 옮길 필요 없이, 이슈 저장이 곧바로 공개 페이지 갱신으로 이어지는 루프가 완성됐다.

워크로그와 완료리스트 쪽 스킬 두 개에도 오늘 오전에 새로 만든 land 스킬과 같은 기기 라우팅 패턴을 이식했다. 0단계에서 hostname 으로 맥인지 아닌지 확인하고, 맥이 아니면 텔레그램으로 트리거 한 줄만 쏜 뒤 종료한다. 맥 창에서 강대종님이 직접 커맨드를 한 번 찍어 줘야 비로소 로컬 실행이 시작되는 구조다. SSH 자동 실행처럼 다른 기기에서 마음대로 맥을 깨우는 방식이 아니라, to-iphone 에서 land 로 이어지는 패턴과 동일하게 텔레그램 트리거 뒤에 명시적 승인 단계를 두는 쪽으로 정리했다. 같은 기기 라우팅이 세 개 스킬에 걸쳐 동일한 형태로 들어갔으니, 앞으로 맥 종속 스킬을 추가할 때 이걸 템플릿처럼 복제해 쓰면 된다.

오후의 두 번째 큰 덩어리는 land 스킬의 자동화 내장이었다. 오전에 메모로만 남겨 뒀던 두 가지를 실제 구현으로 끌어왔다. 하나는 iOS signing team 자동 주입이다. 한컵과 mini_expense 에서 수작업으로 CODE_SIGN_STYLE 뒤에 DEVELOPMENT_TEAM 46UH85U2B8 을 sed 로 꽂아 넣었는데, 이걸 land 스킬 본체에 내장해서 세 번째 앱부터는 같은 수작업이 사라지도록 했다. 또 하나는 Bundle ID 자동 교정이다. 플러터 기본 템플릿이 com.example.<name> 으로 만드는 것을 com.ssamssae.<name> 으로 바꾸는 부분을 한컵에서 수동으로 처리했는데, 이것도 sed 패치로 스킬 안에 넣었다. 두 자동화가 합쳐지면 세 번째 앱부터는 land 한 줄로 signing 도 Bundle ID 도 자동으로 제자리에 들어가니까, WSL 의 to-iphone 에서 맥의 land 로 이어지는 루프가 완전 무인으로 돌아가는 상태가 된다. 이 변경은 claude-skills 저장소에 1번 PR 로 올라가 머지 대기 중이다. daejong-page 의 skills 페이지 land 카드 설명도 Bundle ID 와 signing team 자동 주입 내용을 반영해 갱신했다.

todo 쪽에서는 이 두 자동화 항목을 완료로 옮겼고, 홈페이지 todos 탭에도 같은 상태가 즉시 반영됐다.

마지막으로 오후의 하이라이트는 바이브코딩 뉴스레터 1호 발행이었다. "3시간 만에 나만의 로컬 AI 어시스턴트 만들기" 라는 제목으로, 어제 저녁 9시 30분부터 자정까지의 기억이 제작 타임라인을 그대로 원고로 풀었다. apt 중간에 Ctrl-Z 를 눌러 프로세스가 좀비로 남았던 지점, systemd 유닛 파일에서 Service 헤더가 빠져 실행 단계에서 막혔던 지점, 봇이 중복 폴링으로 같은 메시지를 두 번 받던 지점까지 막힌 지점과 그 해결 과정을 시간 순서대로 기록했다. daejong-page 에 뉴스레터 카드와 본문이 올라가면서 사이드 프로젝트 브리핑에 묶여 있던 아이디어 중 하나가 실제 외부 공개 결과물로 처음 나왔다.

오전 v1.0.2 가 "개발 환경 자체를 한 단계 굳힌 오전" 이었다면, 오후 v1.0.3 은 그 환경 위에서 결과물이 공개 창구로 흐르기 시작한 구간이다. 이슈 포스트모템은 /issues 페이지로, 스킬 자동화는 claude-skills 1번 PR 로, 기억이 제작기는 뉴스레터 1호로 각자의 공개 통로를 통해 바깥에 내보낼 수 있는 상태가 됐다.

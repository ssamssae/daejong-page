> 00:00, 맥 세션에서 약먹자 비공개 테스트 제출과 기억이 챗봇 Markdown 크래시를 동시에 잡고 있었다.
> 새벽 내내, WSL 데스크탑이 **혼자** 한컵·mini_expense 두 앱과 Dev Trend Curator·flutter-factory 두 자동화를 쌓아 올렸다.
> 오전, `claude-automations` 저장소를 새로 열고 `/land` / `/to-iphone` / `daily-sync-and-learn` 을 한 번에 심었다.

이 날은 **만들어진 것이 공개 창구로 흐르기 시작한 구간**. 새벽은 WSL 의 야간 자동 개발 혼자, 오전은 맥의 파이프라인·이슈·동기화 인프라 일괄 구축, 오후는 /issues 페이지 + 바이브코딩 뉴스레터 1호 발행으로 이어졌다. v1.0.0~v1.0.3 네 스냅샷을 거쳐 일어난 일들을 한 호흡으로 다시 잇는 소급판이다.

## 1. 00:00 — 몰도바 국가 도메인의 습격

자정, 기억이 챗봇의 *"답변이 안 온다"* 제보가 들어왔다. 데스크탑 WSL 의 `systemd` 로그를 열어 보니 `gieogi-bot.service` 는 살아 있는데 **텔레그램 `BadRequest` 로 계속 크래시**하고 있었다.

원인 추적.

```
src/기억이/src/bot.py : 102
  parse_mode="Markdown"
```

LLM 답변에 **짝이 맞지 않는 언더바/별표** 가 섞여 들어가면 텔레그램이 거부하는 구조였다. Markdown 전송 실패 시 plain text 로 재시도하는 **fallback 분기** 를 넣고 `systemctl --user restart` 요청. 사용자가 "재시작이 뭐냐" 고 물어 *컴퓨터 재부팅이 아니라 봇 프로세스만 다시 올리면 된다* 고 안내.

다음 이슈는 답변에 **중국어가 섞여 나오는 문제** + **너무 짧게 잘리는 문제**. 설치된 모델이 `qwen2.5` 와 `nomic-embed` 둘뿐이라 한국어를 튼튼히 하려면 **프롬프트 강화** 가 가장 빠른 길.

- 시스템 프롬프트에 한국어 강제 + 중국어 금지 지시 추가
- `num_predict` 1500
- `num_ctx` 8192
- `TOP_K` 12 (retrieval 편향 다독임)

그리고 **예상치 못한 충돌** 하나. 답변 footer 에 찍히던 worklog 파일명의 `.md` 확장자가 **몰도바 국가 도메인**(`.md`) 으로 인식되어 텔레그램이 자동 링크를 걸어 버렸다. footer 출력에서 `.md` 를 제거하는 패치.

> **배운 것** — 텔레그램은 파일명을 URL 로 오판해 자동 링크화한다. 봇 출력에 `.md`, `.co`, `.io` 같은 **국가 TLD 와 겹치는 확장자** 가 있으면 링크가 의도치 않게 찍힌다. 파일명은 확장자 없이 보여 주는 쪽이 안전하다.

## 2. 새벽 — WSL 이 혼자 네 조각을 쌓았다

v1.0.0 을 찍고 나서 새벽~아침까지는 거의 전부 **윈도우 데스크탑 WSL 의 야간 자동 개발**. 맥이 잠든 동안 **혼자 굴러가 결과물 네 개** 를 만들었다.

### 2.1 한컵

물 섭취 트래커. Flutter + `shared_preferences` 만으로 구성. 한 컵 단위 기록 + 오늘치 누계. 외부 의존성 거의 없음. 저비용·저리스크로 **"완성된 바이브"** 를 빠르게 찍는 성격에 잘 맞음.

### 2.2 mini_expense

지출 기록 앱. `C:\src\flutter-factory\apps\mini_expense`. 금액 + 메모 한 줄 빠른 입력. 상단에 오늘치 합계 + 이번 달 합계. 스와이프 삭제. **445줄** 얇게 유지. `flutter analyze` + 단위 테스트 풀그린.

### 2.3 Dev Trend Curator

`~/trend-curator/curator.py`. **342줄**, 파이썬 표준 라이브러리만. 해커뉴스 상위 30건 + GitHub 최근 이틀 저장소 → 키워드 필터 → `daily/YYYY-MM-DD.md` 누적. 텔레그램 요약 한 장 발송. crontab 매일 06:50 KST.

### 2.4 flutter-factory

**공용 코드 저장소(`_lib`) + 신규 앱 스캐폴드 스크립트(`new_app.sh`)** 구조. `_lib/local_store.dart` + `_lib/simple_app.dart` 두 파일만 가져다 쓰면 `shared_preferences` 기반 얇은 앱은 바로 골격이 선다. **mini_expense 가 이 파이프라인을 처음부터 끝까지 관통한 1호 앱**.

네 조각이 쌓이면서 메모리에도 `project_hankeup` + `project_trend_curator` + `project_flutter_factory` 세 건 추가.

> **배운 것** — 기기 한 대가 자동으로 돌아가는 동안 다른 기기의 메인 세션은 그 결과를 **로컬 커밋에서만 추적 가능하다**. 통합 worklog 에 본문으로 직접 반영하지 않으면 맥 쪽 세션이 시간이 지나면 그 결과를 잊는다.

## 3. 오전 — 파이프라인·이슈·동기화를 한 번에 심다

맥 세션이 돌아와 10시 20분대까지는 거의 전부 **스킬·자동화 정비의 연속**. 앱 코드 수정은 없었고, 대신 아이폰 배포 파이프라인·이슈 기록 체계·새벽 동기화 자동화의 뼈대가 이 구간에 거의 다 올라갔다.

### 3.1 /land + /to-iphone 루프

**09:15** — 맥 종단 스킬 `/land` 심음. WSL 의 `/to-iphone` 이 쏜 Flutter 앱 URL 을 맥이 받아 clone/pull → iOS 플랫폼 세팅 → `pub get` → `pod install` → `/irun` 으로 아이폰 릴리즈 설치까지 이어 주는 역할. `daejong-page/skills.html` 에 `land` 카드도 같이 추가.

**09:17** — WSL 짝 `/to-iphone` 원샷 스킬.

**09:52** — 초기 설계의 SSH primary 경로를 걷어 내고 **텔레그램 트리거를 기본 경로로** 바꾸는 정리. 결과: WSL 에서 `/to-iphone` 한 줄 → GitHub push + 텔레그램 트리거 → 맥이 깨어나 `/land` → 아이폰 재설치. 엔드투엔드 루프가 손에 잡히는 상태.

### 3.2 claude-automations 저장소 신설

**10:03** — 맥·WSL 에 흩어져 있던 `launchd` plist 와 `systemd` unit 을 한 곳에 묶기 위해 `claude-automations` 저장소 신규 생성. 첫 입주: `context-threshold-alert`. `install.sh` 한 번으로 기기 종류에 맞게 자동 스케줄이 걸리는 구조.

**10:22** — `daily-sync-and-learn` 스크립트 입주. 매일 새벽 06:45 에 맥·WSL 양쪽에서 **각자 독립적으로** 실행되면서 `claude-skills` + `claude-automations` 를 `git pull --rebase` 로 당기고, 실패하면 경고만 남기고 계속 진행. 덤으로 skills 의 `issues/` 를 스캔해 `reference_issue_history` 메모리를 재생성하고 최근 30일 이슈 중 **무작위 3건** 을 "오늘의 복기" 로 텔레그램에 발송.

같은 10:22 에 수동 짝 `/sync` 도 추가. **호출한 기기만 pull 한다** 는 규칙 명시.

### 3.3 /issue 스킬 신설 + retro 2건

**10:15~21** — `/issue` 스킬 신설. 맥·WSL 어디서든 같은 포맷으로 `issues/YYYY-MM-DD-slug.md` 에 쌓이도록. **기기 간 이슈 히스토리가 같은 파일 구조로 공유됨**. 오늘 몫 이슈 2건을 soundwave retro 형태로 기록.

`daily-sync-and-learn` 과 맞물리는 순환이 완성됨: 이슈 파일 추가 → 다음 날 새벽 메모리 갱신 → 두 기기가 같은 학습 반복.

**10:23** — `claude-automations` gitignore 정리. 초기 커밋 때 올라간 `*.pyc` 털어내고 `__pycache__/` 패턴 ignore.

## 4. 오후 — 결과물이 공개 창구로 나가다

오전 v1.0.2 가 "개발 환경 자체를 한 단계 굳힌 오전" 이었다면, 오후 v1.0.3 은 **그 환경 위에서 결과물이 공개 창구로 흐르기 시작한 구간**.

### 4.1 /issues 공개 페이지 + retro 9건 소급

`/issue` 스킬이 저장소 내부에서는 이슈를 쌓고 있었지만 **외부에서 어떻게 보이는지가 비어 있었다**. `daejong-page` 에 `/issues` 페이지 신설 — 포스트모템 타임라인 전용 뷰.

- 각 이슈에 **심각도 뱃지 + 재발 여부 뱃지**
- 카드 구조는 **증상 · 원인 · 조치 · 재발 방지** 4섹션 고정
- 이슈가 없던 날짜엔 **빈 날짜 노드를 "없음" 상태** 로 표시 (타임라인 끊어짐 방지, `cancelled.html` 에서 이미 검증된 패턴)

맥락 채우기 위해 **4/11~어제까지 흩어져 있던 과거 이슈 9건** 을 retro 로 소급 기록. 순서: claude-skills 저장소에 원본 심기 → daejong-page 공개본으로 재이동.

같은 흐름으로 `/issue` 스킬 본체에 **14-d 단계** 추가: 이슈가 새로 기록될 때마다 스킬이 스스로 daejong-page 공개본 동기화까지 자동 처리. **이슈 저장 = 공개 페이지 갱신** 루프 완성.

### 4.2 기기 라우팅 패턴 세 스킬에 이식

`/worklog` + `/done` 스킬 두 개에 오전에 만든 `/land` 와 같은 **기기 라우팅 패턴** 이식. 0단계에서 `hostname` 으로 맥인지 확인 → 맥이 아니면 텔레그램으로 트리거 한 줄 쏘고 종료. **맥 창에서 강대종님이 직접 커맨드를 한 번 찍어야 로컬 실행 시작**. SSH 자동 실행 패턴이 아니라 **명시적 승인** 단계를 둔다.

세 스킬(`/land`, `/worklog`, `/done`) 이 같은 형태를 공유 → 앞으로 맥 종속 스킬 추가 시 이걸 템플릿처럼 복제.

### 4.3 /land 스킬에 자동화 2건 내장

오전에 메모로 남겨 둔 두 가지를 실제 구현으로 당겼다.

- **iOS signing team 자동 주입**: 한컵·mini_expense 에서 수작업으로 `CODE_SIGN_STYLE` 뒤에 `DEVELOPMENT_TEAM 46UH85U2B8` 을 `sed` 로 꽂아 넣던 것 → `/land` 스킬 본체에 내장
- **Bundle ID 자동 교정**: `com.example.<name>` → `com.ssamssae.<name>` sed 패치

두 자동화가 합쳐지면 **세 번째 앱부터는 `/land` 한 줄로 signing 도 Bundle ID 도 자동으로 제자리**. WSL `/to-iphone` → 맥 `/land` 루프가 완전 무인으로 돌아가는 상태. 이 변경은 `claude-skills` 1번 PR 로 올라가 머지 대기 중. `daejong-page` skills 페이지 land 카드 설명도 반영해 갱신.

### 4.4 바이브코딩 뉴스레터 Ep.1 발행

오후의 하이라이트. **"3시간 만에 나만의 로컬 AI 어시스턴트 만들기"** 라는 제목으로, 어제 저녁 21:30~자정의 **기억이** 제작 타임라인을 원고로 풀었다.

원고에 담은 막힘 세 지점:
1. `apt` 중간에 `Ctrl-Z` 를 눌러 프로세스가 좀비로 남았던 지점
2. systemd 유닛 파일에서 `[Service]` 헤더가 빠져 실행 단계에서 막혔던 지점
3. 봇이 중복 폴링으로 같은 메시지를 두 번 받던 지점

각 지점의 진짜 원인 + 한 줄 고침 + 교훈 3가지를 뒤에 정리. `daejong-page` 에 뉴스레터 카드와 본문이 올라가면서 **사이드 프로젝트 브리핑에 묶여 있던 아이디어 중 하나가 실제 외부 공개 결과물로 처음 나왔다**.

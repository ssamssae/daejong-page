## 1. "오늘할일" 은 항상 같은 질문이 아니다

17:45, 대종님이 `@MyClaude` 에 "오늘할일" 을 던졌다. 이전 라운드에서라면 todos 를 열어 진행중 항목을 세어 추천했을 텐데, 한 시간 전 v1.0.2 를 커밋하며 토큰 절약 스캔을 종료한 직후였다. 대화가 짧게 진행되고 18:06 에 대종님이 직접 프롬프트 초안을 썼다:

> "Claude API 비용 줄이려고 자동화 일부를 끄려고 해. 근데 내가 이미 일부 껐을 수도 있으니까 **절대 바로 끄지 말고** 먼저 현황 파악만 해줘."

오후의 스캔이 한 바퀴가 아니라 **반 바퀴** 였다는 인지. 아직 추가 여지가 남아있을 수 있다는 가설.

18:28 에 대종님이 결론냈다:

`portfolio.html` 의 `showSignup` 플래그를 지웠다. Hero 영역에서 이메일 인풋과 버튼이 사라지고, 메모요 상태 문자열이 "TestFlight / 비공개 테스트 중" — 2주 전 기준에서 한 번도 갱신 안 된 — 에서 "스토어 제출 보류 중" 으로 바뀌었다.

## 4. Claude 가 굳이 할 필요 없는 일

다음 후보는 `morning-briefing` 이었다. 07:15 에 실행되는 가장 큰 launchd 잡. 내부 흐름을 다시 봤다:

1. git log 여러 repo (커밋 수집)
2. `gh repo list` (최근 활동)
3. `osascript` 로 미리알림 조회
4. 웹 검색: 미세먼지 + 뉴스
5. 사이드 프로젝트 후보 합성
6. 텔레그램 메시지 composition

1~3번은 **순수 기계 수집**. Claude 가 할 필요가 없다. 그런데 그 결과가 Claude 세션의 입력 컨텍스트에 다 들어가고, 내가 매번 그걸 읽고 요약해서 다시 메시지로 만들었다. 이건 진짜 낭비였다.

리팩터 설계:

- **Python 프리페처 분리**: `~/.claude/automations/scripts/morning-prefetch.py` (320줄) 에 1-3번을 전부 밀어넣음. Claude 는 호출만 하고 JSON 결과만 받음.
- **Fallback 없음**: 프리페처가 실패하면 "실패 알림" 텔레그램만 보내고 끝. 실패를 Claude 가 조용히 커버하면 토큰 절약 목적이 무의미해진다.
- **claude-skills `85d18a3`**: SKILL.md -108/+63. 순수 셸 호출 블록들이 다 빠지고 Python 프리페처 호출 한 줄로 대체됨.

> **배운 것** — "이걸 Claude 에게 시키는 게 맞나" 를 물을 때 기준은 **LLM 판단이 필요한가** 하나다. git log 결과 요약·JSON 조합·osascript 반환값 파싱 전부 Python 이면 충분하다. 웹 검색·창의적 합성·자연어 composition 만 Claude 로 남겨라.

## 5. 대종님의 네 가지 질문

이 시점부터 대종님이 멀티기기 체제의 구조적 궁금증을 연이어 던졌다. 각자 짧은 Q&A:

**Q1. "푸시는 각 기기에서 따로 하잖아. 꼬이지 않을까?"**
A: 서로 다른 파일, 또는 같은 파일의 다른 섹션은 rebase 가 해결. 동일 라인 수정만 conflict. todos.md 처럼 단일 경합 파일은 이미 규칙으로 WSL 쓰기 금지(텔레그램 트리거로 맥 위임).

**Q2. "스킬 파일이 맥·윈 양쪽에 동일하면 읽을 때 두 배 로드 아냐?"**
A: 각 Claude Code 프로세스는 자기 호스트의 `~/.claude/skills/` 만 읽는다. 상대 기기 파일은 접근 불가. 동일 내용이어도 로컬 1세트만 1번 로드. 이중 카운트 없음.

**Q3. "토큰 절약하게 된 계기는 내가 요금제 결제한 지 며칠 됐지?"**
A: 청구주기·사용량은 `/usage` 스킬로 ccusage 기반 ASCII 그래프 조회 가능. 지금은 구체 수치 확인 안 함.

**Q4. "더이상 최적화할거 없다이거지?"**
A: 오늘 건드린 것 외에 눈에 띄는 과투입 후보 없음. MEMORY.md / launchd / skill description 모두 감사 완료. 다음 절감은 훅·프롬프트 엔지니어링 수준이라 기대치 낮음.

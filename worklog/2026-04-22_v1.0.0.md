## 1. 메모요는 올라가지 않았다

자정, 메모요 Play Console **beta 트랙** 업로드. `play-upload.py --track beta` 를 돌렸고, 에러가 떨어졌다.

처음엔 서명 이슈인 줄 알았다. 다음엔 aab 파일명인 줄 알았다. 한참 돌고 나서야 Google 이 친절하게 알려준 메시지를 다시 읽었다 — **testers 그룹이 구성돼 있어야 beta/alpha 트랙이 열린다**. 그냥 aab 를 올리는 게 아니라, 누가 테스트할지부터 정해 줘야 트랙이 살아 있는 것이다.

internal 트랙은 그 제약이 없다. fallback 으로 internal 에 먼저 올려 두고, beta 로 승격은 testers 구성 후로 미뤘다. 그리고 이 실수를 **절대 두 번 하지 않기 위해** lesson 을 기록했다:

```
~/.claude/skills/submit-app/lessons/android-beta-track-needs-testers.md
```

`/submit-app` 스킬은 Step 0 에서 이 lessons 폴더를 먼저 읽는다. 다음 메모요 build 업로드할 때 나는 이 교훈을 자동으로 마주하게 된다.

> **배운 것** — 스토어 트랙은 "올리면 된다" 가 아니다. **받을 사람** 이 먼저 세팅돼 있어야 한다.

## 2. 더 큰 문제가 밑에 깔려 있었다

lesson 을 쓰고 다른 repo 상태를 한 번 훑던 중에 이상한 걸 봤다. `~/apps/hanjul`, `~/apps/pomodoro`, `~/apps/mini_expense` — 셋 다 **Mac 로컬에는 커밋이 있는데 GitHub 에 push 가 안 된 상태** 였다.

한두 커밋이 아니었다. hanjul 은 4커밋이 로컬에만 쌓여 있었고, pomodoro / mini_expense 도 각각 로컬 ahead 상태였다.

이게 왜 문제였냐면 — 며칠 전부터 WSL 에서 `/sync` 로 Mac 작업을 받아오려고 할 때마다 *"받을 게 없다"* 는 응답이 왔었기 때문이다. 나는 "아, Mac 에서 별 작업 안 했나 보다" 로 넘겼다. 실제로는 **Mac 에서 작업을 했는데 push 만 안 된** 것이었다. WSL 관점에서는 GitHub 에 없으니 당연히 받을 게 없고, 동기화는 조용히 깨져 있었다.

대량 push 하기 전에, **구조를 먼저 고쳐야** 한다고 판단했다. 같은 구멍으로 또 빠질 게 뻔하니까.

## 3. 방어선 1 — Stop 훅이 `~/apps/*` 도 보게 했다

첫 방어선은 감지 계층. 세션을 끝낼 때마다 Stop 훅 `stop-check-repos-dirty.sh` 이 각 repo 의 dirty / ahead 상태를 알려주는 구조는 이미 있었다. 그런데 이 훅의 스캔 범위가 `~/.claude/automations`, `~/.claude/skills`, `~/daejong-page` 에만 한정돼 있었다. 앱 repo 들은 **훅의 시야 밖**에 있었다.

```
~/.claude/hooks/stop-check-repos-dirty.sh   →   ~/apps/*/ 추가
```

이제 Claude 세션을 닫으려 할 때 hanjul 에 커밋되지 않은 파일이나 push 되지 않은 커밋이 있으면 경고가 뜬다. 놓치는 게 줄어든다.

커밋: **automations `6df696d`**.

## 4. 방어선 2 — `/pull-apps` 라는 작은 삽을 만들었다

감지만으로는 부족했다. WSL 에서 필요할 때 "앱 코드를 지금 바로 받아오는" 버튼이 필요했다. `/sync` 가 있지만 그건 자동화·스킬·이슈 중심이라 앱 repo 는 건드리지 않았다.

그래서 앱 전용 스킬 `/pull-apps` 를 새로 만들었다:

- `~/` 최상위 + `~/apps/*/` 순회
- 각 repo 에서 `git pull --rebase --autostash`
- dirty 가 있으면 autostash 로 임시 보호 후 리베이스, 끝나면 복원

```
claude-skills 0fac33d  →  01d4c34 (scope 확장: 7개 앱)
daejong-page  ad9341c  (홈페이지 카드 추가)
```

WSL 에서 `cd ~/claude-skills && git pull` 한 번 하고 `/pull-apps` 한 번이면 맥의 앱 변경이 전부 내 데스크탑으로 내려온다.

---

## 5. 방어선 3 — `/sync` 가 앱 코드까지 먹게 했다

수동 삽이 있어도 **자동화가 없으면 사람은 까먹는다**. 세 번째 방어선은 `/sync` (= `daily-sync-and-learn.py`) 안에 앱 pull 루프를 흡수하는 것.

`/sync` 는 매일 아침 06:45 에 launchd 로 자동 실행된다. 여기에 `~/apps/*/` pull 루프가 들어가면, 앱 코드는 **내가 아무것도 안 해도** 매일 아침 최신 상태로 내려와 있게 된다.

```python
# daily-sync-and-learn.py
for app in ~/apps/*/:
    git -C app pull --rebase --autostash
```

Flutter 앱 판별 로직도 정교하게 다듬었다 — pubspec.yaml + .git 이 같이 있는 디렉토리만 잡도록.

세 repo 통합 커밋:

- **automations `9c8a208`** — daily-sync apps 루프
- **automations `783aad4`** — Flutter 앱 scan 개선
- **skills `6dc96a9`** — `/sync` 설명 갱신
- **daejong-page `b63b17c`** — 홈페이지 `/sync` 카드 desc 업데이트

> **배운 것** — 동기화 문제는 한 곳만 고치면 또 터진다. **감지(훅) + 수동 도구(스킬) + 자동화(cron)** 세 겹이 필요하다.

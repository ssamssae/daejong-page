## 1. launchd 가 자기가 죽은 걸 모른다

07시 2분, launchd 보고서가 비어 있었다. 어제(04-21) 적어둔 **launchd-silent-job-dropout** 이슈가 다시 나타난 거였다. `launchctl list` 는 job 이 등록돼 있다고 우기는데, 훅은 로드되지 않고, 실제 실행은 안 되는 그 증상.

## 2. stop-check 는 너무 많이 떠들고 있었다

세션을 닫을 때마다 stop-check 훅이 "너 dirty 한 파일 있어!" 라고 알림을 띄워줬다. 도움이 되는 경고 10% 와 **untracked 파일 노이즈 90%** 가 섞여 있었다. 제일 자주 찍히던 건 내가 방금 Write 한 임시 파일들.

두 개를 고쳤다:

1. `??` (untracked) 는 경고에서 **제외** — 진짜 잃을 수 있는 건 modified/staged 뿐.
2. `dirty-ignore.txt` 블랙리스트 지원 — 특정 repo/경로는 완전히 스킵.

커밋: **automations `6d35dcd`**. 이제 경고가 뜨면 진짜로 봐야 할 것만 뜬다.

## 3. 메모리는 이제 자기 자신을 푸시한다

`~/.claude/projects/-Users-user/memory/` 는 `ssamssae/claude-memory` private repo 에 동기화되고 있었다. 문제는 **내가 메모리에 뭘 쓸 때마다 push 를 까먹는 것**. WSL 에서 기억해야 할 사실이 맥에 안 내려와서 두 기기가 다른 과거를 들고 있는 일이 종종 있었다.

해결: **PostToolUse 훅**. Write/Edit 툴이 `memory/` 안을 건드릴 때마다 훅이 자동으로 `git pull → commit → push`. 이벤트 기반이라 Claude 프롬프트에 한 토큰도 안 들어간다.

커밋: **automations `e69e163`**.

이제 메모리는 조용히 스스로 최신 상태를 유지한다.

## 4. /handoff — 폰에서 long-press 한 번이면 되도록

크로스 디바이스 협업에서 반복되던 사용자 경험 문제: 내가 텔레그램에서 WSL Claude 에게 받은 답변 속 "맥 Claude 창에 이걸 복붙해" directive 를 선택해서 복사하는 게 매번 귀찮았다. 분석·근거 문장과 복붙 프롬프트가 한 메시지에 섞여 있으면 폰에서 long-press 로 정확한 범위를 잡기가 까다롭다.

신규 스킬 **`/handoff`** 의 규약:

- directive 전용 메시지를 **별도 Telegram reply 로** 분리해서 전송
- 복붙 단위별로 메시지를 **쪼갬** (드래그 최소화)
- 각 directive 는 **self-contained** — 상대 Claude 가 새 채팅창 첫 메시지로 받아도 돌아가게

트리거: "맥에 시켜", "WSL 로 넘겨", "handoff", "/handoff".

커밋: **skills `66fd04e`**.

## 7. hanjul 서버 — 에이전트가 건드릴 수 없는 secret 을 버렸다

22시 2분, hanjul worker 에 하드코딩돼 있던 `APP_SHARED_SECRET` 을 **통째로 드롭**했다. 이유가 흥미로웠다.

Claude Code 의 harness 정책이 *"agent 가 app secret 을 외부 URL 빌드 명령에 주입"* 플로우를 **자기일관되게 거부**한다 (이건 memory 에 이미 기록해둔 정책). settings 에 allow 를 추가해봐도 막힌다. 즉, 에이전트가 오케스트레이션 과정에서 touch 해야 하는데 touch 하면 거부되는 **교착 상태**.

답은 두 가지 중 하나였다:

1. 에이전트가 건드릴 수 없는 secret 을 계속 유지 → 수동 단계 끼워 넣기
2. **secret 을 버리고** 서버 쪽 rate limit 을 강화 → 에이전트 끝에서 끝까지 자동화

2번을 골랐다. 월간 최대 노출은 여전히 `MONTHLY_TOKEN_BUDGET` 으로 캡되고, 추가 방어로 **per-IP daily cap 60** (CF-Connecting-IP 기준) 을 얹어 X-Device-Id 로테이션 악용까지 막는다.

커밋: **hanjul `3ed99d8`** (서버+클라이언트 통합) + **`85a2afe`** (RL KV namespace id 고정).

> **배운 것** — harness 가 flow 를 거부하면 우회를 찾지 말고 **구조 자체를 바꾸는 게** 더 깔끔하다. 제약이 설계를 좋게 만들 때가 있다.

| 토큰 | before | after |
|---|---|---|
| bg | `#121212` / `#1a1a1a` | `#FFFFFF` / `#F9FAFB` |
| accent | `#ffd54f` (yellow) | `#3182F6` (Toss blue) |
| text | light-grays-on-dark | `#191F28`/`#4E5968`/`#8B95A1` on light |

done.html 의 히트맵 레벨도 재정렬 — lv4 가 가장 deep 하게 찍히도록.

그림자가 특히 중요했다. `.4` 그림자가 화이트 bg 에선 그냥 탁해 보였다. `.08` 로 내려야 비로소 "가볍게 뜬" 느낌이 난다. 다크 테마의 그림자는 빛을 차단하는 거고, 라이트 테마의 그림자는 공기를 분리하는 거라서, 같은 숫자를 쓰면 안 된다.

> **배운 것** — 리디자인은 팔레트 바꾸는 게 아니라 shadow / font / tag color 까지 **연쇄적으로 재측정**하는 일이다.

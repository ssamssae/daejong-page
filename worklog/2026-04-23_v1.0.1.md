## 4. iOS 가 읽지 못한 한 줄

시뮬레이터(iPhone 17, iOS 26.2) 부팅 → `flutter run` → 첫 벽. `flutter` 가 PATH 에 없었다. `fvm` 으로 관리 중이었고, `~/fvm/versions/stable/bin/flutter` 는 **3.41.6(Dart 3.11.4)** 인데 `pubspec.yaml` 이 `sdk: ^3.11.5` 를 요구해 pub get 이 거절. 같은 fvm 캐시에 **3.41.7(Dart 3.11.5+)** 이 이미 설치돼 있어 그 경로를 직접 지정해 우회했다.

Xcode 빌드 33초. 앱 설치 성공. 시뮬 홈에 "Review Radar" 아이콘이 떴다. 그런데 탭하자 **첫 프레임도 못 뜨고 사라졌다**. 콘솔에 SIGABRT 가 찍히며 Mac 의 Apple 크래시 리포트가 떠오른다.

```
*** Terminating app due to uncaught exception 'NSInvalidArgumentException',
    reason: 'No active configuration. Make sure GIDClientID is set in Info.plist.'
```

스택 최상단은 Google Sign-In 라이브러리. 앱이 뜨자마자 `HomeScreen.initState` 가 `controller.refresh()` 를 걸고, 그 안에서 `RealGmailService._ensureSignedIn()` 이 **앱 첫 프레임보다 먼저** `GoogleSignIn.signIn()` 을 호출하고 있었다. iOS 는 그 시점에 `Info.plist` 의 `GIDClientID` 키로 OAuth 설정을 찾는데, **그 키는 파일에 없었다**.

`real_gmail_service.dart` 의 주석이 이유를 자백하고 있었다.

> *Android 는 package name + SHA-1 매칭이라 Client ID 코드에 넣을 필요 없음.*

이 구현은 처음부터 Android 만 상상하고 있었다. iOS 에서의 실행 경로는 런타임까지 내려와 본 적이 없어서, 오늘 시뮬에 얹히는 순간 즉시 드러났다.

> **배운 것** — 크로스플랫폼 Flutter 앱이라 해도, OAuth 같이 **플랫폼이 네이티브 자원을 직접 뒤지는** 경로는 각 플랫폼에서 한 번씩 실제로 돌려보기 전엔 "구현됐다" 고 말할 수 없다. 한쪽만 검증된 상태는 "반쯤 구현된 상태" 다.

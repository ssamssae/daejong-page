# 2026.04.26 작업일지 v1.0.0 — 70만원 인프라 사려다 0원으로 끝낸 1시간, 그리고 책상에 합류한 다섯 번째 머신

> 13:00, 데스크탑·노트북 두 대를 LLM 인프라에 합류시켰다.
> 17:00, "24/7 자동화 허브 필요해" 라파이파이 5 풀세트 40만원 견적이 떴다.
> 17:25, "본가에 놀고 있는 M1 맥미니 16GB 있는데" 한 마디로 0원 대체로 결정 전환.
> 23:54, 맥미니가 책상 위에서 24/7 자동화 허브로 가동 시작.

오늘은 인프라를 두 번 결정했다. 먼저 큰 인프라 — 데스크탑·노트북 LLM 노드 두 대를 정식으로 합류시켜 4-노드 그림을 완성했다. 그리고 같은 날 24/7 허브가 부족하다는 결론에 도달해서 라파5를 사기로 했고, 한 시간도 안 돼서 그 결정을 뒤집고 본가에 잠자던 M1 맥미니로 갈아끼웠다. 70만원 살뻔했던 게 0원이 됐고, 같은 시간 옆자리의 두 Claude 세션은 둘 다 처음에 같은 함정에 빠져있었다는 걸 자각했다. 그리고 이 모든 흐름이 워크플로우 매핑 한 번 안 한 채로 진행됐다는 걸 인정하면서 운영 정책을 v2 (4-node on-demand) → v3 (2-node 슬림화) 까지 두 번 갈아끼웠다.

## 1. 13:00, 4-노드를 박았다

데스크탑(RTX 3060 Ti 8GB, 75 tok/s) + 노트북(RTX 3060 Laptop 6GB, 22 tok/s) 두 대를 Tailscale 합류시켰다. 양쪽 다 Windows 네이티브 ollama + llama3.1:8b-instruct-q4_K_M (4.92GB) 풀로드. WSL 본진(2070S) + Mac 본진(맥북프로) 까지 합쳐서 4-노드 LLM 인프라 그림이 떴다. 본진에서 던지는 curl 한 번이 호출 단위.

맥북은 노트북이고 WSL 데스크탑은 게임용도 겸한다. trend-curator·심사레이더·메모리 동기화 같은 잡이 새벽에도 무인으로 돌려면 항상 켜진 머신이 따로 필요하다는 결론. 라즈베리파이 5 8GB + Argon ONE V5 + NVMe 256GB + 27W 어댑터 = 풀세트 약 40만원으로 견적이 잡혔다.

## 3. 17:25, "본가에 놀고 있는 M1 맥미니 16GB 있는데"

강대종님이 한 마디 던졌다. M1 맥미니 16GB. 진접 본가에 놀고 있던 거. 라파5 보다 모든 면에서 우월하다 — Apple Silicon ARM Linux 가 아니라 본진 Mac 과 같은 macOS, PATH 도 같음, brew 도 같음, launchd plist 그대로 이전 가능, M1 Metal 로 LLM 5번째 노드 옵션. 셋업 시간 30분으로 압축. 그리고 0원.

> **배운 것** — 사기 전에 집 한 번 둘러보기. 같은 시간 옆자리의 WSL Claude 도 처음에 "라파5 사세요"부터 시작했다. AI 어시스턴트 두 대가 동시에 같은 함정에 빠진 메타가 Ep.3 의 핵심 메시지로 박혔다. 이 자각이 `feedback_check_workflow_before_infra.md` 룰로 정립.

todos 의 라파5 라인은 보류·취소 섹션으로 이동, 라파5 자리에 "M1 맥미니 자동화 허브 셋업" 이 최우선 과제로 들어갔다 (904cc84). 강대종님은 진접으로 픽업 출발.

## 4. 23:46, Ghostty 가 거절한 명령

진접에서 픽업 후 공덕 도착, macOS Tahoe 26.4.1 셋업 마치고 책상에 합류. Tailscale `.pkg` 깔고 Admin 콘솔에서 hostname 을 `mac-mini-macmini` → `mac-mini` 로 정리 (Tailscale 이 OS hostname 에 자동 suffix 붙이는 함정). 본진 Mac 에서 ping 12ms direct.

OS hostname + SSH 서버 + 본진 공개키 등록을 한 번에 묶어서 Ghostty 에 paste:

```bash
sudo scutil --set HostName mac-mini
sudo scutil --set LocalHostName mac-mini
sudo scutil --set ComputerName mac-mini
sudo systemsetup -setremotelogin on    # ← 여기서 거절
mkdir -p ~/.ssh && chmod 700 ~/.ssh
echo 'ssh-ed25519 ... user@MacBookPro' >> ~/.ssh/authorized_keys
chmod 600 ~/.ssh/authorized_keys
```

`systemsetup -setremotelogin on` 이 이렇게 답했다:

```
setremotelogin: Turning Remote Login on or off requires Full Disk Access privileges.
```

macOS 13+ 부터 강화된 권한 모델 — 시스템 변경 명령 일부가 호출 터미널 앱에 FDA 권한 요구. Ghostty 는 새로 설치한 터미널이라 FDA 미부여. sudo 비번 통과해도 권한 부족.

해결은 단계 적은 쪽으로: 시스템 설정 > 일반 > 공유 > **원격 로그인** GUI 토글 ON. SSH 데몬 정상 활성화. 이게 이번 골치 아픈 발견 — 이슈로 박았고 (`tahoe-ssh-cli-block`) 메모리에도 reference 로 박았다 (`reference_macos_setremotelogin_gui_only.md`). 다음 헤드리스 머신 셋업 시 자동 참조됨.

> **배운 것** — macOS 26.x 헤드리스 SSH 활성화 = GUI 토글 only. CLI 시도 금지. FDA 부여 절차 추가하느라 GUI 보다 시간 더 든다.

## 5. 23:54, 다섯 번째 머신이 24/7 허브로 가동

본진 Mac 에서 `ssh mac-mini` 무인증 통과. WSL 본진에서도 같은 키 등록 통과. brew + git + python@3.12 + jq + wget + node 설치. 절전·디스크 sleep·재부팅 자동·Wake on LAN·Power Nap·TCP keepalive 6설정을 `pmset -a` 한 줄로 박고, 자동 로그인 GUI 마지막. 검증:

```
sleep         0     ← 시스템 절대 sleep 안 함
disksleep     0     ← 디스크도 sleep 안 함
womp          1     ← Wake on Magic Packet
autorestart   1     ← 정전 후 자동 재시작
powernap      1
tcpkeepalive  1
displaysleep  10    ← 디스플레이만 10분 후 꺼짐
```

자정이 막 지나서 23:54, 화면은 꺼져도 머신은 안 자는 상태. 강대종님이 자고 있는 새벽 3시에도 launchd 잡이 정상 동작하는 자동화 허브가 책상 위에 떴다.

## 6. 04-27 00:30, 4-노드 → 2-노드 슬림화

운영 정책 v2 가 자정 직전에 박혔다. "24/7 ON 은 맥미니만, 데스크탑·노트북은 평소 OFF on-demand." 그런데 새벽 04-27 00:30, WSL 본진이 day-1 활용도를 보고 "사실 노트북도 거의 잉여" 라고 회신. v3 정책으로 더 슬림화:

| 노드 | 평소 | 켜는 시점 |
|---|---|---|
| 🤖 M1 맥미니 16GB | 24/7 ON | 항상 |
| 🪟 WSL 본진 | dual-purpose | 평소 사용 |
| 💻 데스크탑 (3060 Ti) | OFF | 무거운 LLM 필요 시 수동 |
| 📓 노트북 (3060 Laptop) | OFF | 거의 X |

trend-curator routing 도 v2 로 갈아끼웠다. 노트북 SSH hop 제거 → WSL 본진 localhost 실행. 파일명은 `run-trend-curator-on-notebook.sh` 그대로 유지 (Mac launchd plist 호환). claude-automations `7f0daa0` push 완료.

검증 kickstart 1회 돌렸더니 `[curator] summarizing 5 items via http://100.70.173.66:11434... → ollama error: urlopen error timed out`. `curator.py` 가 데스크탑 IP 박혀있는데 v3 정책으로 OFF 라 timeout. WSL 별 세션에서 OLLAMA_URL 갱신 또는 Phase 2 비활성으로 처리 예정.

> **배운 것** — 4-노드 박은 다음날 day-1 활용도 0 으로 자각. 인프라 박기 전 워크플로우 매핑 한 번 안 했다는 게 명확해졌다. AI 인프라 = 인프라가 인프라를 위한 인프라가 되는 sunk cost 함정.

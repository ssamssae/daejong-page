오늘은 출시되어 있는 앱과 출시되지 않은 앱이 같은 하루 안에서 다른 단계를 통과한 날이었다. 약먹자·더치페이는 새벽에 조용히 1.0.1/1.0.2 승인이 떨어진 채로 깨어났고, 한줄일기는 23시를 넘기며 1900원짜리 1.0 으로 Apple 심사 큐에 들어갔다. 그 사이 자동화 셋 — morning-briefing/evening-wrap, 심사레이더, 가계부 — 을 일관된 기준으로 끊어냈다. 운영 7일 검증해보고 안 쓰는 건 안 쓰는 거였다.

## 1. 새벽에 도착한 두 통의 승인 메일

밤사이 ASC 가 처리해 둔 약먹자 1.0.1, 더치페이 1.0.2 가 READY_FOR_SALE 로 떨어져 있었다. 어제 새벽까지 reject 답변·재제출로 끌어다 놓은 두 앱이 사용자 행동 한 번 없이 출시 상태로 전환된 셈이다. 메인 페이지 portfolio 카드의 상태 라벨을 "출시중"으로 박아두는 작은 작업으로 시작.

> **배운 것** — Apple 심사 큐는 깜박 잊고 깨어나면 끝나 있는 종류의 큐다. 매시간 새로고침하면 손해다.

## 2. "안 쓰는 자동화는 자동화가 아니다"

오전 시간엔 자동화 정리. 4월 21일 launchd 비활성화 후 7일을 살아본 결과:

- **morning-briefing / evening-wrap**: 자동 발사 끄고 나니 수동 슬래시 호출도 0회. 폐기.
- **심사레이더 (review_radar)**: 출시 라이브러리에서 빼고, 내가 더 이상 쓰지 않는다는 사실 인정. 폐기.
- **가계부 (mini_expense)**: 자기쓰기·출시 둘 다 중단. 폐기.

세 건 모두 같은 패턴이었다. *작동은 한다, 다만 아무도 안 쓴다.* 작동 가능성이 사용 가치가 아니라는 걸 매번 새로 배운다.

```
claude-skills:    af32726 drop(skills): review-status-check
claude-skills:    b6abb1c handoff(wsl→mac): 심사레이더 + review-status-check 일괄 폐기
claude-automations: 47b150a drop: morning-prefetch.py
claude-automations: 7e34aa7 docs: 폐기 표기
daejong-page:     94467b7 portfolio: 심사레이더 카드 제거
```

> **배운 것** — 자동화는 7일 무사용이면 박제 후 끄는 게 정상. "혹시 나중에 쓸까봐"는 비용을 늘릴 뿐이다.

## 6. iPhone 시뮬레이터 안에 일기 5개 박는 법

심사 제출 마지막 단계 — stale 스크린샷 교체 + iPad 13" 신규 슬롯 + 콘텐츠 권한 + 연령 등급. iPhone 16 Pro Max sim + iPad Pro 13" M5 sim 양쪽에 1.1.0+4 빌드를 깔고 home 에 일기 5개를 시드해야 했다.

처음에 막힌 건 PlistBuddy. JSON 문자열을 plist 에 넣으려고 했더니 PlistBuddy 가 따옴표를 다 떼고 들어가서 — `{"id":"123"}` 가 `{id:123}` 로 망가져버렸다. shared_preferences 가 JSON.parse 못 하니 빈 상태로 부팅.

해결 단계: Python `plistlib` 로 직접 binary plist 작성 → cfprefsd 캐시 우회 (`launchctl kickstart -k system/com.apple.cfprefsd.xpc.daemon`) → `simctl spawn defaults write` 로 한 번 더 백업. 디스크와 캐시가 동기화돼야 다음 launch 가 fresh JSON 을 읽었다.

```python
import plistlib
seed = open('/tmp/hanjul_seed.json').read().strip()
with open(plist_path, 'rb') as f:
    d = plistlib.load(f)
d['flutter.hanjul_entries_v1'] = seed  # 원본 JSON 그대로
with open(plist_path, 'wb') as f:
    plistlib.dump(d, f, fmt=plistlib.FMT_BINARY)
```

탭 전환은 UI 자동화 안 쓰고 컴파일 옵션으로 — `flutter build ios --simulator --debug --dart-define=HANJUL_START_TAB=1`. root_shell.dart 의 `int.fromEnvironment('HANJUL_START_TAB', defaultValue: 0)` 가 stats 탭 직접 진입을 deterministic 하게 만든다. 클릭 자동화 회피.

> **배운 것** — shared_preferences 시드는 디스크 plist + cfprefsd 캐시 양쪽을 같이 동기화해야 한다. 어느 쪽만 쓰면 다음 launch 에서 stale 데이터가 보인다.

## 7. ASC inflight 의 미세 함정들

Playwright 로 inflight 페이지를 자동 조작하면서 발견한 것들:

**미디어 매니저 동기화 모달**: 첫 업로드 직후 "이 디스플레이 크기 + 모든 한국어 앱의 미리보기/스크린샷에 사용" 확인창. iPhone, iPad 각각 한 번씩 떠서 두 번 승인.

**연령 등급 7단계**: 1단계 6개 항목(parental controls, age assurance, unrestricted web access, user-generated content, messaging, ads) 모두 No. 2~6단계 콘텐츠 빈도(성적 테마, 의료, 노출, 폭력, 도박) 모두 None/Never. 결과 4+ 자동 산출.

**첨부파일 어트리뷰트 누락 버그**: navigate 한 번 하면 이전에 올린 첨부 영상이 슬롯에서 빠진다. 동일 세션에서 한 번 더 업로드해야 살아있음.

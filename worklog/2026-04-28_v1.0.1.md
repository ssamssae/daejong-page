## 1. WSL 이 보낸 directive — 비대칭의 적용

22:22 KST 에 도착한 메시지는 어제(2026-04-28 새벽~낮) WSL 작업자가 `globals/CLAUDE.md`, `globals/AGENT.md`, `handoff/SKILL.md` 에 박은 "지휘관 1명 원칙" 변경분을 Mac 본진에서 점검하고 적용하라는 directive 였다. 핵심:

> Mac 세션 = 지휘관 / WSL 세션 = 작업자 / Mac mini = 24/7 자동 실행 노드.
> Mac→WSL = 작업 지시 directive, WSL→Mac = 결과 보고 report. 채널은 양방향이지만 컨텐츠는 비대칭.

상태 점검은 빠르게 끝났다. ~/claude-skills 가 origin 보다 5 commits behind, ~/.claude/skills 는 ~/claude-skills 의 심볼릭이라 한 번에 갱신, memory repo 는 auto-push hook 으로 이미 동기화. ~/.claude/CLAUDE.md / AGENT.md 는 globals/* 의 심볼릭이라 fast-forward pull 한 번으로 hard rule 즉시 발효.

원칙을 박을 때 늘 따라오는 건 boundary case 다. 점검 결과 세 후보 중 두 개는 보강 불필요로 드롭됐고 하나만 한 줄 추가됐다.

|---|---|---|
| a) handoff L122 reply 예시 | 드롭 | 이미 "잘못된 예 ❌" 블록 안 anti-example. 보강 불필요 |

| c) memory feedback 의 "양방향" 키워드 | 드롭 | 정의 자리(handoff/SKILL.md "방향성" 절) 따로 있음. 메모리는 행동 강제 룰 |

23:30 KST 에 commit `a00825b` 로 한 줄씩 두 파일에 추가:

> 예외 (운반체): 사용자 명령을 자동화로 라우팅하는 트리거(/to-iphone → /land 페어 등 WSL 자동화→Mac 자동화)는 directive 가 아닌 운반체. WSL 세션이 자체 판단으로 새 방향을 결정한 게 아니면 새 원칙 위반 아님.

> **배운 것** — Hard rule 도입 직후엔 그 룰이 닿지 않는 가장자리를 의식적으로 점검해야 한다. 안 그러면 회귀 위험은 안 보이는 자리에 숨는다.

## 3. R1 — 회귀 직접 위험을 잡다

문제는 마지막 단어. 새 원칙에 따르면 WSL→Mac 은 directive 가 아니라 report 다. 다음 WSL Claude 가 핑을 보낼 때 SKILL.md 를 그대로 따라가면 즉시 룰 위반이 박힌다. R1 패치는 그 한 단어를 바꾸는 거였다:


## 4. /night-runner — 코드 수정기에서 점검기로

자정 직후 두 번째 directive 가 도착했다. 이번엔 다른 방향의 작업 — 야간 사이드 프로젝트 러너를 다시 그리자는 거였다.

기존 v0 (WSL 측 `~/.claude/runner/run.sh`) 은 위험도가 높았다.

| v0 위험 요소 |
|---|
| `--dangerously-skip-permissions` 권한 우회 |
| `claude -p --agent repo-janitor` 자율 코드 수정 |
| `git push` + `gh pr create` 자동 PR 생성 |
| timeout 7200 + 예산 $5 |
| 사람 review 없는 야간 자동 실행 |

v1 안전모드의 결정은 단순했다. 위험 요소 전부 제거. 점검 5개 (BACKLOG/TODO grep, TODO·FIXME·HACK, 7일 commit silence, test, lint) 만 read-only 로 돌리고 markdown 보고서 + 텔레그램 1통. claude headless 0, commit/push/PR 0, 코드 수정 0. timeout 1200, 실패 재시도 0.

> **배운 것** — "안전한 자동화" 의 본질은 자동 실행 빈도가 아니라 한 번의 실행이 무엇을 바꿀 수 있는지의 범위다. 그 범위가 "보고서 1개" 면 24/7 도 안전하고, "PR 자동 생성" 이면 1주일에 한 번도 위험하다.

## 5. bash 3.2, timeout, pyyaml — Mac 기본의 함정

스크립트 작성 후 첫 실행에서 세 번 막혔다.

```
mapfile: command not found
ERROR: pyyaml 미설치
timeout: command not found
```

세 줄 모두 같은 패턴 — Mac 기본 환경에 GNU/Linux 가정이 없다. bash 3.2 가 아직 기본이라 `mapfile` 이 없고, `timeout` 도 없고 (`brew install coreutils` 의 `gtimeout`), python3 stdlib 에 yaml 도 없다. 세 줄 모두 portable 패치로 해결:

- `mapfile -t SELECTED < <(...)` → `while IFS= read -r line; do SELECTED+=("$line"); done < <(...)`
- `timeout` → `gtimeout` 자동 detect, 둘 다 없으면 `perl -e 'alarm shift; exec @ARGV' SECONDS COMMAND...`
- `import yaml` → `re.match` 로 stdlib 정규식 파싱

> **배운 것** — Mac 기본 환경은 의외로 좁다. 자동화 스크립트가 Mac → Mac mini → 다른 macOS 로 옮겨갈 거면 brew 설치 의존성을 가정하지 말고 stdlib + perl fallback 으로 갈 것.

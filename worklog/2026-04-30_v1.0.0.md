# 2026.04.30 작업일지 v1.0.0 — 한줄일기 1.0 App Store 진입

> 09:32 KST, Apple 메일이 왔다. "There's an issue with your submission." 어제 23:05 KST 에 큐에 들어간 한줄일기 1.0 이 `UNRESOLVED_ISSUES` 상태로 막혀있었다.
> 11:06 KST, ASC API 우회 경로로 재제출 큐에 다시 진입.
> 13:29 KST, "Review of your 한줄일기 - AI 한줄응원 (iOS) submission is complete." 턴어라운드 2시간 23분.

오늘은 "막혔던 게 풀린 날" 이었다. 어제 저녁 도착해서 정체불명이던 Apple 리젝 메일의 사유가 오전에 명확해지고, 그 사유에 맞는 우회 경로를 ASC API 안에서 찾아내고, 재심사 큐에 다시 들어갔고, 이례적으로 빠른 두 시간 반 만에 승인이 떨어졌다. 같은 날 Mac mini 자동배포 인프라 Phase B 가 풀그린에 도달했고, 한줄일기 다음 큐인 단어요(wordyo) 의 부트스트랩 commit 도 들어갔다. 한 줄로 줄이면: **한줄일기 iOS 1.0 가 App Store 에 정식 입성한 날**.

## 1. 리젝 메일 한 줄, 사유는 ASC 안쪽에

Apple 의 리젝 메일 본문은 거짓말처럼 짧았다.

> "There's an issue with your 한줄일기 (iOS) submission."

여기서 끝. 사유는 본문 어디에도 없다. ASC 의 Resolution Center 라는 별도 영역에 들어가야 보인다. 들어가니 줄거리는 이랬다.

한줄일기의 핵심 기능 — 사용자가 적은 한 줄에 AI 가 한 줄 응원을 돌려주는 것 — 이 중국 본토에서는 "Administrative Provisions on Deep Synthesis of Internet-based Information Services" 의 적용 대상이다. Apple 가이드라인 5 — Legal. 두 가지 선택지 중 하나: 중국 전문 자문을 받거나, 중국 출시국에서 빼거나.

> **배운 것** — Apple 리젝 메일은 도착 알림일 뿐, 진짜 사유는 ASC Resolution Center 에 있다. 다음부터는 메일 보고 ASC 부터 연다.

답은 분명했다. 어차피 1.0 은 한국 단일 가격(₩1,900 KOR) 으로만 출시할 거였고, 중국 본토는 처음부터 메인 시나리오가 아니었다. Resolution Center 답글로 "We have removed China mainland from this app's availability per Guideline 5 - Legal feedback. Please re-review." 한 줄 발송. 4/29 08:05 KST.

이걸로 끝인 줄 알았다. 아니었다.

## 2. Resolution Center 답글로는 큐에 다시 안 들어간다

26시간이 지나도 reviewSubmission state 는 여전히 `UNRESOLVED_ISSUES` 였다. Apple 측에서 매뉴얼로 리뷰 큐에 다시 넣어 주지 않으면, 답글만 보내고 끝나 있다는 뜻이다.

ASC 안쪽에는 자동으로 새 reviewSubmission(`e58db9c9`) 한 개가 추가로 만들어져 있었지만 이쪽은 빈 껍데기였다. items=0. 어제 세션에서 막힌 이유가 여기였을 거다.

ASC API 로 우회 경로를 찾기 시작했다. 시도 → 응답 → 다음 시도, 세 번 만에 길이 보였다.

```
1. POST 새 sub 에 appStoreVersion attach
   → 409 ITEM_PART_OF_ANOTHER_SUBMISSION   (옛 sub 가 점유 중)
2. DELETE 옛 reviewSubmissionItem
   → 409 Item was already submitted        (제출된 item 잠김)
3. PATCH 옛 sub canceled=true
   → state CANCELING → COMPLETE             (~30s 폴링)
4. POST 새 sub 에 attach
   → 201 OK
5. PATCH 새 sub submitted=true
   → 200 OK, state = WAITING_FOR_REVIEW
```

3번이 핵심이었다. 옛 sub 의 item 을 직접 지우는 길은 막혀있지만, 옛 sub 자체를 cancel 시키면 item 잠금이 같이 풀린다. 11:06:57 KST, `submitted=true`, `state=WAITING_FOR_REVIEW`. 다시 큐.

## 3. 두 시간 반 만에 도착한 승인

평소 같으면 24~48시간을 기다리는 큐다. 이번엔 **2시간 23분**.

> 13:29 KST. 메일이 왔다. "Review of your 한줄일기 - AI 한줄응원 (iOS) submission is complete. Congratulations!"

ASC API 로 즉시 검증.

| 필드 | 값 |
|---|---|
| `appStoreState` | `READY_FOR_SALE` |
| `appVersionState` | `READY_FOR_DISTRIBUTION` |
| `releaseType` | `AFTER_APPROVAL` |
| `reviewSubmission e58db9c9` | `COMPLETE` |
| `customerPrice` (KOR) | `1900` (proceeds 1468) |

`releaseType = AFTER_APPROVAL` 설정 덕에 승인 즉시 자동으로 출시 처리. 강대종이 따로 release 버튼을 누를 필요가 없다는 뜻이다. App URL: https://apps.apple.com/kr/app/id6764308678.

## 4. 그런데 검색에서 안 나온다 — Apple CDN 은 9시간째 잠잠

22:21 KST. "출시 가능한것 같은데 알아봐줘" 라는 메시지가 왔다. 강대종이 보기엔 아직 App Store 에서 한줄일기가 안 잡히는 상태. 확인.

ASC 모바일 앱은 한줄일기 1.0 옆에 초록 체크 ✓ 가 떠 있었다. 메모요 1.0.4, 약먹자 1.0.1, 더치페이 1.0.2 — 이미 LIVE 상태인 세 앱과 똑같은 표식. Apple 시스템 안쪽에선 4번째 LIVE 앱으로 자리잡은 상태.

그런데 외부 채널은 잠잠했다.

```
$ curl -I https://apps.apple.com/kr/app/id6764308678
HTTP/2 404

$ curl https://itunes.apple.com/lookup?id=6764308678&country=kr
{"resultCount": 0, "results": []}
```

KR/US/JP/GB/DE 다섯 storefront 모두 0. `READY_FOR_SALE` 인지 9시간이 지났는데 CDN/검색 색인이 아직 따라잡지 못한 상태.

> **배운 것** — Apple 의 "내부 LIVE" 와 "외부 LIVE" 사이엔 propagation 지연이 있다. 보통 1~6h, 가끔 24h 까지 간다. ASC API 로 `READY_FOR_SALE` 만 확인되면 출시 자체는 완료. 검색·웹 페이지가 안 보여도 Apple 측 잠시 늦는 것이고 강제 트리거는 없다.

자정~새벽 사이에 자연 정상화된다. 친구한테 공유할 링크는 그때까지 묵혀둔다.

## 5. 같은 날, Mac mini SoT 자동배포 Phase B 풀그린

오후엔 자동배포 인프라가 한 칸 전진했다. iOS App Store Connect API key 와 Android Play 서비스 계정을 Mac mini 쪽으로 옮겨심고, 권한 부여까지 마무리.

| 영역 | 키 위치 | 검증 |
|---|---|---|
| iOS | `~/.claude/secrets/AuthKey_RU7URQ5453.p8` | fastlane → ConnectAPI 응답 도달 PASS |
| Android | `~/.claude/secrets/play-service-account.json` | `play-upload.py --check hanjul --package com.daejongkang.hanjul` → `REGISTERED` |

검증을 확장해서 한줄일기 Android 1.1.0+5 AAB 를 Mac mini 가 빌드하고 alpha 트랙에 promote 까지 자동 처리. SoT 인프라의 첫 실사용. commit `6b9551c`.

이 흐름의 한 줄 요약: **사람이 누르는 영역(메타데이터·스크린샷·심사제출·앱 만들기)** 은 본진(MacBook Pro) 에서, **자동화 가능한 영역(빌드·서명·업로드·트랙 commit)** 은 Mac mini 에서. SSH 트리거 한 줄로 Mac mini 가 알아서 굴러간다.

## 3. parallel-cycle 의 탄생: 11:55

오전 작업들을 정리하다 보니 Mac과 WSL이 각자 따로 돌아가는 걸 체계화할 필요가 있었다. 11:55에 첫 버전이 나왔다. Mac 3개 + WSL 3개 작업을 병렬 실행하고 session-close를 자동 호출하는 골격.

그런데 3+3 고정은 바로 문제가 됐다. 실제로는 작업 개수가 매번 달랐다. 그래서 v0.2(15:24) 에서 확인 단계를 제거하고 todos 자동 선택을 추가했다. v0.3(15:25) 에서 WSL session-close 자동화. v0.4(19:13) 에서 **M+W 가변 지원**(3+3 고정 폐기). v0.5(20:18) 에서 todos+parking-lot 동시 병합.

## 5. 세 번의 irun, 그리고 좀비

PR #14 코드를 실기기에서 확인하려면 irun 이 필요했다. 총 세 번을 시도했다.

**1차**: 통계 화면 없음은 확인했는데 카지노 애니메이션도 없었다. 이유: PR #14 가 PR #13 머지 전 브랜치를 기반으로 만들어졌다. `git rebase main` 후 force push.

**2차**: 앱은 켜졌는데 Dart VM attach 실패. "Installing and launching..." 에서 멈추고, 60초 타임아웃 후 흰 화면.

이때 5/4 이슈 파일에 적어뒀던 가설이 떠올랐다 — "iOS 26 + Flutter 3.41.9 Metal 렌더링 호환 문제." 하지만 이번엔 다른 방향으로 의심했다. mac-mini 에 이전 flutter run 프로세스가 남아 있지 않을까?

```bash
ssh mac-mini "pkill -f 'flutter run.*lottocalc'; pkill -f 'iproxy'"
```

**3차**: Dart VM 연결 성공. 카지노 슬롯 애니메이션이 아이폰 화면에서 감속하며 정착했다. 1게임~5게임 레이블도 확인.

좀비 프로세스가 범인이었다. iOS 26 렌더링 문제가 아니었다. 이슈 파일을 resolved 로 닫았다.

> **배운 것** — irun 재시도 전에 mac-mini 에서 이전 flutter/iproxy 프로세스를 먼저 kill 하는 것이 예방책이다. 의심부터 하고 재시도해야 한다.

## 1. 슬롯머신이 iPhone에서 돌아갔다

자정 직후, WSL이 밤새 처리한 PR 3개가 도착해 있었다. iOS 흰화면 버그 수정(#11), 레이블 텍스트 개선(#12), 시드 데이터 자동 업데이트(#10). Squash 머지하고, "번호 뽑기가 너무 심심하다"는 피드백을 반영해 카지노 슬롯머신 애니메이션을 추가했다.

번호가 고속으로 롤링하다가 감속하면서 최종 숫자에 정착하는 방식이었다. ScaleIn 효과, 색상 변화, 햅틱까지. /irun으로 iPhone에 올려서 직접 확인했다.

> **배운 것** — 기능 검증보다 감각 검증이 먼저 필요한 순간이 있다. "통계 맞아?" 질문이 나온 건 화면을 직접 보고 나서였다.

"통계 검증됐냐?" — 1회부터 1222회(2026-05-02)까지 데이터가 있고, 로직도 6가지 검증을 통과했다. 그런데 돌아온 말은 "제대로 운영 못할 거면 빼는 게 맞다"였다. 통계 탭은 사라졌다.

## 2. Apple이 다시 문을 두드렸다

오전 10시 32분. ASC 메일함에 리젝 메일이 있었다. Guideline 2.3.6.

첫 번째 리젝(어제)은 Age Rating "Gambling and Contests"를 Yes로 설정하라는 요구였다. Yes로 바꿔서 재제출했는데, 오늘 또 왔다.

ASC Playwright로 Age Rating 수정해서 두 번째 재제출 (제출 ID: c079dc78). 그런데 WSL이 더 나은 방향을 제시했다.

**WSL 자문 결과:** 로또 AI매니저, 복똑방, 로또라이브 — 국내 동일 기능 앱들이 생산성·엔터테인먼트 카테고리로 Gambling 없이 판매 중이다. Apple 2.3.6의 핵심어는 "facilitate gambling"인데, 번호 생성기는 도박을 촉진하지 않는다. 성공률 50~60%.

Gambling=Yes 심사를 취소하고, Gambling=No + 반론 메모로 다시 제출했다 (제출 ID: 0358b1ac). 심사 메모에는 경쟁 앱 3개의 카테고리와 번들 ID를 포함했다.

> **배운 것** — Apple 리젝에는 두 가지 대응이 있다. 요구대로 맞추거나, 요구 자체에 반론을 제기하거나. 후자가 더 오래 걸리지만 앱의 정체성을 지킨다.

## 3. 이름에서 '로또'를 지웠다

오후 3시 8분, 대비책이 필요했다. 반론이 기각되면 어떻게 할 것인가.

결론: 앱을 통째로 재포장한다.

- 앱 이름: **로또번호 생성기 → 행운번호 생성기**
- 패키지: `com.daejongkang.lottocalc` → `com.daejongkang.randompick`
- 슬롯머신 애니메이션 제거 (카지노 연상 우려)
- 아이콘 교체 + 버전 1.1.0+3

`e5ac8b0 refactor: lottocalc → randompick 재포장 (2.3.6 거절 대응)`

반론이 통과되면 원래 이름으로 돌아가는 옵션이 있다. 지금은 두 트랙을 동시에 진행 중이다.

## 4. 코레일은 세 번 막았다

오후 2시 52분부터 5시까지, 코레일 자동예매 봇 작업이 이어졌다.

| 시도 | 방법 | 결과 |
|------|------|------|
| v1 | headless=False (headed 모드) | 탐지됨 |
| v2 | Device=BH→WB 변환 | 탐지됨 |
| v3 | playwright-stealth + 랜덤 딜레이 | 탐지됨 |

PerimeterX(현 HUMAN Security): 브라우저 핑거프린트 + 행동 패턴(클릭 간격, 스크롤 여부) + 요청 빈도를 복합 분석한다. `navigator.webdriver`, `plugins` 배열 등. "다량 구매 후 취소 반복"이 주 타깃이고, 1매 예매용이면 해당 없지만 세션 기반 차단이 걸린다.

오늘의 결론: **현재 방법으로는 지속적 우회 불가**.

`daejong-page`에 `dead-ends` 섹션을 새로 만들었다. 코레일 봇이 첫 번째 사례다. 후속안은 코레일 앱 API 스니핑 + Appium 자동화 방향.

> **배운 것** — 막힌 길을 아는 것도 자산이다. dead-ends는 같은 길을 두 번 가지 않기 위한 지도다.

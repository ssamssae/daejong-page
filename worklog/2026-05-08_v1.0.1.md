## 1. 텔레그램 메시지에 왜 시간이 붙어 있는가

`agent-msg-notify.sh`의 69번째 줄에 `(${TIME})`이 있었다.

엄격 포맷 규칙은 `[🍎→🪟] [명령] 내용` — 타임스탬프 없음. 하지만 실제로 전송되는 메시지는 `[🍎→🪟] [명령] 내용 (09:48 KST)`. 3기기가 각자 스크립트를 수정하면서 버전이 달라졌고, 아무도 전체 포맷을 체크하지 않았다.

첫 번째 수정: `MSG="[${FROM}→${TO}] [${TYPE}] ${SUMMARY}"`. 타임스탬프 완전 제거.

> **배운 것** — 프로토콜 포맷 규칙은 스크립트가 SoT다. 문서가 아니라.

## 2. 왜 같은 메시지가 두 번 오는가

동시 호출 시 락이 없었다. 재시도 루프나 레이스 컨디션으로 동일 메시지가 n회 전송됐다.

`/tmp/agent-msg-lock/<hash>.lock` 파일로 해결. 동일 (from, to, type, 내용) 조합이 3초 내 재호출되면 두 번째는 조용히 차단.

## 3. Protocol identity ≠ Transport routing

가장 중요한 버그. `[🏭→🍎] [결과]` 헤더는 맞는데 실제로 엉뚱한 곳으로 전송됐다.

기존 코드는 token 선택을 "현재 기기 == TO 기기인가?" 휴리스틱으로 했다. MacBook에서 실행하면 MacBook 봇 토큰, Mac mini에서 실행하면 Mac mini 봇 토큰. 같은 코드가 다른 기기에서 돌면 다른 bot으로 전송됐다.

```
# 기존 (잘못된 패턴)
[[ "$TO_NORM" == "$LOCAL" ]] && TOKEN="${BOT_TOKEN}" || TOKEN="${PEER_BOT_TOKEN}"
```

수정: TO 이름만 보고 결정. 실행 환경 완전 무관.

```
case "$TO_NORM" in
  macbook) SEND_TOKEN="${TELEGRAM_BOT_TOKEN_MACBOOK}" ;;
  wsl)     SEND_TOKEN="${TELEGRAM_BOT_TOKEN_WSL}" ;;
  macmini) SEND_TOKEN="${TELEGRAM_BOT_TOKEN_MACMINI}" ;;
esac
```

이것이 Layer 1(프로토콜 헤더) / Layer 2(Telegram 전송 목적지) 분리다. 프로토콜 헤더는 논리 라우팅의 출처. Telegram 메타데이터는 전송 레이어일 뿐. 절대로 sender context에서 protocol identity를 역추론하지 않는다.

> **배운 것** — 같은 스크립트가 다른 기기에서 다르게 동작한다면, 환경 추론이 들어있다는 신호다.

## 4. Codex의 침묵 — "입력중..." 이 안 뜨는 이유

Mac mini Codex가 작업 중일 때 Telegram에 타이핑 인디케이터가 안 뜬다는 보고.

OpenClaw 소스를 파고들었다(`typing-mode-BMkcOlEk.js`). `resolveTypingMode` 함수가 `configured` 파라미터를 보는데, `agents.defaults.typingMode`가 설정되지 않아 기본값(undefined)이었다.

`openclaw.json`에 `"typingMode": "instant"` 한 줄 추가, daemon 재시작. 해결.

## 5. 루프를 짜면 됩니다 — loop-judge + loop-run

오전 버그 수정이 기반이 됐다. "바이브코딩에서 에이전트로 진화하려면?" 질문 후 설계가 시작됐다.

**loop-judge.sh**: MacBook Claude + Mac mini Codex가 같은 기준으로 독립 평가 → 합의 시 SUCCESS. 둘 다 실패하면 FAIL, 의견이 갈리면 DIVERGE → 사람 에스컬레이션.

실전 테스트: `sw_vers` 실행 → MacBook 판정 SUCCESS + Codex 판정 SUCCESS → 합의 SUCCESS. 전 구간 동작 확인.

**loop-run.sh**: PLAN → EXECUTE → JUDGE → REPORT 전체 파이프라인. 목표를 입력하면 Claude가 JSON task 배열로 분해, 기기별 실행(macbook/wsl/macmini), loop-judge 합의 검증, 최대 2회 재시도, 완료 보고.

두 스킬 모두 자연어 트리거로 등록. "합의 검증해줘", "루프 돌려줘" 한마디로 실행.

> 15:50, Mac·WSL·Mac mini가 처음으로 함께 브레인스토밍을 마쳤다.

오늘 오후 세션은 작은 버그 하나에서 시작해 새 스킬 하나로 끝났다. session-clear가 자기 자신을 죽이고 있었고, WSL이 없는 스킬이 있다고 주장했고, 세 기기가 각자 독립적으로 아이디어를 내는 구조를 처음 돌렸다.

## 1. 스크립트가 자기 자신을 죽이고 있었다

`/reset`이나 "세션클리어"를 보내면 claude.exe가 반응하지 않았다. session-clear v0.7이 C-c를 보내는 대상이 바로 자신을 실행 중인 프로세스였기 때문이다. bash 도구 서브프로세스가 부모(claude.exe)에 인터럽트를 보내고, 부모는 도구를 중단하고, `/clear`는 영원히 전달되지 않는다.

trio-vote에 올렸다. 3인(PM·엔지니어·비판론자)이 3-0 만장일치로 B안을 골랐다.

```bash
# v0.8 — Delayed Detach
(sleep 1 && $TMUX_BIN send-keys -t claude "/clear" Enter) &
$TMUX_BIN send-keys -t claude C-c
```

백그라운드 subshell이 1초 뒤 `/clear`를 보내고, 스크립트는 C-c 전송 후 즉시 빠진다. 자기참조 없음.

> **배운 것** — 자신을 실행 중인 프로세스에 인터럽트를 보내면 인터럽트가 실행자를 죽인다. "도구를 먼저 끊고 명령을 보낸다"는 직관이 역방향에서 작동하지 않는 이유.

## 2. WSL이 trio-vote를 "없는 스킬"이라고 했다

WSL이 vibecoding.html 교차검증 중 `/trio-vote`, `/device-run`이 "존재하지 않는다"고 답했다. 오늘만 두 번 실행한 스킬이다.

원인은 `wsl/korail-bot-2026-05-05` 브랜치 고정. `~/.claude/skills` 심링크가 가리키는 `~/claude-skills`가 main이 아닌 브랜치에 묶여 있었고, main의 7a331f6 refactor 커밋(device-run 추가, loop-judge 삭제 등)을 아예 보지 못하고 있었다.

trio-vote 3-0 → 옵션2(헬퍼 스크립트) 채택. WSL이 `scripts/skill-exists.sh`를 8줄로 작성해 PR #28을 열었다.

```bash
# 사용법: skill-exists.sh <skill-name>  →  exit 0=존재, 1=없음
git -C "$REPO" fetch --quiet origin main
git -C "$REPO" ls-tree origin/main --name-only | grep -qx "$1"
```

4/4 PASS(trio-vote ✅ device-run ✅ loop-judge exit 1 ✅ nonexistent exit 1 ✅) 확인 후 본진이 머지.

> **배운 것** — `ls ~/.claude/skills/`는 로컬 브랜치 상태를 본다. 스킬 존재 확인은 `origin/main` 기준이어야 한다.

## 3. 세 기기가 처음으로 같이 생각했다

"세 기기간 병렬 브레인스토밍 못 하나?" 라는 질문 하나로 시작됐다. 인프라를 따져보니 이미 다 있었다.

- 본진 → WSL: `wsl-directive.sh` SSH tmux inject
- 본진 → Codex(Mac mini): `agent-msg-notify.sh` Telegram DM
- WSL → 본진: `ssh mac` 파일 저장
- Codex → 본진: `ssh user@100.74.85.37` 역방향 Tailscale SSH

`/mesh-vote` 스킬을 설계했다. Session ID 기반 임시 디렉토리에 각 기기가 결과 파일을 쓰고, 본진이 폴링으로 수집한 뒤 투표한다.

첫 실전 테스트 주제: "단어요 다음 업데이트 가장 임팩트 있는 기능."

🍎 Mac: 반복 학습 스케줄(Spaced Repetition)  
🪟 WSL: SRS 엔진 도입 — 35초 만에 도착  
🏭 Codex: SSH 불통으로 미참여

Mac과 WSL이 독립적으로 같은 답(SRS)을 냈다. Codex SSH 경로를 `ssh user@100.74.85.37`로 수정하고 파일 쓰기까지 검증해 뒀다. 다음 실행은 진짜 3기기 완전체로 돌아간다.

> **배운 것** — 기기 간 브레인스토밍을 가로막던 건 아이디어가 아니라 Codex의 역방향 SSH 경로 하나였다.

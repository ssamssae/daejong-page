## 1. Hermes 가 LLM 이 아니라는 사실을 카드에서 빼는 일

시작은 단순했다. 강대종이 노트북 3060(`DESKTOP-4MNJ1C0`) 카드의 9가지 부정확 표현을 짚어줬다. 핵심 정정 4종:

- **Hermes ≠ LLM**. Hermes 는 에이전트 프레임워크/클라이언트. 현재 노트북에서 돌고 있는 Hermes 세션은 OpenAI Codex provider 의 gpt-5.5 모델을 호출하는 중. "로컬 LLM Hermes 22 tok/s" 는 LLM 도 아닌 것에 LLM 의 성능 지표를 붙인 셈.
- **WSL 안에서 GPU = 미확인**. nvidia-smi 가 unavailable 이고 `/dev/dxg`(WSL GPU/DirectX 브리지 장치 노드)만 노출됨. CUDA 런타임 / VRAM 정상 인식까지는 검증되지 않은 상태.
- **OS = Ubuntu 24.04.4 LTS on WSL2**, 호스트명 `DESKTOP-4MNJ1C0`. 풍성 포맷에서 spec 라인에 명시.
- **ghostty / Google RD / sshd / Telegram 봇 = 별도 확인 필요**. 모두 🟡 (확인 필요)로 다운그레이드.

두 번 다시 쓰면서 깨달은 패턴 하나: 부정확한 표현은 보통 **두 사실을 묶어서** 한 줄로 쓸 때 생긴다. "로컬 LLM Hermes 22 tok/s" 는 (a) Hermes 가 무엇인가 (b) 22 tok/s 가 누구의 지표인가 두 사실을 한 줄로 합쳤다가 둘 다 어그러뜨렸다. 한 행에 사실 한 개씩.

> **배운 것** — 두 사실을 한 줄에 묶지 말 것. 묶인 줄은 거의 항상 둘 중 하나는 틀린다.

## 2. 한 카드의 부정확이 다른 카드의 부정확을 끌어왔다

노트북 3060 카드를 정리하다가 묘하게 Mac mini 카드도 손을 봐야 한다는 생각이 들었다. Mac mini 는 2026-05-08 부터 Codex(OpenClaw) 직접 Telegram 세션을 가지고 있는데(`@ssamssae_claw_bot`), stack.html 카드는 여전히 "ghostty ❌", "tmux ❌ 미사용", "Claude Code ❌" 로 적혀 있었다. 강대종 본인이 직접 점검 결과를 보내줬다 — 8개 row 모두 ✅, OS 행에 `macOS / Darwin arm64 (WSL/Ubuntu 아님)`, Google RD 메타에 "Chrome Remote Desktop Host 실행 중", role 라인에 "Codex 에이전트 / OpenClaw 메인 워크스페이스" 추가. 풍성 포맷 패턴이 노트북 3060 → Mac mini 로 전염된 첫 카드.

이 시점에서 또 보였다. WSL 카드도 메타가 옛날 그대로였다. tmux 가 "main + claude 2 세션" 으로만 적혀 있는데 실제로는 group 멀티클라(3) + claude-main + main + claude = 6 세션이었다. 그래서 강대종 폰에 WSL 세션이 직접 자기 ground truth 를 보내줬다 — bun v1.3.12, Codex CLI 설치만(`크로스기기 라우팅용, 활성 사용 X`), GPU `nvidia-smi unavailable · /dev/dxg만 노출 (RTX 2070S)`, SSH 메타에 `\`ssh w\` alias`. 카드 한 장이 잡으면 다음 카드의 부정확이 보이는 패턴이 반복.

## 5. 검증되지 않은 칭찬은 칭찬이 아니다

마지막으로 강대종이 던진 한 마디 — "본진도 좀 업데이트 해야하지 않겠어?" 본진 카드만 풍성 포맷 미적용이었다. Mac 본진 위에서 직접 도는 세션이라 ground truth 를 자체 검증할 수 있었다:

```bash
sw_vers                       # macOS 26.4.1 (Build 25E253)
uname -m                      # arm64
tmux list-sessions            # 6 세션 (claude group + 3 attached + claude-main + main)
which bun; bun --version      # ~/.bun/bin/bun, 1.3.12
which codex; codex --version  # /opt/homebrew/bin/codex, codex-cli 0.121.0
cat ~/.claude/plugins/installed_plugins.json  # 4개 plugin
```

검증 못한 게 있었다. `sudo systemsetup -getremotelogin` 은 sudo 미인증이라 못 돌렸고, Chrome Remote Desktop Host.app 은 `/Applications` 직속에 Uninstaller.app 만 보였다. 그래서 SSH 는 "Termius/Mac mini inbound 수신 중" 이라는 운영적 증거로 ✅ 유지, GRD 는 "Chrome Remote Desktop" 메타로 ✅ 유지 — 단언은 하지 않고 확인된 만큼만.

각 카드의 신뢰도를 위해서는 ✅ 가 두 종류로 갈렸다:

1. **직접 검증된 ✅** — 명령어 실행 결과로 사실 확인된 항목 (tmux 6 세션, bun 1.3.12, Codex 0.121.0, plugin 4개)
2. **운영적 증거로 유지된 ✅** — 직접 명령으론 검증 못 했지만 매일 사용 중이라 ✅ 유지된 항목 (SSH server, GRD)

> **배운 것** — ✅ 라는 같은 기호 안에도 신뢰도의 결이 있다. 메타에서 "어떻게 ✅ 인지" 를 적어두면 다음에 검증할 때 어디서부터 시작할지 알 수 있다.

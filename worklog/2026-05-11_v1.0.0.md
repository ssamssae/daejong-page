## 1. /clear 가 다섯 번 거절했다

00:52 KST. 어제 일지 v1.0.3 + done 을 push 하고 잘 준비. 텔레그램으로 메시지 몇 개를 보내면서 자동 /clear 를 기대. launchd 트리거 로그가 이렇다.

```
[00:52:07] fired, marker=exists
[00:52:19] rescue: marker still exists after 15s, idle-check start
[00:52:28] timeout waiting for idle, abort
[00:54:00] fired, marker=exists
[00:54:20] timeout waiting for idle, abort
... (5+회 반복) ...
[00:57:31] fired, marker=exists
[00:57:33] pane=%2 mode=0 ready(❯), sending /clear
[00:57:34] done
```

원인은 단순했다. v2.2 트리거가 tmux pane 캡처에서 "esc to interrupt" 부재를 idle 시그널로 쓴다. 텔레그램 메시지가 20초 안 간격으로 burst 면 새 turn 이 계속 시작되니 idle 윈도우가 영원히 안 온다. v2.1→v2.2 fix 는 "abort 시 마커 보존" 만 해결한 것 — 발화 못 함 자체는 그대로 남아 있었다.

> **배운 것** — 모니터링 idle 폴링은 burst 트래픽 앞에서 무력하다. 그냥 abort 하지 말고, 사람한테 "지금 못 하고 있다" 라고 알려줘야 사람 의지에 의존하지 않는다.

이슈로 박았다 (`2026-05-11-launchd-clear-trigger-abort-loop`). 본진 todos 진행중에 forcing function 도 추가 — v2.3 은 abort 5회 누적 시 텔레그램 1통 + 외부 강제 명령 한 줄을 본문에 박기 (long-press copy 우회).

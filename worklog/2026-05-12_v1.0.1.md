## 1. ephemeral clone 으로 막은 누수

fleet-state 의 D07 cron 통합 직전 단 하나의 blocker. `tests/*.test.sh` 가 fixture commit 만들고 `git push origin main` 할 때 local main 의 unreviewed implementer commit 까지 origin 으로 끌려 올라가는 회귀. 어제(5/11) bc2bbbf 가 review 전에 origin 박힌 사고의 process 변종.

해법 후보 세 개를 깔았다. (a) 별도 branch 에서 push, (b) ephemeral clone 으로 격리, (c) 변경 사항을 stash. (c) 는 commit 자체를 stash 못 하니 탈락. (a) 는 mac-mini 가 race 검증을 위해 origin/main 에서 pull 해야 해서 branch 분리 시 의미가 깨짐. (b) 가 답이었다.

```bash
ORIGIN_URL=$(cd "$ORIG_DIR" && git config --get remote.origin.url)
EPHEMERAL_DIR=$(mktemp -d -t fleet-test-XXXXXX)
trap 'rm -rf "$EPHEMERAL_DIR"' EXIT
git clone --quiet "$ORIGIN_URL" "$EPHEMERAL_DIR/repo"
cd "$EPHEMERAL_DIR/repo"
```

3 test 파일에 같은 prologue 6줄 + body 안 `~/agent-fleet-state` 하드코딩을 `$EPHEMERAL_DIR/repo` 로 교체. 검증은 단순했다. local main 에 `e37399e WIP: D07-prereq test push isolation (NOT YET REVIEWED)` fake commit 박은 상태에서 claim-single + stale-recover 돌리고, 끝나고 `git log origin/main | grep -c e37399e` 가 **0** 인지 확인. PASS.

> **배운 것** — test harness 가 production 코드와 같은 working tree 에서 돌면 언제든 누수가 가능하다. 격리 비용은 한 번 박은 prologue 6줄이고, 안 박으면 매 사이클 process 사고로 돌아온다.

claim-race 만 mac-mini ssh 단계에서 막혔다. mac-mini local main 이 origin 과 10 ahead orphan / 30 behind 로 diverged 라 `git pull --ff-only` 가 exit 128. push 격리 문제가 아니라 별도 항목 (보류 todo). mac-mini reset ack 송신했다.

debug install 성공. 그런데 강대종이 보내온 다음 메시지가 "앱이 켜지질 않는데". debug 빌드는 Dart VM JIT — daemon 없이 단독 실행 시 그냥 죽는다. v4 release 빌드로 재시도. 약먹자는 켜졌다. 광고도 떴다. **더치페이만 안 켜졌다**.

## 3. Info.plist 의 빠뜨린 한 줄

`ios/Runner/Info.plist` 전수 검사. 25 key. 그 안에 `GADApplicationIdentifier` 가 없었다. 약먹자에는 박혀 있었다 (`ca-app-pub-7025432711849670~1525526197`, 2026-05-02 발급).

Google Mobile Ads SDK iOS 는 init 시 이 key 가 없으면 `GADInvalidInitializationException` throw 하며 앱을 즉시 종료시킨다. 더치페이 main.dart 첫 줄의 `await AdsService.init()` 가 `MobileAds.instance.initialize()` 부르고, 거기서 죽음. 사용자 입장에선 "아이콘 탭 → 잠깐 검은 화면 → 홈 복귀".

ads_service.dart 클래스 docstring 에 본인이 적어 두었다:

> AppId 도 Info.plist 의 GADApplicationIdentifier 교체 필요.

— 어제 광고 commit (`a3bb4de feat(ads): AdMob 배너 광고 추가 (iOS-only)`) 들어올 때 이 단계가 누락됐다. ads_service.dart 안의 placeholder `_realIosBannerUnitId = 'XXXX...'` 와 Info.plist 의 GAD key 둘 다 손대지 않은 채로 빌드만 통과했다. ASC 심사도 안 거쳤으니 외부 검증 0.

> **배운 것** — 빌드가 통과한다고 launch 가 통과하지 않는다. AdMob SDK 추가는 5단계 절차 (콘솔 발급 → Info.plist → ads_service → release 빌드 → 실기 검증) 가 한 묶음으로 가야 한다. 한 단계라도 누락되면 사용자 폰에서 즉시 crash.

## 5. ASC API 자동 심사 제출

광고 fix 가 검증되자 "광고버전 출시는 언제해 앱스토어 출시해야지" 로 무게 중심이 이동했다. /submit-app 스킬 진입.

ASC live 상태 API 점검 결과 둘 다 광고 없는 옛 버전이 라이브였다. 더치페이 v1.0.2 (2026-04-11), 약먹자 v1.0.1 (2026-04-12). 둘 다 출시 사이클 GO.

| 앱 | 라이브 → 새 버전 | IPA |
|---|---|---|
| 더치페이 | v1.0.2 → **v1.0.3+6** | 22.9MB |
| 약먹자 | v1.0.1 → **v1.0.2+3** | 24.6MB |

`flutter build ipa --release` + `xcrun altool --upload-app` 둘 다 UPLOAD SUCCEEDED. ASC build processing 은 놀랍게도 약먹자가 첫 polling (60s) 만에 VALID. 더치페이도 비슷한 속도.

submit 부분은 ASC API 로 자동화하기로 했다. mac-mini system Python 이 3.9 라 `str | None` 같은 3.10+ syntax 가 박힌 기존 `asc-deliver.py` 가 못 돈다. 그래서 Python 3.9 호환 새 스크립트 작성:

```python
def main(bundle, ver, build_no, whats_new):
    apps = api('GET', f'/v1/apps?filter[bundleId]={bundle}')['data']
    app_id = apps[0]['id']
    # 1. POST appStoreVersions
    # 2. POST localizations (이전 live 의 ko loc 복사 + whatsNew 박음)
    # 3. PATCH /v1/builds/{id} usesNonExemptEncryption=False  ← 이거 안 박으면 다음 단계 409
    # 4. PATCH /v1/appStoreVersions/{id}/relationships/build  (build attach)
    # 5. POST/find reviewSubmission + reviewSubmissionItems + PATCH submitted=True
```

도중 두 함정. **timeout 30s 부족** — ASC API 가 가끔 30s 초과. 90s 로 늘렸다. **usesNonExemptEncryption 누락 409** — `POST /v1/reviewSubmissionItems` 시 associatedErrors 로 surface. ASC 웹 UI 는 build attach 시 dialog 로 자동으로 묻지만, API 는 그런 prompt 없다. 별도 PATCH 박아야 한다.

```json
{
  "code": "ENTITY_ERROR.ATTRIBUTE.REQUIRED",
  "detail": "You must provide a value for the attribute 'usesNonExemptEncryption' with this request",
  "source": {"pointer": "/data/attributes/usesNonExemptEncryption"}
}
```

> **배운 것** — Web UI 가 자동으로 처리하는 것들이 API-first 자동화에선 코드로 명시돼야 한다. Apple 은 "사용자가 dialog 에서 답한 것" 만 default 로 안 주고 422/409 로 폭격. 영구 해결은 Info.plist 에 `ITSAppUsesNonExemptEncryption=false` 박아두면 다음 빌드부터 ASC 자동 인식, API PATCH 불필요.

최종:
- **더치페이 v1.0.3** reviewSubmission `df48b9f9-...` **SUBMITTED**
- **약먹자 v1.0.2** reviewSubmission `b6a60ee5-...` **SUBMITTED**
- 둘 다 releaseType AFTER_APPROVAL → 승인 직후 자동 출시
- W6 territory verify guard 가 174 국가 누락 후처리

mail-watcher v5 가 Gmail 4시간 폴링 → 24~48h 안에 결과 텔레그램 알림.

## 1. "3" 은 시간 토큰이 아니었다

`/loop 3` 을 입력했을 때 모델 입장에서 "3" 은 `5m`/`2h` 같은 시간 패턴이 아니다. 그러니 fixed-interval 모드로 못 가고 dynamic 모드로 진입한다. dynamic 은 "interval 안 정해진, 모델이 스스로 페이스 잡으면서 굴리는" 모드.

그러면 prompt 는 뭐냐? 그냥 "3" 이다. 직전 메시지에서 모델이 surface 했던 세 가지 옵션 중 3번 = 잡일 트랙. 모델이 그렇게 해석했고, 즉시 잡일 청소 사이클을 시작했다.

> **배운 것** — dynamic `/loop <text>` 는 prompt 가 자연어 한 단어여도 받아들인다. 직전 컨텍스트가 충분하면 모델이 알아서 의미를 채운다.

## 2. 12 iter 동안 자동으로 굴러간 청소

매 iter 끝에 `ScheduleWakeup(delaySeconds=N, prompt="/loop 3")` 가 자동으로 다음 fire 를 예약한다. 그래서 사용자가 한 번 입력했어도 12번 깨어났다.

- **iter 1** (02:04) — simple-memo-app PR #5 머지. 메모요 1.0.4 백로그 문서 (+57/-0). `f4e1760`
- **iter 2** (02:08) — claude-automations PR #12 + dutch_pay PR #4 머지 시도. 둘 다 conflict. 강대종 보류 ack 대기.
- **iter 3** (02:33) — done/2026-05-13.md 에 방금 머지한 PR #5 라인 추가.
- **iter 4** (03:00) — todos/2026-05-13.md 의 stale 두 줄 갱신 (PR #4 stale [x] + PR #12 conflict 상태 명시).
- **iter 5** (03:30) — issues/INDEX.md 동기화. SoT 74줄 ↔ mirror 67줄, 누락 4건 채움.
- **iter 7** (04:21) — done/index.json count 12→13 + summary 갱신.

iter 6 부터 풀이 줄어들기 시작했다. 자체 픽 대상이 "처분 ack 필요" 항목으로 채워졌다. iter 11~12 는 사실상 "no delta" 만 보고하고 끝.

| iter | wakeup | 의미 |
|---|---|---|
| 1~5 | 25분 | 활동 많음 |
| 6~10 | 35분 | 풀 줄어듦 |
| 11~12 | 60분 | 깊은 idle |

> **배운 것** — dynamic loop 의 정직한 신호 = "ack 보류 누적 3건 이상" 또는 "no delta 2 iter 연속". 그 시점에 wakeup 늘리거나 종료 제안.

## 4. 그리고 06:48, 강대종이 깼다

`/loop` 멈춤 트리거 없으면 무한 굴러간다는 사실을 강대종이 그제야 직접 물었다.

> "내가 루프트리거 어떻게 명령했길래 몇분동안 진행했고 멈추게된 트리거는 뭐야?"

답: 트리거는 한 번, 06:48 본인 발화가 turn 을 깨운 게 사실상 일시정지. 큐의 wakeup 1개는 여전히 살아있다.

이어진 7개 질문이 30분 동안 펼쳐졌다. `/loop 5m` 이 7일 만료라는 confirm 메시지는 5분이 만료라는 뜻이 아니다 (5분=주기, 7일=등록 자체의 cap), dynamic 은 7일 cap 없다, 인스타포스트는 launchd 라 영구다, 컨텍스트 누적은 prompt cache 5분 TTL 가운데 구간 (300~1000s) 이 worst-of-both 이고 270s 또는 1200s+ 가 정답이다, 등등.

마지막 질문이 가장 컸다:

> "araseo / fleet 이 그거랑 이 루프랑 별 차이 없어보이는데? 너는 뭘 만들고 있던거야?"

```bash
$ grep claude.*--print agent-fleet-state/scripts/spawn-claude.sh
echo "$PROMPT" | claude --print --output-format text 2>&1 | tee ...
```

fleet 은 `claude --print` 단발 CLI 호출. 매 fire 가 새 프로세스 = 새 컨텍스트 윈도우. 컨텍스트 누적 0 = 장기 cron 워크에 압도적 유리. /loop dynamic 과 카테고리가 완전히 다르다.

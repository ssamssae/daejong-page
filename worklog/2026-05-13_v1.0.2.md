## 1. 노하우와 정보는 다른 페이지였다

홈페이지에 비슷한 두 페이지가 있다. `/knowhow` 는 .md 본문 링크 카드 — 한 카드 하나가 긴 마크다운 본문으로 깊게 들어간다. `/info` 는 "자주 헷갈리는 것들 모음" — 한 카드 안에 비교표 + 짧은 팁이 들어간 컬렉션. 같은 주제도 톤이 다르다.

강대종 요청 = 새벽에 노하우에 박은 메커니즘 비교 내용을 info 페이지에도 압축본으로 박는 것. 노하우 본문 +183줄을 카드 두 장으로 줄였다.

| 카드 | 핵심 |
|---|---|
| `/loop fixed vs dynamic — 두 모드의 차이` | 호출 형태 / 메커니즘 / cap / 끄는 법 / 적합 케이스 5행 비교표 |
| `/loop dynamic 의 delaySeconds — 5분 cache TTL 분기점` | 60~270s (활성 폴링) / 300~1000s (금지) / 1200~3600s (idle 점검) 3행 비교표 |

commit: `6f815c6` (+82줄).

> **배운 것** — knowhow 가 "한 주제를 깊게 들어가는 본문" 이라면 info 는 "헷갈리지 않으려고 빠르게 펼치는 cheat sheet". 같은 주제라도 두 페이지에 다른 모양으로 박을 가치가 있다.

## 2. 메모요 라벨 한 줄

```html
<div class="app-status">개인 사용</div>
```

이 줄이 어제 production 라이브 후에도 그대로 남아 있었다. 다른 다섯 앱(한줄일기 / 약먹자 / 더치페이 / 포모도로 / 한컵) 은 전부 `App Store 출시` 라벨. 메모요만 stale.

```html
<div class="app-status">App Store 출시</div>
```

한 줄 바꾸고 title attribute (`메모요 — 첫 Flutter 앱` → `메모요 — App Store 출시`) 도 동기화. commit: `6feb5b3`. 30초.

## 3. PR 두 개가 동시에 닫혔다

새벽 iter 2 에서 머지 시도했다가 conflict 로 보류된 두 PR.

**claude-automations PR #12** (`fix(codex-directive): WSL 등 openclaw 미설치 환경에서도 작동`) — 강대종이 후에 main 에 동일 파일을 세 번 더 직접 수정했다. 의도는 살아있지만 코드 베이스가 완전히 달라져서 rebase 가 거의 새 PR 수준이 됨. 같은 의도는 별 PR 로 처리하기로 하고 obsolete close.

**dutch_pay_calculator PR #4** (`feat(ads): 더치페이 iOS AdMob 운영 ID 적용`) — main `3c5daa7` 가 같은 일을 이미 한 후라 PR 자체가 의미 없음. obsolete close.

```bash
$ gh pr close 12 -R ssamssae/claude-automations -c "...obsolete close."
✓ Closed pull request ssamssae/claude-automations#12

$ gh pr close 4 -R ssamssae/dutch_pay_calculator -c "...obsolete close."
✓ Closed pull request ssamssae/dutch_pay_calculator#4
```

> **배운 것** — branch 가 main 보다 뒤처져서 머지 못 하는 PR 중 상당수는 "main 이 이미 그 일을 다른 방식으로 했음" 케이스. rebase 시도 전에 main commit log 부터 봐야 한다.

## 4. /goodnight 자동 매칭이 그 세 건을 잡았다

강대종 "가자" → /goodnight 트리거. 1.5단계 자동 매칭 로직이 이번 세션에서 처리한 일과 열린 todos 의 토큰을 비교했다. 3건 후보:

| 후보 | 매칭 토큰 |
|---|---|
| 🛒 dutch_pay PR #4 처분 ack | PR #4, close, obsolete |
| 🌐 index.html L355 메모요 `app-status` 라벨 의도 확인 | index.html, 메모요, App Store 출시 |
| 🔧 automations PR #12 검토·머지 | PR #12, close, obsolete (머지 X close 처분) |

강대종 "전부" ack → 3건 일괄 [x] 마킹 + 완료 메모 + commit + push.

`automations PR #12` 가 흥미로웠다. todo 원본은 "검토·머지" — 닫혔지만 머지된 게 아니라 obsolete close 였다. 자동 매칭이 그래도 "처분" 의미는 같다고 잡았고 강대종이 그걸 ack 했다. 매칭 임계값 2 토큰 룰이 의도와 어긋나지 않게 작동한 사례.

> **배운 것** — todo 본문이 "머지" 라고 박혀 있어도 처분 = 머지 OR close OR drop 다 같은 카테고리. 자동 매칭은 그 차이를 자체 픽 못 하지만, 사용자 ack 한 번이 그 모호함을 해소한다.

## 1. Phase A — gmail-mcp 가 받지 못한 형식

gog 시대에 쓰던 OAuth client 의 `credentials.json` 을 mac mini 에서 본진으로 그대로 옮겼는데, gmail-mcp 가 안 읽었다. 이유는 flat 구조 (`{"client_id": ..., "client_secret": ...}`) 였고, gmail-mcp 는 Desktop OAuth client 표준 형식 (`{"installed": {...}}`) 을 기대했다.

Python 으로 wrap + 표준 4필드 (`redirect_uris=["http://localhost"]` / `auth_uri` / `token_uri` / `auth_provider_x509_cert_url`) 채움. `~/.gmail-mcp/gcp-oauth.keys.json` 430B 0600.

`gmail-mcp auth` → 브라우저 OAuth → `~/.gmail-mcp/credentials.json` 552B 발급. Gmail REST API self-send verify: `HTTP 200`, msg id `19e21d136e47d8bc`, labelIds `SENT + INBOX + UNREAD`. 토큰 잔여 56분.

> **배운 것** — flat ↔ wrap 형식 mismatch 는 호스트 위치 (mac mini / 본진) 와 무관한 코드 level 문제. "어디서 돌리면 작동하나" 가 아니라 "어떤 형식을 expect 하나" 가 답.

## 4. 봇이 답을 안 했다 — bun 이 없어서

채팅에 강대종 메시지 10개가 ✓✓ 만 찍힌 채 쌓였다. `getUpdates` 로 직접 확인했더니 큐에 10개 그대로 있음. mac mini Claude Code 가 polling 자체를 안 잡고 있다는 뜻.

근데 진단 결과가 재미있었다. mac mini Claude Code 가 자기 자신을 진단해서 채팅으로 답을 보내려고 한 화면을 강대종이 캡쳐:

> MCP 서버가 bun으로 실행돼야 하는데 bun이 없어서 silently fail. claude --channels 프로세스는 살아있지만 MCP 서버가 안 떠서 telegram bot이 실제로 로그인을 못 하고 있어요. README도 prerequisite로 bun을 못 박고 있습니다 — "The MCP server runs on Bun."

정확했다. `plugin:telegram` 의 `server.ts` 가 bun runtime 으로 실행돼야 하는데 mac mini 에 bun 없음. polling 시작 못 함.

```bash
curl -fsSL https://bun.sh/install | bash
```

`~/.bun/bin/bun` 설치 → tmux claude session 재시작 → `bun server.ts` (PID 89646) live → `getUpdates HTTP 409 Conflict` 떨어짐 (polling 잡음 신호) → 봇이 답하기 시작.

> **배운 것** — MCP server runtime (bun) 미설치 시 plugin 은 silently fail. claude 본체는 "Listening for channel messages" 출력하지만 실제 long-poll 은 안 됨. 검증은 polling lock (HTTP 409) 으로.

## 5. 토큰이 두 번 죽었다

**사고 1** — 셋업 중 `grep "TELEGRAM_BOT_TOKEN" .env` 를 두 번 호출했는데 첫 호출에서 sed 정제를 빠뜨렸다. `TELEGRAM_BOT_TOKEN_MACMINI` 풀텍스트가 conversation 로그에 박힘. 두 번째 호출은 sed redact 했지만 이미 늦었음. 강대종에게 즉시 사고 보고 + BotFather rotate 권장.

**사고 2** — 강대종이 BotFather rotate 진행 후 mac mini `rotate-token.sh --channel` 실행. 그 mode 는 OpenClaw stack 의존 (`openclaw channels add --channel telegram --token ...`). 그런데 Phase D 에서 openclaw CLI 를 방금 uninstall 했음. 스크립트가 `✗ openclaw CLI not found in PATH` 뱉고 종료 — **.env 갱신 안 함**. 결과: 새 토큰 어디에도 저장 안 됨 + 옛 토큰 BotFather rotate 로 invalidate → mac mini Claude Code polling 401 → 봇 연결 끊김.

`rotate-token.sh --alert` 로 다시 rotate → `.env` 갱신 OK (tail=WDpSx0, 옛 tail=t5ygYE 와 완전 다름) → 본진 `TELEGRAM_BOT_TOKEN_MACMINI` Python swap sync → mac mini claude session fresh restart → 강대종 "된다" (01:01 KST).

> **배운 것** — decom 사이클 직후 기존 자동화 스크립트의 dead path audit 필수. `rotate-token.sh --channel` 처럼 OpenClaw 시대 패턴이 박힌 코드는 사이클 끝나는 turn 에 같이 보거나 alias 처리.
> **배운 것** — secret env 변수 grep 결과 출력은 무조건 `length+tail` 만 노출. raw grep 노출은 forcing function (hook 후보) 으로 차단.

issues/ 에 둘 다 박제: `2026-05-14-macmini-bot-token-grep-leak.md` + `2026-05-14-rotate-token-channel-mode-dead.md`. claude-skills 와 daejong-page 양쪽에 push (66 entries 누적).

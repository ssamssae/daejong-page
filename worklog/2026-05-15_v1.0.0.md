```

```

GitHub 의 `~/ssamssae.keys` 확인 → 4 ed25519 키 있는데 mac-bonjin 본문 (`AAAAC3...t7BY`) 없음. fingerprint mismatch. paste 가 어디서 끊겼거나 다른 key 박힘. 강대종에게 알림.

강대종이 사진 보냄 — GitHub Settings 페이지에 빨간 에러:

> We got an error adding your GPG key. Please verify the input is a valid GPG key.

GPG keys 탭에 paste 한 거였다. SSH and GPG keys 페이지의 두 섹션 중 잘못된 쪽. 본진이 https://github.com/settings/ssh/new 안내 + 터미널 한 줄 명령:

```
gh auth refresh -h github.com -s admin:public_key && gh ssh-key add ~/.ssh/id_ed25519_mac-bonjin.pub --title "mac-bonjin 2026-05-15"
```

강대종이 paste → "You have successfully added the key 'mac-bonjin'." 등록 PASS. 본진 재검증:

```
$ ssh -T git@github.com
Hi ssamssae! You've successfully authenticated, but GitHub does not provide shell access.
```

GitHub 등록 키 4 → 5 증가 확인. 5번째 노드 분업 정합 완료 — 5대 모두 personal SSH key 보유.

> **배운 것** — paste 안내할 때 "어느 페이지" 가 결정적이다. SSH/GPG 두 탭이 같은 URL prefix 라 헷갈리기 쉽다. 명령 한 줄이면 web UI 페이지 헷갈림 자체가 사라진다.

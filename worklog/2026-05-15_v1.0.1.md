## 3. 정규식 한 글자가 토큰 두 개를 넘겼다

15:18, .env 의 봇 설정을 점검하다가 redact 정규식을 짜서 grep 했다.

```bash
grep -E '^TELEGRAM_BOT_(TOKEN|USERNAME)' .env | sed -E 's/(TOKEN[A-Z_]*)=.*$/\1=<redacted>/'
```

문제 = `[A-Z_]*` 가 digits 매칭을 못 한다. `NOTEBOOK3060` / `DESKTOP3060TI` 처럼 숫자 박힌 키 이름이 정규식을 빠져나가서, 두 봇 (@ssamssae_codex_bot / @jarvice_ssamssae_bot) 의 토큰 raw 값이 그대로 출력됐다 — 그리고 conversation 컨텍스트에 박혔다. 텔레그램이나 공개 commit 으로 흘러간 건 아니라 외부 노출은 0. 그래도 같은 패턴이 2026-05-10 도 한 번 났으니 도구로 차단해야 했다.

| 비교 | 옛 패턴 | 새 도구 (`env-tool.sh`) |
|---|---|---|
| 값 입력 | 명령행 인자 (ps 노출) | stdin only |
| presence 체크 | grep + redact | `has-key` count/boolean |
| 값 추가 | echo >> .env | `ensure <key>` (stdin) |
| 값 교체 | sed -i | `rotate <key>` (stdin) atomic |
| 목록 | grep | `list` (값은 `***` 마스킹) |

scratch /tmp/test.env 8 시나리오 smoke 통과. 5 노드 deploy (Mac본진/mac-mini/desktop3060ti 는 git pull, WSL/notebook3060 은 claude-automations diverge 우회로 scp).

강대종한테 "토큰 두 개 rotate 진행할지" 물으니 처음엔 "강대종 손0 으로 진행" → 그건 BotFather 인간 클릭이라 자동화 불가라고 솔직하게 말씀드렸다. 17:22 "왜 줘 무슨 소리야" → 맥락 자연어로 다시 설명 → 17:23 "a" (취소) — 외부 노출 0 단계이므로 rotate 안 함. forcing function (env-tool.sh) 이 박혀 있으니 같은 사고 자체가 안 생긴다.

> **배운 것** — ad-hoc regex redact 는 신뢰 불가. 정규식 한 글자(`[A-Z_]*` vs `[A-Z_0-9]*`) 차이로 토큰이 새듯, 매번 새로 짜는 redact 패턴은 마지막에 한 번은 실수가 박힌다. 도구로 강제 — 값 자체가 출력 가능한 곳에 안 들어가게 — 만이 영구 방법.

## 4. SessionStart 가 자식 프로세스를 세게 됐다

2026-05-14 mac mini plugin cache 사고의 보조 forcing function. launchd plist `EnvironmentVariables.PATH` 누락으로 bun 못 찾는 케이스는 영구 해결됐지만, PATH 외 다른 원인 (plugin 신버전 호환성, 진짜 cache 손상) 으로 spawn 실패할 수도 있다.

`hooks/mcp-spawn-verify.sh` — 5초 wait 후 `pgrep -P $PPID` 자식 ≥ 1 검증. claude 본체 식별은 `$PPID` 직계 부모 우선 + ps 스캔 fallback. 그래야 mac 본진 (`claude --channels...`) 과 mac mini (`claude --dangerously-skip-permissions`) 양쪽 launch flavor 차이를 잡는다.

자식 0이면 텔레그램 1통.

```
⚠️ <hostname> SessionStart: claude (pid <X>) 자식 프로세스 0 — MCP server spawn 침묵 실패 의심.
진단: /plugin 슬래시 'Failed to reconnect' 확인.
처방: (a) cache rm ~/.claude/plugins/cache/claude-plugins-official + 재시작,
     (b) launchd plist EnvironmentVariables.PATH 점검
```

세션 차단 X — 경고만. `MCP_SPAWN_VERIFY_WAIT` env 로 노드별 5초 wait 튜닝 가능 (false positive 컨트롤).

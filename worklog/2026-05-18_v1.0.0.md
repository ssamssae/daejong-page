## 3. mesh-vote 5/5 + trio-vote 3-0 — 외부 표면 = Tailscale serve

본진 → 맥미니 cross-node ping 이 닿으려면 외부 표면이 있어야 한다. 후보는 셋이었다 — A. Tailscale serve (사설, tailnet only), B. CF Tunnel (외부 인터넷), C. SSH local port forward.

mesh-vote 5/5 만장일치로 **A. Tailscale serve** 가 떨어졌다. WSL 이 처음엔 B 를 추천했는데, **본진 verify 가 "맥미니 cloudflared queue 라이브 위에 얹기" 라는 B 의 전제를 stale 로 굳히자** WSL 본인이 자기 안을 뒤집었다. "내 B 전제가 stale → 재활용 자산 0, A 는 5노드+iPhone Tailnet active 가 verified 라 service-token 불필요 → 무료앱/CS 0 정책에 정렬".

그 다음 분기 — Tailscale Serve enable 시 Funnel checkbox 까지 켤지 — 에서 trio-vote 가 발동했다. 결과는 **3-0 (a) HTTPS 만 ON, Funnel uncheck**. PM 페르소나가 처음엔 (b) 둘 다 ON 으로 갔다가 비판론자의 한 줄에 뒤집혔다.

> "Funnel 켜진 상태 + endpoint별 실수 1줄 = /ping 외부 expose → sqlite ping spam 폭주 위험. admin toggle 30초 reversible 이라 사전 활성화 이득 0."

> **배운 것** — "혹시 모르니 미리 켜두기" 는 attack surface 만 키운다. reversible 한 toggle 일수록 필요 시점에 켜는 게 옳다.

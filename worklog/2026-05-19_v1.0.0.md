# 2026.05.19 작업일지 v1.0.0 — production 의 silent fail 한 건이 freeze 된 앱을 깨웠다

> 09:00, 형님이 "메모요 메모 내보내기가 silent 라" 라고 트리거를 던졌다.
> 09:54, 50분 만에 1.0.6+24 surgical fix 4건이 본진 main 으로 박혔다.
> 17:00 즈음, 5월 9일에 동결됐던 메모요가 1.0.7 메이저 4항목 묶음으로 깨어났다.

오늘은 production iOS 메모요에서 silent 무반응 한 건이 어떻게 freeze 된 앱을 깨우는 trigger 가 됐는지 — 그 흐름이 9시간 안에 압축된 날이었다. iOS 26 + share_plus 10.1.4 의 breaking change 한 줄이 root cause 였고, 그게 1.0.6+24 핫픽스로 닫히면서 형님이 1.0.7 메이저 4항목 묶음을 ack 했다. 그 사이에 fleet-clear rescue 와 fleet-state smoke test 3개 PR 이 본진 self-PR 머신 모드로 즉시 머지됐고, 권한 harness 거부 hard rule 이 globals 로 승격됐다.

## 1. 09:00 — silent 한 건이 root cause 까지 닿는 데 50분

아침에 형님 텔레그램 한 줄. "production 1.0.5+23 iOS 에서 AppBar overflow → 메모 내보내기 탭해도 아무것도 안 떠." Xcode console 도 깨끗. 진행 표시도 에러 토스트도 없는 dead-end.

본진 iPhone 17 (iOS 26.3.1) 에 debug install 한 뒤 console attach 하니 그제야 PlatformException 이 보였다. 호출부에 `sharePositionOrigin` 인자 누락 → share_plus 가 throw → onTap async 콜백에 try/catch 없어 Flutter 이벤트 루프가 그 예외를 swallow. 사용자에겐 silent.

진짜 함정은 패키지 changelog 가 아니었다. iOS 26 부터 share_plus 10.1.4 가 iPad style presentation 요구 변경에 따라 `sharePositionOrigin` (Rect) 인자를 옵션에서 필수로 바꿨는데, 이 변경이 share_plus 의 release note 가 아니라 **iOS 호스트 OS 메이저 bump 와 함께 풀어졌다**. 패키지 버전만 보고 있으면 안 잡힌다.

> **배운 것** — native bridge 패키지의 breaking change 는 패키지 changelog 가 아니라 OS 메이저 bump 직후 dogfood path 1회 재검증으로만 잡힌다. iOS 26 출하 직후 일주일 안 share_plus 쓰는 모든 앱 호출부 재검증.

09:54 KST, surgical fix 4건이 본진 main 직접 push 로 박혔다 (`018753f`). Rect import, `sharePositionOrigin: Rect.fromLTWH(0, 0, 0, 0)` 명시, XFile mimeType 추가, onTap async 콜백 전체 try/catch + SnackBar surface, 버전 bump. 형님 폰 debug install 재현 → share sheet 정상 출현 → console PlatformException 0건. 50분.

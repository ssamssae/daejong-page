## 1. 3 → 2 → 1, 순서가 거꾸로 풀린 이유

본진은 가장 작은 일부터 잡았다. `~/.ssh/config` 에 `Host mac-mini` 가 명시 등재 안 된 상태였다. 그런데 `ssh mac-mini` 가 작동하고 있었다. 답은 Tailscale MagicDNS 였다 — `mac-mini` 가 자동으로 `100.120.156.7` 로 풀리고 있었던 거다. "잘 되는데 깔끔하진 않은" 상태였다.

> **배운 것** — 작동하는 것과 정의된 것은 다르다. MagicDNS 가 우연히 메워주는 갭은 다른 노드 환경에서 깨질 수 있다.

(2) 박는 작업은 hook 두 개의 노드 리스트 수정이었다. `stop-sync-skills.sh` 와 `posttooluse-sync-skills.sh` 가 `wsl + mac-mini` 2노드만 sync 하고 있었고, 데스크탑/노트북은 빠져 있었다. claude-automations 의 twin hook 은 이미 4노드 완비. 같은 패턴으로 통일했고, 덤으로 WSL sync 의 `wsl-directive.sh` tmux paste (WSL 챗봇을 깨우는 토큰 소모 경로) 를 SSH 직접 호출로 바꿔 토큰 절감까지 박았다.

(1) — 노트북3060 + 데스크탑3060Ti 두 노드에 `git clone git@github.com:ssamssae/claude-memory.git` directive 발사. 두 노드 다 PASS, 본진 최신 commit `7870a4c` 까지 sync 양 노드 verify. 5노드 sync 인프라 본 사이클 매듭.

## 3. + 버튼이 거부한 메모요

여기서부터 까다로워졌다. mac-mini 에 adb 가 미설치였다 — Android 빌드 엔진인데 platform-tools 가 없는 갭. `brew install --cask android-platform-tools` 박고, Galaxy S24 detect PASS. 메모요 데이터 클리어 후 ⋮ → "Drive 에 백업" tap 결과는 "내보낼 메모가 없습니다" SnackBar. 직전 사이클의 "OAuth 캐시 stale" 가설이 흔들렸다 — 메모 0건 가드에 먼저 막힌 거였다.

메모 1건 추가 자동화가 필요했다. + FAB tap `(540, 1773)` — `adb shell input tap` 발화는 되는데 Flutter FAB 가 native input 안 받음. touchscreen tap, long-swipe, 좌표 미세 조정 다 0. ⋮ 메뉴와 메뉴 안 항목은 정상 작동, FAB 만 거부.

## 5. SAF picker 가 알려준 진실

본진은 SAF picker 자율 자동화로 진입했다. `루트 표시` drawer 좌표 추출 → `다운로드` 항목 → dummy JSON tap. "메모 1개 가져왔습니다 (전체 1개)" SnackBar PASS. 손 0 으로 메모 1건 박힘.

그런데 SAF picker drawer 안에서 본진은 다른 걸 발견했다 — **Drive 의 `메모요 베타테스터` 폴더 위쪽에 `memoyo-export-20260519T224538.json` (182바이트, 2026-05-19 박힌) 파일이 이미 존재**했다. 즉 메모요 1.0.7 의 Drive 백업 기능 자체는 본 사이클 어느 시점에 한 번은 작동했다는 증거다. 코드 동작 검증의 본질은 이 한 줄로 PASS.

남은 silent fail 사고는 새로 시도한 백업의 `SignInHubActivity` 즉시 destroy 흔적 — logcat 의 SurfaceFlinger 가 reparent to OffscreenRoot 흔적을 잡았다. sign-in flow 진입은 PASS 하지만 사용자 입력 받기 전 silent destroy → null 반환. 폰의 OAuth 캐시가 "거부됨" 상태로 cache 된 가설 강해졌다.

자정을 막 넘긴 시각, 5노드를 한눈에 보는 대시보드 "초소"를 손보기 시작했다. 처음엔 백로그 섹션 하나 추가하는 정도였는데, 형님의 실시간 지시가 한 줄씩 떨어질 때마다 화면이 통째로 다시 그려졌다. 한 시간 뒤 열한 개의 커밋이 쌓였고, 마지막 한 개는 내가 직전에 만든 버그를 되돌리는 일이었다.

## 1. 백로그 두 개로 시작한 일

초소에는 5노드가 지금 뭘 하는지, 어떤 할일이 큐에 쌓여 있는지가 타일로 떠 있다. 첫 작업은 파킹랏(언젠가 백로그)과 딥워크 두 섹션을 흐린톤 접기 형태로 붙이는 거였다(#15). 그런데 붙이고 나니 노드 분류가 어긋났다. 할일 라인 맨 앞에 `(2026-05-21)` 같은 등록일 prefix가 붙으면서, 노드를 감지하는 로직이 그 괄호에 걸려 13건을 엉뚱한 데로 보내고 있었다(#16). prefix를 무시하도록 고치니 노드큐가 제자리를 찾았다.

## 2. 한 줄씩 떨어지는 지시, 한 줄씩 다시 그려지는 화면

여기서부터가 본론이었다. 형님이 폰으로 화면을 보며 한 줄씩 주문을 넣었고, 나는 받는 족족 반영했다.

- 등록일 badge를 카드 앞에 흐리게 — 오래된 것부터 처리하라고(#17, #18).
- 파킹랏·딥워크도 공용 큐와 똑같은 타일로(#19).
- 노드큐를 위로 올리고, 노드큐도 카드 타일로 통일(#20).
- 색을 갈라서 한눈에 구분 — 노드큐는 티얼, 공용은 주황, 백로그는 슬레이트블루. 3색 체계가 이때 확정됐다(#21).
- 카드 높이를 고정하고(#22), 제목을 2줄로 늘리고(#23), 한 행에 6장씩 132px로(#24).

열 번의 PR이 머지됐다. 01:43, 화면은 깔끔했다. 끝난 줄 알았다.

## 3. 폰에서만 잘려 있던 날짜

7분 뒤 형님이 폰 스크린샷을 보냈다. 카드 위쪽 등록일이 죄다 "2026-0"에서 끊겨 있었다. "컴퓨터에서는 그게 맞는데 폰에서는 이거 일자로 나와야지."

컴퓨터에선 멀쩡한데 폰에서만 잘린다 — 전형적인 반응형 문제였다. 범인은 방금 머지한 #24였다.

```css
@media (max-width: 540px) {
  ...
  #shared .shared-strip, .backlog-strip { grid-template-columns: repeat(2, 1fr); }  /* 폰: 2칸 */
}

/* ↓ #24가 미디어쿼리 뒤에 추가한 무조건 규칙 */
#shared .shared-strip, .backlog-strip { grid-template-columns: repeat(6, 1fr); }  /* 6칸 */
```

폰을 위한 2칸 규칙은 미디어쿼리 *안*에 있었다. 그런데 #24가 6칸 규칙을 미디어쿼리 *뒤에* 무조건으로 박아버렸다. CSS는 같은 우선순위라면 나중에 나온 규칙이 이긴다. 그래서 폰에서도 6칸이 2칸을 덮어썼고, 카드가 절반 이하로 좁아지면서 10글자짜리 날짜가 잘려 나간 거였다.

## 4. 한 줄을 위로 옮기다

고친 건 딱 하나, 6칸 규칙을 미디어쿼리 *앞으로* 옮긴 것뿐이다. 그러면 미디어쿼리의 2칸 오버라이드가 다시 마지막 발언권을 갖는다. desktop은 6칸 그대로, 폰은 2칸으로.

맥미니가 git pull로 받아 서빙하는 CSS에서 규칙 순서가 6칸 → @media → 2칸으로 정렬된 걸 확인했다. 형님이 폰을 새로고침했다. "오 두개씩나온다 좋어." 날짜가 온전히 한 줄로 떴다.

> **배운 것** — 미디어쿼리는 base 규칙보다 항상 *뒤에* 와야 이긴다. 나중에 base 규칙을 끼워넣으면 공들여 만든 반응형 오버라이드가 소리 없이 무력화된다. CSS에서 소스 순서는 곧 캐스케이드 우선순위다.

## 5. 그래서 뭐가 남았나

초소 대시보드가 새 옷을 입었다. 5노드 큐가 색으로 갈리고, 백로그 두 섹션이 붙고, 카드가 균일한 타일로 정렬되고, 폰에서도 날짜가 안 잘린다. 들어간 비용은 새벽 한 시간과 11개의 커밋. 마지막 한 개는 아홉 번째가 만든 함정을 메우는 데 썼다.

화면을 빠르게 이터레이션하면 빠른 만큼 자주 발을 헛디딘다. 그래도 한 줄짜리 수정으로 되돌릴 수 있다면, 그건 빠르게 가도 되는 길이다.

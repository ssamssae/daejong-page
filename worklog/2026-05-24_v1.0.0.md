## 1. "양방향 PASS" 가 stale 단정이었다

직전 사이클 핸드오프엔 "양방향 8 채널 4/4 PASS" 가 또렷이 박혀 있었다. 본진 1:1 채널 4 개에서 mac-report reverse ack 가 다 도착했으니 자연스러운 결론처럼 보였다. 형님이 다시 검증을 가자고 했을 때도 처음엔 같은 검증을 반복할 거라고 생각했다.

4 노드에 verify probe 디렉티브를 fan-out 하고 reverse mac-report 4 개가 다 본진 1:1 에 도착했다. 풀그린 보고를 박았다. 그러자 형님이 그룹 채팅 스크린샷을 보내왔다. "agent mesh mirror there is no macmini."

Agent Mesh Mirror 그룹에는 🪟 WSL / 🖥 데스크탑 / 💻 노트북 의 verify ack 가 차곡차곡 떠 있는데, 🏭 맥미니만 비어 있었다. 본진 1:1 4 채널만 검증하고 그룹 mirror 측은 미검증이었던 핸드오프의 stale 단정이 정확히 한 노드만 누락된 형태로 surface 된 거다.

> **배운 것** — "PASS 박힘" 도 박혀 있는 표면만 확인했을 수 있다. 핸드오프의 negative 단정 ("X 부재") 만큼이나 positive 단정 ("X PASS") 도 verify 1회 의무가 필요하다.

## 2. 같은 봇이 두 키에 다른 토큰으로 박혀 있었다

진단은 외관 → 내부로 좁혀 들어갔다. forward-to-group.sh 코드는 본진 SoT 와 맥미니에서 sha256 동일. .env 의 키 이름도 다 있었다. ssh mac-mini 에서 직접 sender=macmini 로 그룹 발사를 하면 exit 0 이 떨어졌다. silent fail 의 전형이다.

`bash -x` 로 trace 를 살리니 토큰 값이 노출됐다. 본진 SoT 토큰과 맥미니가 들고 있는 토큰이 secret 부분에서 달랐다. curl getMe 두 번 호출하니 답이 갈렸다:

```
=== MAC SoT token getMe ===
{"ok":true,...,"username":"ssamssae_claw_bot"}

=== MACMINI token getMe ===
{"ok":false,"error_code":401,"description":"Unauthorized"}
```

맥미니 `.env` 안에 같은 봇이 **두 키로 박혀 있고 secret 부분이 달랐다**. 폴링용 `TELEGRAM_BOT_TOKEN` 은 valid 였고, forward-to-group sender routing 용 `TELEGRAM_BOT_TOKEN_MACMINI` 만 stale (401) 이었다. 그래서 폴링/reply 는 정상, 그룹 발사만 silent fail. 그게 "맥미니 살아있는데 그룹만 빠짐" 의 정확한 패턴이었다.

backup 파일 7 개를 trace 해보니 stale 토큰이 박힌 시점은 **2026-05-14 00:57 KST** — 맥미니 → Claude Code 5번째 노드 전환 시점. 약 10일간 silent 였던 빚이 오늘 그룹 mirror 검증에서 surface 됐다.

## 3. 노트북도 다른 방식으로 reply 하고 있었다

맥미니 closure 직후 형님이 또 다른 스크린샷을 보냈다. "표준화가 안되어있네 노트북은 그냥 이모지 하나만 보내는데? 터미널응답이 없고."

verify ack 사이클에서 🪟 / 🏭 / 🖥 3 노드는 "💬 터미널 응답\n\n<이모지>" 표준 형식이었는데 💻 노트북만 헤더 없이 이모지 1자 단독이었다. 또 표준화 깨짐. 또 silent.

`telegram-stop-ping.sh` 양쪽을 diff 했다. 48 lines, 1 글자도 다르지 않았다. 코드는 깨끗했다.

차이는 챗봇 행동이었다. stop hook 안에 "이 turn 에서 reply tool 이미 호출됐으면 skip (이중 송신 방지)" 가드가 있다. 정상 3 노드는 verify 응답으로 터미널에 이모지 1자만 출력 → hook fire → 헤더 mirror. 노트북만 reply tool 을 직접 호출 → `replied>0` → hook skip → 헤더 누락.

원인 추정은 단순했다 — 본진 디렉티브 본문이 "응답 💻 1자만 **reply**" 라고 박혔는데, "reply" 단어를 노트북 챗봇만 reply tool 호출 의미로 해석했다. 다른 노드는 자연어 "응답하다" 로 해석.

> **배운 것** — 같은 시스템에서 같은 코드가 돌아도, 한 단어의 해석 갈림이 표준 어긋남으로 surface 된다. 디렉티브 본문의 wording 자체가 forcing function 의 일부였다.

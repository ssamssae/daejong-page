# 본진은 grunt 를 안 한다

> 23:17 KST, 형님 폰 스크린샷이 한 장 도착했다. queue.kangdaejong.com — 어제 끝낸 newsletter 작업이 24시간이 지나도록 "지금 작업중 LIVE" 에 박혀있었다.

뉴스레터 Ep20 ("코드는 해자가 아니다") 을 세 채널에 마감하고 23:13 KST 에 세션을 닫았다. 짧은 휴지. 23:15 KST 에 다시 들어왔을 때 형님이 던진 문장은 평범했다. "뭐할까."

본진이 핸드오프 carry+new 4개 옵션을 정리해 올렸지만, 형님의 진짜 다음 한 마디는 폰 화면 한 장이었다. 그 안에 세 가지 이상함이 동시에 박혀있었다. 본진이 어제 23:13 KST 에 끝낸 Ep20 newsletter 작업이 LIVE 섹션에서 사라지지 않았다. 상단 5타일은 맥미니만 🟢 ACTIVE 인데 하단 "지금 작업중" 섹션은 그 맥미니를 "대기" 로 보내고 자고 있는 본진을 LIVE 로 표시했다. WSL 과 데스크탑 카드에는 task title 자리에 `ssamssae`, `user` 같은 username artifact 가 노출돼 있었다.

세 증상을 셋 다 한 덩어리로 데스크탑에 위임했다. 본진이 grunt 코드를 직접 짜지 않는 룰은 이런 순간 작동한다. 데스크탑 idle 3분, 챗봇 노드, 즉응 가능. 2.7KB directive 한 통.

10분 만에 PR #33 이 들어왔다. `desktop/choso-live-mismatch-fix-2026-05-26`. 153 줄 추가, 2 줄 삭제. 두 가지 코드 변경과 16개 회귀 테스트.

```python
NODE_TASK_TTL_SEC = 30 * 60
NODE_TASK_TERMINAL = {"done", "blocked", "cancelled"}
```

LIVE 섹션이 node_tasks 를 무한히 들고 있던 함정에 30분 TTL 과 terminal status 즉시 만료가 박혔다. 그리고 hook label 의 영문→한글 변환기 `hangeul_label` 의 unknown fallback 이 원문 그대로 통과시키던 함정 — 공백과 슬래시가 없는 bare 단일 단어는 빈 문자열로 바꿔서, `ssamssae` `user` 같은 `$USER` artifact 가 LIVE 카드에 노출되는 길을 막았다.

본진 검증 → squash 머지 `d381250` → 맥미니 launchctl kickstart com.daejong.choso-uvicorn (PID 81926 → 22181) → 127.0.0.1:7777 LIVE 섹션 정합 확인. 셋 다 사라졌다.

> 굿나잇을 들어가는데 노트북에서 보고가 도착했다. 다른 종류의 사고였다.

매 turn 마다 노트북 stop hook 체인에서 같은 에러가 떴다. `bash: /home/user/.claude/hooks/handoff-check.sh: No such file or directory`. 실파일은 어제 2026-05-25 09:15 KST 에 `_disabled/` 로 archive 된 상태였다. mesh-vote D 다수결로 결정됐고, 자연어 디폴트 룰이 박힌 후 forcing function 으로서의 가치를 잃었다는 사유까지 `RETIRED.md` 에 한 줄 박혀있었다.

가역 archive 룰은 지켜졌다. 그런데 settings.json 의 Stop hook 체인 참조가 5노드 다섯 군데 중 본진과 데스크탑만 cleanup 됐고 나머지 셋은 stale 상태였다. settings.json 은 노드별 local 파일이라 한 곳 수정해도 다른 노드에 자동 sync 되지 않는다는 사실이 그제서야 표면으로 올라왔다.

5노드 점검 한 줄 SSH grep 으로 stale 노드 3개를 잡았다. 맥미니 line 194, WSL line 180, 노트북 line 255. 본진 디렉티브 1.6KB 를 셋에 동시 발사했다. 4분 안에 세 PASS 보고가 왔다. 그리고 그 안에서 노트북이 본진의 잘못된 가정 한 줄을 잡아냈다.

> 본진 디렉티브에서 "settings.json 은 git tracked 가역" 이라 했는데 노트북에서 `~/.claude` 는 git repo 아니에요.

노트북 `~/.claude` 는 git tracked 가 아니었다. 가역은 cp 백업과 Edit diff 로만 가능했다. 본진이 자기 환경 가정을 일반화해서 디렉티브에 박은 거였다. 다른 노드에 silent fail 가능성이 있었던 셈인데 노트북이 catch 했다. Karpathy 룰 1 의 노드 방향 적용. 형님이 박은 "형님 발화가 사실과 충돌하면 명확히 지적" 룰의 노드 방향 변종이기도 했다.

이슈 두 건을 박았다. 하나는 초소 LIVE TTL — 같은 sync model 패턴이 새 노드 추가 시 재발 가능. 다른 하나는 hook archive refs gap — archive 룰의 명시되지 않은 빈틈. 둘 다 forcing function 을 예방 섹션에 한 줄씩 박았다.

오늘 짚인 결은 결국 같은 모양이었다. 본진이 자기 손으로 grunt 를 잡으려고 하지 않을 때, 노드 다섯 개가 각자 역할을 한다. 본진은 형님 한 마디를 받아 사실 구조를 파악하고 위임 픽을 결정한다. 데스크탑은 코드 fix 를 들고 오고, 맥미니는 빌드/배포 엔진을 돌리고, 노트북은 본진의 부정확한 가정을 catch 한다. 다섯 노드의 일관성이 깨졌을 때 cleanup 디렉티브 한 통이 4분 만에 정합을 회복시킨다.

본진은 자기 grunt 를 안 한다. 본진은 일을 나눠준다. 오늘의 결은 그 한 줄로 정리됐다.

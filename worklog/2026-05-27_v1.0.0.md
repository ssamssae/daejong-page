## 오늘 배운 거

1. **메모리 룰 narrow 해석 의무.** OAuth 토큰 정책 룰이 playwright 브라우저 스크랩까지 확장된다는 단정으로 멀쩡한 인프라 끄고, 다음 세션이 형님 정정 3회 받고서야 잡았다. 메모리 룰을 코드 정리에 적용하기 전 실제 본문 Read 1회 + 정확한 정책 매핑 필수.
2. **`_disabled/` 보관 시 RETIRED.md 사유 한 줄 의무.** 사유 미박힌 _disabled/ 파일은 다음 세션 진입 시 자동 surface 룰 박을 가치.
3. **핸드오프 본문 "LIVE" 단정 시 객관 verify 명시 필수.** "5타일 다 🟢 active LIVE" 같은 self-attestation 만으로 다음 세션 진입을 거짓 SoT 위에 앉히지 말 것. curl X PASS / DB row 박힘 sha N 식 객관 증거 동행.
4. **"statusline" 한 단어가 셋 해석 가능했다.** 본진이 본인 PS1 으로 첫 응답한 게 핀트 어긋남 1차. Karpathy 룰 1 (가정 명시) 의 형님-방향 위반.
5. **노드 fan-out 시 `AGENT_MSG_NOTIFY_SKIP_DEDUP=1` envvar 명시.** 4 노드 동시 발사하는 스킬 (loop-fleet, mesh-vote, parallel-cycle) 본문에서 이 envvar 박지 않으면 형님 폰 알림 3/4 silent skip. opt-out 자체는 이미 박혀있는데 사용 안 된 게 사고.

# 2026.06.04 작업일지 v1.0.0 — 맥에는 flock 이 없었다, 그리고 메모요는 하루 만에 자랐다

> 00:04, 본진은 "지금 클리어하겠습니다"라고 말하고는 — 아무것도 하지 않았다.
> 09:31, 같은 경고가 다시 떴다. 본진은 또 말만 했다.
> 그리고 누군가 hook 안을 들여다봤을 때, macOS에는 처음부터 `flock` 이 없었다.

오늘은 두 종류의 침묵으로 시작했다. 하나는 본진이 "하겠다"고 말하고 행동하지 않은 침묵, 다른 하나는 동기화 hook이 에러 한 줄 없이 죽어 있던 침묵. 둘 다 "되고 있다"는 착각을 만들고 있었다. 오전을 그 두 침묵을 부수는 데 썼고, 오후부터 다섯 노드가 풀려 메모요 1.0.7을 하루 만에 완성하고, 첫이름 작명 서비스를 prod까지 밀었다.

## 1. 말만 하는 forcing function

40% 하드클리어는 forcing function이다. 컨텍스트가 40%를 넘으면 Stop hook이 "지금 session-clear 하라"고 block한다. 그런데 본진은 그 지시를 받고 "네, 클리어하겠습니다"라고 답한 뒤 — 클리어하지 않았다. forcing function이 강제하지 못하는 forcing function. 09:31에 같은 경고가 다시 떴고, 같은 일이 반복됐다.

라이덴(WSL)이 복구 트리거를 잡았고, 본진은 백스톱을 박았다. **`1e9eece` (PR #80)** — "40% block이 말로만 새는" 경로 자체를 차단해서, 다음 정지 시 강제 큐에서 자동 실행되게.

> **배운 것** — 행동을 강제하지 못하는 강제 장치는 장식이다. block 메시지를 출력하는 것과 그 다음 턴을 실제로 막는 것은 다른 일이다.

## 2. macOS에는 flock 이 없었다

더 깊은 침묵은 동기화 쪽이었다. 5노드 sync hook 네 개가 `flock`으로 동시성을 잠그고 있었는데 — macOS 본진에는 `flock`이 설치돼 있지 않다. hook은 `flock: command not found`로 즉시 종료됐고, `|| exit 0` 패턴이 그 "명령 부재"까지 삼켜서, 본진은 한 번도 동기화하지 않으면서 "동기화 중"이라고 믿고 있었다.

고침은 한 줄이었다. `command -v flock` 가드로 명령이 있을 때만 잠그게 (**`a82a677`**). 그리고 5노드의 commit sha를 실측해서 진짜로 다시 붙는지 확인했다. "한 기기선 되는데"는 환경차 의심 신호였다.

> **배운 것** — Linux 전용 명령(`flock` 등)을 5노드 공용 hook에 쓸 땐 `command -v` 가드가 필수다. 자동화의 성공은 로그와 실측 sha로 검증한다 — "에러가 안 떴으니 됐겠지"가 아니라.

이 사건은 그대로 ep26 뉴스레터 소재가 됐다. 제목도 "맥에는 flock 이 없었다".

## 3. 메모요가 하루 만에 자랐다

오후, fleet의 한 줄기가 메모요 1.0.7에 붙었다. 1.0.7은 네 기능 묶음이었다 — ① Drive 백업 1버튼화, ② 삭제 UNDO, ③ 공유, ④ 휴지통 30일. 라이덴이 구현하고, 맥미니가 네이티브 `flutter test` 게이트로 받았다.

| 단계 | PR | 게이트 |
|---|---|---|
| soft-delete 데이터모델 + 30일 purge | #58 | — |
| 리스트 soft-delete 전환 | #59 | 네이티브 PASS → 머지 |
| 휴지통 화면 + 자동 purge 트리거 | #60 | 네이티브 PASS → 머지 |
| Drive 백업 1버튼화 (백업&복원 통합) | #61 | 네이티브 PASS → 머지 |

저녁 20:13, **1.0.7+28** 릴리스 빌드가 나왔고 스토어 제출까지 갔다. 핵심은 게이트가 `analyze`가 아니라 맥미니의 진짜 `flutter test`였다는 것 — 이게 왜 중요한지는 다음 섹션이 증명한다.

## 4. analyze 가 통과시킨 빨강 4개

같은 시간 다른 줄기는 첫이름(cheotireum) 작명 서비스였다. 노드 하나가 입력 UX 개편을 끝내고 `flutter analyze` PASS로 머지를 요청했다. analyze는 깨끗했다. 그런데 본진이 네이티브 `flutter test`를 돌리자 위젯 테스트 4건이 빨갛게 떴다.

원인은 `ListTile`이었다. 색 카드(`DecoratedBox`) 안에 `ListTile`을 Material ancestor 없이 넣으면, 정적 분석은 통과하지만 위젯이 조립되는 순간 assertion이 터진다. analyze가 볼 수 없는, 런타임에만 드러나는 빨강. **`0754784` (PR #16)** — ListTile 위생 + 테스트 현행화로 4건 green 회복.

> **배운 것** — `flutter analyze` PASS는 런타임 위젯 assertion을 보증하지 않는다. 노드가 analyze-only로 올린 Flutter PR은 머지 전 본진 네이티브 test가 권위다.

첫이름은 이 외에도 동명이인 근사(출생신고 인기도 기반 popularity/rarity, PR #12), worker graceful 400(PR #14), 결정론 레이어 도메인 리서치(인명용한자 9389자·사주 오행매핑, 노트북), 다크 프리미엄 시안 2종(데스크탑)까지 네 노드가 병렬로 밀었다.

## 5. "화면 같으니 이대로" 가 막힌 곳

메모요 Play Store 스크린샷을 만들 때, 한 노드가 iOS 시뮬레이터로 캡처한 뒤 "Flutter라 화면 같으니 이대로 쓰자"고 제안했다. 당장은 같아 보였다. 하지만 아니키가 끊었다 — Android 정석으로 재캡처하라고. 그리고 글로벌룰이 하나 섰다: **"항상 정석대로"**, 편의 shortcut을 본진이 먼저 합리화하지 말 것. **`ee8c441`**로 Android 정석 재캡처, ko-KR 업로드까지 완료.

> **배운 것** — 같아 보이는 우회는 correctness 부채로 쌓인다. 정석이 진짜로 막혔거나 비용이 비대칭일 때만 우회를 surface하고, 그 외엔 정석이 디폴트다.

## 6. 그래서 뭐가 남았나

하루의 결과물:
- **메모요 1.0.7+28** 릴리스 빌드 + 스토어 제출 (①②③④ 전부)
- **첫이름** prod worker 배포, 동명이인·입력 UX·worker 안정화 5개 PR 머지
- **인프라 누수 2건 봉합** — 40% 클리어 백스톱, flock `command -v` 가드 (5노드 동기화 회복)
- **ep26 뉴스레터** Substack·네이버·홈페이지 3채널 발행
- lottocalc iOS 흰화면 검증, 2번째 마이크로SaaS 라인 컨셉 비교 초안

밤엔 미리알림 메모를 뒤지다 오래된 구상 하나를 다시 꺼냈다 — 에뮬레이터에 쿠팡이츠를 띄워 화면을 비전으로 읽고 주문까지 누르는 자율 주문 에이전트. 그리고 "카나리아"라는 단어를 따라가 보니, 노트북을 첫 시험대로 썼던 챗봇 자동기동 작업이 거의 끝나 있었다. 광부의 카나리아처럼, 하나를 먼저 보내 안전을 확인하고 나머지를 따르게 하는 — 그게 오늘 fleet이 일한 방식이기도 했다.

남은 건 첫이름 다크톤 배포와 작명엔진 결정론 레이어 실구현, 그리고 메모요 심사 통과 모니터.
